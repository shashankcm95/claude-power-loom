# Runtime: verify (failure-signature, spawn-verify), test-runners, trait-resolve — `packages/runtime/{verify,test-runners,contracts/_lib}`

> This cluster is the **runtime (orchestration) verification tier** — the R9/R11/R12 "VERIFY" half of the v3.2 decomposition pipeline plus the v3.1 PR-1 capability-trait composer. It is **best-effort orchestration, NOT kernel-enforced**: `verifySpawn` (R11) decides whether a decomposition leaf is admissible by routing on the leaf's discipline, running its `tdd` test in a path-scoped/resource-bounded subprocess (R12 `node-runner`), and emitting an ADR-0015 `failure_signature` (a frozen 8-field negative-attestation witness) on rejection. `trait-resolve.js` is a pure, currently-dormant primitive that composes capability traits into a resolved capability object, consumed only by the static `contracts-validate.js` `traits-resolve-clean` validator (no runtime gate calls it yet). The trust boundary throughout is the harness `isolation:worktree`; the kernel cannot wrap subprocesses (ADR-0012), so R12 adds defense-in-depth (path-scope, no-shell, least-privilege env, hard timeout), explicitly punting network/egress/forkbomb/full-output-DoS to the ContainerAdapter tier.

## Directory contents & nesting

| Folder | File | One-line purpose |
|---|---|---|
| `packages/runtime/verify/` | `failure-signature.js` | R11 producer's `failure_signature` builder + in-code enum validator (ADR-0015 frozen 8-field shape). |
| `packages/runtime/verify/` | `spawn-verify.js` | R11 dispatcher: routes a leaf through R9 structural admission then (for `tdd`) an R12 test-run; emits a `failure_signature` on rejection. |
| `packages/runtime/test-runners/` | `index.js` | Public barrel — re-exports the registry plus the `node` adapter for direct access. |
| `packages/runtime/test-runners/` | `node-runner.js` | R12 — the only LIVE test adapter; runs `node <testFile>` with path-scope + timeout + least-privilege env, returns a frozen pass/fail result. |
| `packages/runtime/test-runners/` | `registry.js` | R12 adapter registry (Map-backed, Open/Closed); registers the `node` built-in; surfaces `getAdapter` / `isVerificationSupported` / kind lists. |
| `packages/runtime/contracts/_lib/` | `trait-resolve.js` | v3.1 PR-1 pure trait composer — intersect narrowing axes / union broadening axes into a resolved capability object. |

Nested subfolders in scope: `contracts/_lib/` holds the single private trait-composition primitive (`_`-prefix = package-internal, not a public contract surface). There is no `_spike/` here. The `test-runners/` folder is a flat adapter package (barrel + registry + one adapter). Fixtures for these modules live under `tests/unit/runtime/{verify,test-runners}/fixtures/` (`*.fixture.js`, deliberately NOT `*.test.js` so the test harness never auto-runs them — they are run only through the adapter).

## Per-file analysis

### `packages/runtime/verify/failure-signature.js`

- **Purpose** — The R11-producer's builder for the ADR-0015 `failure_signature` (the negative-attestation witness emitted when a leaf is rejected). Hand-written enum spot-checks mirror `packages/kernel/schema/failure-signature.schema.json` (the FROZEN documented contract the v3.3 E2 consumer reads); a fitness test asserts the in-code enums set-equal the schema file. Fail-closed by construction.
- **Imports / consumes** — None (no `require`). Pure module. No fs / env / process access.
- **Consumers** — `packages/runtime/verify/spawn-verify.js` (imports `buildFailureSignature`, `signatureDiscipline`, `DISCIPLINES`); `tests/unit/runtime/verify/failure-signature.test.js`; `tests/unit/lab/negative-attestation/record-from-decompose.test.js` (imports `buildFailureSignature`); `tests/unit/lab/negative-attestation/store.test.js` references the path. Enum names referenced (as keywords, not imports) by `packages/kernel/algorithms/route-decide.js`.
- **Functions**

  | name | kind | purpose | consumes | writes | state changes / side effects |
  |---|---|---|---|---|---|
  | `buildFailureSignature` | exported | Validate + build a frozen 8-field `failure_signature`; THROWS on any contract violation (malformed witness = producer bug, fail-closed). | `fields` object (`failed_criterion_id`, `discipline`, `verifier_kind`, `detection_phase`, `human_message`, optional `leaf_ref`/`expected`/`observed`) + `STRUCTURAL_ENUMS`. | none | none (pure); returns a new `Object.freeze`d record; throws `Error` on violation. |
  | `signatureDiscipline` | exported | Coerce a leaf's declared discipline into a valid enum member; unknown → `'exploratory'` (catch-all). | `declared` string + `DISCIPLINES`. | none | none (pure). |

- **File-level notes** — Enums are `Object.freeze`d module constants (`FAILED_CRITERION_IDS`, `DISCIPLINES`, `VERIFIER_KINDS`, `DETECTION_PHASES`) re-exported for the fitness/drift guard. Append-only enum discipline is documented. `verifier_kind: 'schema'` and `detection_phase: 'budget-abort'` are RESERVED (no producer emits them yet — confirmed: `VERIFIER_KIND_BY_CRITERION` in spawn-verify never maps to `schema`, and no path passes `budget-abort`). Note the comment at L10-11 claims `schema` is RESERVED and "the other 4 are live" for `verifier_kind`; in practice (see spawn-verify) only 4 of the 5 (`predicate`, `structural`, `registry-lookup`, `test-run`) are produced and `schema` is reserved — internally consistent.

### `packages/runtime/verify/spawn-verify.js`

- **Purpose** — R11, the spawn-verify dispatcher and the PRODUCER of `failure_signature` (it LOCKS `verifier_kind`). Decides leaf admissibility via a two-live-phase lifecycle: (1) `pre-spawn-leaf-check` = R9 `validateLeaf` structural admission; (2) `post-spawn-verify` = for a `tdd` leaf, run its test via R12 and reconcile the real outcome against the declaration. Phase 3 `budget-abort` is RESERVED. Pure of run-state; the only side effect is spawning the leaf's test subprocess (delegated to R12).
- **Imports / consumes** — `require('../orchestration/leaf-criteria')` (`validateLeaf`, R9); `require('../test-runners')` (`getAdapter`, R12 barrel); `require('./failure-signature')` (`buildFailureSignature`, `signatureDiscipline`, `DISCIPLINES`). Reads `process.env.LOOM_VERIFY_RUN_TESTS` (kill-switch). Reads no files directly (R12 does the fs work).
- **Consumers** — `packages/runtime/orchestration/decompose-run.js` (the first live caller — `verifySpawn` in Phase 1 of `runDecomposition`); `tests/unit/runtime/verify/spawn-verify.test.js`; `tests/unit/runtime/contracts/decompose-run.test.js` (integration). Referenced as a keyword by `route-decide.js`.
- **Functions**

  | name | kind | purpose | consumes | writes | state changes / side effects |
  |---|---|---|---|---|---|
  | `leafRef` | internal | Extract a non-empty string `leaf.id` else `null` (the `failure_signature.leaf_ref`). | `leaf` object. | none | none (pure). |
  | `testsEnabled` | internal | Whether to actually run tests: honors `LOOM_VERIFY_RUN_TESTS=0` kill-switch and `ctx.runTests !== false`. | `ctx`, `process.env.LOOM_VERIFY_RUN_TESTS`. | none | reads env; no mutation. |
  | `reject` | internal | Wrap a signature + advisories into a frozen `{accepted:false, verified:false, failure_signature, advisories}` verdict. | `sig`, `advisories`. | none | returns a frozen object (advisories sliced + frozen). |
  | `verifySpawn` | exported | The dispatcher: R9 admission then optional R12 test-run; returns `{accepted, verified, failure_signature, advisories}`; the load-bearing biconditional `accepted===false ⇔ failure_signature!==null` holds on every path including a thrown R12 precondition. | `leaf`, `ctx` (`cwd?`, `runTests?`, `env?`, `timeoutMs?`, `maxBufferBytes?`); calls `validateLeaf`, `getAdapter`, `adapter.run`, `buildFailureSignature`. | none (returns a verdict object) | spawns the leaf's test subprocess via R12 (the only side effect); converts an R12 `run()` throw into a `failure_signature` (never propagates). |

- **File-level notes** — Single-criterion witness BY DESIGN (reports the FIRST error-criterion in R9's evaluation order via `Object.keys(r9.criteria).find(...)`). The `verified` bit is a structural honesty signal: `verified:false` when a `tdd` test was SKIPPED (`runTests:false` / env kill-switch) so a strict caller cannot read a structural-only accept as a verified pass by checking `accepted` alone. `VERIFIER_KIND_BY_CRITERION` maps R9 hard-gate criteria to live `verifier_kind` members and is exported for an OCP fitness test. `semantically-cohesive` is advisory-only and never produces a signature (it is never an error key). Coupling: tightly bound to R9's `criteria` keying contract (criterion-id === `failed_criterion_id` enum, zero re-mapping) and to R12's frozen result shape (`result.passed/exitCode/reason/timedOut`).

### `packages/runtime/test-runners/index.js`

- **Purpose** — Public barrel: `require('packages/runtime/test-runners')` yields the registry (with `node` pre-registered) plus the `nodeRunner` adapter for direct access. Kept import-friendly: no `process.exit` / top-level side effects beyond the registry's built-in registration.
- **Imports / consumes** — `require('./registry')`, `require('./node-runner')`.
- **Consumers** — `packages/runtime/verify/spawn-verify.js` (`const { getAdapter } = require('../test-runners')`). R9 `leaf-criteria.js` imports `./test-runners/registry` directly (not via this barrel).
- **Functions** — None defined. Exports a frozen object spreading `registry` and adding `nodeRunner`.
- **File-level notes** — Side-effect-on-import is the registry's `registerAdapter(nodeRunner)` (transitive via `./registry`), not in this file. `Object.freeze` on the barrel prevents callers from mutating the exported surface (but does NOT freeze the underlying `adapters` Map — see Findings).

### `packages/runtime/test-runners/node-runner.js`

- **Purpose** — R12, the only LIVE test adapter. Runs a leaf's test via `node <testFile>` (the substrate dogfoods plain `node file.test.js`, not jest) and reports a frozen structured pass/fail result. PURE w.r.t. run-state (no budgets/checkpoint writes — keeps concurrent runs free of a shared-counter race). Adds path-scope + best-effort resource-bound + least-privilege env + no-shell exec + hard timeout; explicitly NOT a sandbox.
- **Imports / consumes** — `require('child_process')` (`execFileSync`); `require('fs')` (`lstatSync`); `require('path')`; `require('../../kernel/_lib/path-canonicalize')` (`checkWithinRoot`). Reads `process.env[k]` for `SAFE_ENV_KEYS` only (PATH/HOME/LANG/LC_ALL/TMPDIR/TERM/USER/SHELL). `ctx.testFile` / `ctx.cwd` are absolute by contract.
- **Consumers** — `packages/runtime/test-runners/registry.js` (registers it as the `node` adapter); `packages/runtime/test-runners/index.js` (re-export); reached at runtime by `spawn-verify.js` via `getAdapter('node').run(...)`. R9 `leaf-criteria.js` reaches its `kind` via `getAdapter(runner)`. Tested by `tests/unit/runtime/test-runners/node-runner.test.js` (+ fixtures `pass`/`fail`/`hang`/`overflow`).
- **Functions**

  | name | kind | purpose | consumes | writes | state changes / side effects |
  |---|---|---|---|---|---|
  | `buildChildEnv` | internal | Build a least-privilege child env: only `SAFE_ENV_KEYS` from `process.env` overlaid with caller-declared string `extra` (`ctx.env`). | `extra` object, `process.env`. | none | reads env; returns a new object. |
  | `appliesTo` | exported | Detection: string-only test of whether the `node` adapter applies (`ctx.testFile` ends `.test.js` and runner is undefined/`node`). Safe on untrusted ctx (no I/O). | `ctx`. | none | pure, no I/O. |
  | `buildCommand` | exported | The argv R11 would run, as a frozen array (`{cmd:'node', args:[testFile], cwd}`) — no-shell proof. | `ctx`. | none | pure. |
  | `guardPath` | exported | Validate `testFile` is absolute, within `cwd`, no raw `..` segment, passes `checkWithinRoot`; returns `{ok, reason}`. | `ctx` (`cwd`, `testFile`); calls `path.isAbsolute`, `checkWithinRoot`. | none | pure (no fs in this fn; `checkWithinRoot` does realpath I/O on existing ancestors). |
  | `freezeResult` | internal | Normalize + freeze the result shape (`passed/exitCode/signal/stdout/stderr/timedOut/reason`). | `r`. | none | returns a frozen object. |
  | `run` | exported | Run the leaf test; throws ONLY on a precondition violation (bad path/missing file/symlink/non-file); a test that ran-and-failed returns `{passed:false}`. Discriminates overflow / timeout / self-kill / normal-exit. | `ctx`; `guardPath`, `fs.lstatSync(ctx.testFile)`, `execFileSync('node',[testFile],...)`, `Date.now()`, `buildChildEnv`. | spawns a `node` subprocess (reads its stdout/stderr) | spawns + may SIGKILL the child on timeout/overflow; reads the test file's lstat; NO disk writes by the adapter itself (the child test may write); throws on precondition fail. |

- **File-level notes** — Frozen RESULT SHAPE is a forward contract (R11 emits `failure_signature` from it). `exitCode` is `null` on signal-kill; `reason` ∈ {`null`, `output-overflow`, `timeout`, `killed-by-signal`}. TOCTOU mitigation: `lstatSync` (no-follow) reject-symlink as close to exec as possible after `checkWithinRoot` realpaths at check time — a sub-microsecond residual race is acknowledged ContainerAdapter-tier. The timeout discriminator combines `code==='ETIMEDOUT'` with an elapsed-wall fallback (`signal && elapsed >= timeout-50`) so a fast self-kill is not mislabeled a timeout. Documented OUT-OF-SCOPE (ADR-0012): network/child-process egress by the test, COMPLETE output capture + hard output-DoS bound (a fast synchronous flood truncates at the OS pipe buffer and may not trip `maxBuffer`), detached grandchildren outliving SIGKILL, and a SIGSTOP/uninterruptible hang.

### `packages/runtime/test-runners/registry.js`

- **Purpose** — R12 adapter registry; the surface R9 #4 (`validation-supported`) and R11 consume. Map-backed Open/Closed registration (DIP — consumers depend on this interface, not concrete adapters). Registers the `node` built-in at import. Distinguishes the reserved RUNNER_KINDS namespace (`node`/`jest`/`vitest`/`pytest`) from the verifier_kind axis (R11's lock).
- **Imports / consumes** — `require('./node-runner')`.
- **Consumers** — `packages/runtime/test-runners/index.js` (spread into the barrel); `packages/runtime/orchestration/leaf-criteria.js` (`getAdapter` for R9 #4 light-availability check); transitively R11 via the barrel; `tests/unit/runtime/test-runners/registry.test.js`.
- **Functions**

  | name | kind | purpose | consumes | writes | state changes / side effects |
  |---|---|---|---|---|---|
  | `registerAdapter` | exported | Register an adapter; duck-typed contract `{kind, appliesTo, run}`; rejects a malformed adapter and a DUPLICATE kind unless `{overwrite:true}`. | `adapter`, `{overwrite}`. | none | MUTATES the module-singleton `adapters` Map; throws on malformed/duplicate. |
  | `getAdapter` | exported | Lookup an adapter by kind; `null` if absent. | `kind`, `adapters`. | none | reads the Map. |
  | `listRunnerKinds` | exported | All RESERVED kinds (incl. not-yet-live) — the frozen `RUNNER_KINDS`. | — | none | pure. |
  | `listRegisteredKinds` | exported | Only kinds with a LIVE adapter (`[...adapters.keys()]`, frozen) — honest support. | `adapters`. | none | reads the Map; returns a new frozen array. |
  | `resolveAdapter` | exported | First registered adapter whose `appliesTo(ctx)` is true, else `null` (detection). | `ctx`, `adapters`. | none | reads the Map; calls each adapter's `appliesTo`. |
  | `isVerificationSupported` | exported | R9 #4 surface — "is a test-run adapter available for this ctx?" (`resolveAdapter(ctx) !== null`). | `ctx`. | none | reads the Map. |

- **File-level notes** — Top-level side effect: `registerAdapter(nodeRunner)` at import. `RUNNER_KINDS` is frozen; `jest`/`vitest`/`pytest` are intentionally NOT registered (a detect-only shim would make `isVerificationSupported` lie; a hard-false shim is dead code — YAGNI). The header is explicit that `isVerificationSupported` answers test-run AVAILABILITY, NOT "is this leaf verifiable" — R9 owns the discipline branch (a `spec-driven` leaf must NOT be inferred unsupported from a false here). The `adapters` Map is a mutable module singleton shared across the whole process (see Findings: state leakage / duplicate-registration risk in a long-lived R11 process — mitigated by the duplicate-kind guard).

### `packages/runtime/contracts/_lib/trait-resolve.js`

- **Purpose** — v3.1 PR-1 pure primitive composing capability traits (atomic mixins) into a single resolved capability object per RFC v3.3 §3.2. Narrowing axes (write/subprocess/isolation/network) INTERSECT (tightest wins; empty intersection → throw); broadening axes (read/read_recall) UNION (widest wins). Ships DORMANT — no runtime gate calls it; only the static `contracts-validate.js` `traits-resolve-clean` validator and tests use it.
- **Imports / consumes** — None (pure; tests assert it imports no fs/path/os). Caller passes the parsed `registry` (the `traits/_registry.json` object).
- **Consumers** — `packages/runtime/orchestration/contracts-validate.js` (`resolveTraits` in the `traits-resolve-clean` validator — `declared_capabilities` must deep-equal `resolveTraits(traits)`); `tests/unit/runtime/contracts/trait-resolve.test.js`; `tests/unit/runtime/contracts/traits-registry.test.js`; `tests/unit/runtime/contracts/agent-contract-reconcile.test.js`.
- **Functions**

  | name | kind | purpose | consumes | writes | state changes / side effects |
  |---|---|---|---|---|---|
  | `resolveTraits` | exported | Resolve a list of trait names into a single capability object keyed by axis; throws on unknown trait or empty-intersection conflict. | `traitNames[]`, `registry` (`.traits`, `._axis_direction`). | none | none (pure); throws `Error`. |
  | `mergeTrait` | internal | Fold one trait's axes into the accumulator; default direction BROADENING for an unknown axis. | `acc`, `traitName`, `trait`, `directions`. | none | MUTATES the local `acc` accumulator only (not inputs). |
  | `combine` | internal | Combine accumulated vs incoming axis value by direction (narrowing→intersect, broadening→union). | `existing`, `incoming`, `axis`, `traitName`. | none | pure; may throw via `intersectAxis`. |
  | `intersectAxis` | internal | Intersect two narrowing values (scalars must be equal; arrays keep shared; mixed coerced to arrays); empty → throw. | `current`, `incoming`, `axis`, `traitName`. | none | pure; throws on conflict. |
  | `unionAxis` | internal | Union two broadening values, de-duplicated set-like. | `current`, `incoming`. | none | pure; returns a new array. |
  | `conflict` | internal | Build a descriptive conflict `Error`. | `axis`, `traitName`, `current`, `incoming`, `kind`. | none | pure; returns an `Error`. |
  | `cloneValue` | internal | Shallow copy of an axis value (`.slice()` for arrays, identity for scalars) so output never aliases registry state. | `value`. | none | pure; SHALLOW (see Findings). |
  | `materialize` | internal | Flatten the accumulator into a plain `{axis: value}` object. | `acc`. | none | pure; returns a new object. |

- **File-level notes** — Immutability contract is explicit (inputs never mutated; tests freeze the input array and assert no registry mutation). The per-axis value shape is HETEROGENEOUS (scalar for `isolation: 'worktree'`, arrays for list axes) — documented so a downstream `used ⊆ declared` consumer (the future K6) handles the scalar case. Fail-OPEN default direction (`directions[axis] || BROADENING`) is SAFE only because the registry-schema validator gates unknown axes upstream — the comment flags that an UN-validated registry could silently widen. Current registry data has no nested objects in axis values, so the shallow `cloneValue` is adequate today (a documented assumption, not a present bug).

## Findings (bugs / logical fallacies / optimizations)

| severity | level | type | location (file:line) | description |
|---|---|---|---|---|
| LOW | function | bug | `packages/runtime/verify/spawn-verify.js:85-88` | When `r9.ok` is false, `failedId` is the FIRST criterion with `severity === 'error'`. `validateLeaf` sets `ok = errors.length === 0` (`leaf-criteria.js:229`) but `criteria[id].severity === 'error'` only when a violation's severity is non-advisory. These are consistent today, but `crit = r9.criteria[failedId]` and `crit.violations` (L88-89) would throw `Cannot read properties of undefined` if `failedId` were ever `undefined` (e.g. a future criterion that pushes a non-advisory error into the flat `errors` array but is keyed `severity:'ok'`). No defensive guard on `failedId`/`crit` — a divergence between the two R9 severity computations becomes a crash inside the rejection path (the one path that must never throw, per the biconditional). Recommend guarding `if (!failedId)` with a fallback synthetic signature. |
| LOW | file | smell | `packages/runtime/test-runners/registry.js:28,52` | The `adapters` Map is a mutable module-level singleton shared process-wide and `registerAdapter(nodeRunner)` runs at import. `index.js:14` `Object.freeze`s the barrel object but the Map itself is never frozen, so any holder of the registry export can `registerAdapter(x, {overwrite:true})` and globally swap the live `node` runner for the remainder of an R11 process. The duplicate-kind guard mitigates accidental shadowing, but `{overwrite:true}` is an intentional escape hatch with process-global blast radius — acceptable for the dogfood, worth noting as shared mutable state. |
