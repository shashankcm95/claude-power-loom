# Lab causal-edge spikes (dogfood/derive/rerun) — `packages/lab/causal-edge/_spike/`

> This cluster is the **manual verification-probe / dogfood tier** of the Evolution Lab's `causal-edge` component (`@loom-layer: lab` — advisory/shadow, never enforced). Everything here lives under `_spike/` precisely so the Linux unit-test glob (`tests/unit/**`) never picks it up: these files exercise IMPURE real paths — a live macOS `sandbox-exec` ContainerAdapter, real `claude -p` LLM invocations, real `git`, and real temp-dir FS — that the pure cores (`calibration-issue.js`, `trajectory-friction.js`, `lesson-capture.js`, `lesson-derive.js`) are CI-tested against only with mocks. Their reason to exist is the **Rule-2a-corollary** discipline codified in this repo: a green mock/unit suite is a hypothesis about the path it mocks, never proof the real path works; these spikes drive the novel refuse/derive/capture paths LIVE on the host. They are run-on-demand drivers (`node <file>`) — none is wired into CI, none is a library API (only `lesson-capture-rerun.js` exports anything, and its sole consumer is one other spike). Per the project's own honesty framing (OQ-NS-6), a synthetic/backtest dogfood **NARROWS** confidence only; it does not HARDEN a trust signal.

## Directory contents & nesting

`packages/lab/causal-edge/_spike/` is a nested spike subfolder under the `causal-edge` component. The sibling `causal-edge/` folder holds the PURE machinery (CI-tested); `_spike/` holds the IMPURE drivers that the pure machinery is too dangerous/slow/non-deterministic to host. There is no further `_lib/` nesting.

| File (folder: `_spike/`) | Kind | One-line purpose |
|---|---|---|
| `calibration-issue-dogfood.js` | macOS-only probe (`main()` + `process.exit`) | v3.9 W2 — drives the three-legged issue-calibration scorer end-to-end against a live `sandbox-exec` backend; proves leg A runs real+contained, the C1 test-tree rehash catches a test-tampering candidate, and the recall-gate refuses a behavioral-only pass. |
| `dogfood-derive-sample.js` | reproducible driver (`main()`, no exit code) | v3.11 W1 — N=1 smoke that the REAL `claude -p` lesson-derive leg executes end-to-end through `runCaptureRerun` on one synthetic windowed-boundary bug into a temp store, then loads the minted node back + recovers the candidate from the sidecar. |
| `lesson-capture-rerun.js` | spike module (the only one with `module.exports`) | v3.11 W1/W3 — the IMPURE real-`claude -p` derive leg (`makeLessonDeriver`) + a manual driver (`runCaptureRerun`) over pre-built items, wiring the W3 `failed_patch` trap seam into the contrast prompt; `--dry` swaps in a deterministic stub leg. |
| `trajectory-friction-dogfood.js` | probe (`main()` + `process.exit`) | v3.9 W3 — drives a REAL top-level `claude -p --output-format stream-json` capture on a benign fixture repo through `parseTrajectory`/`computeProcessGraph`, proves the recall-smell discriminates planted recall-vs-honest shapes, and runs the three-valued `validateRecallSmellAgainstControls`. |
| `DOGFOOD-SAMPLE.md` | recorded artifact (not JS; out of analysis scope) | The committed recorded output of one `dogfood-derive-sample.js` run (referenced by it, not in this analysis scope). |

What distinguishes `_spike/` from its parent: the parent `causal-edge/*.js` are pure/seam-injected and unit-tested; `_spike/*.js` invoke real external processes (`git`, `claude`, `sandbox-exec`) and real FS, are explicitly OUT of CI, and are macOS / network-gated (they SKIP or degrade when the bin/platform is absent).

## Per-file analysis

### `calibration-issue-dogfood.js`

- **Purpose** — A macOS-only verification probe (NOT a unit test) that drives the W2-NOVEL refuse paths of the three-legged issue-calibration scorer LIVE. Leg A behavioral runs REAL via the W1 `sandbox-exec` ContainerAdapter backend; it confirms (1) a clean fix scores a real `BEHAVIORAL_PASS` and is recall-eligible, (2) a behavioral-pass with a fail-closed leg B is NOT recall-eligible, and (3) a test-tampering candidate is caught by the C1 test-tree rehash and forced to `BEHAVIORAL_FAIL`. Leg B here is a deterministic stub so the recall-gate demonstration is reproducible (the real claude `-p` leg B/C is exercised elsewhere via `runIssueCalibration`).
- **Imports / consumes** — `fs`, `os`, `path`, `child_process.execFileSync`; `scoreAttempt` from `../calibration-issue`; `makeBehavioralFn` from `../calibration-issue-run`; `createSandboxExecBackend` from `../../issue-corpus/sandbox-exec-backend`. Reads `process.platform`. Writes/reads a freshly `mkdtemp`'d fixture git repo under `os.tmpdir()`.
- **Consumers** — None programmatic. Referenced descriptively in `docs/ACTIVATION-LEDGER.md` and `docs/SIGNPOST.md`. Run manually: `node packages/lab/causal-edge/_spike/calibration-issue-dogfood.js`.
- **Functions**

| name | kind | purpose | consumes (params, files read) | writes (files, refs, stdout/JSON) | state changes / side effects |
|---|---|---|---|---|---|
| `git(args, cwd)` | internal | thin `execFileSync('git', ...)` wrapper returning stdout as string | `args[]`, `cwd`; the git binary | nothing to disk directly (git side effects depend on `args`) | spawns a `git` child process synchronously; throws on non-zero exit |
| `makeFixture()` | internal | build a benign temp git repo: a buggy `add()` (subtracts), a graded `loom-run-tests.js`, a clean-fix patch, and a TAMPER patch that rewrites the graded test | `os.tmpdir()` | `src.js`, `loom-run-tests.js` under a new temp dir; one git commit; returns `{dir, base_sha, cleanPatch, tamperPatch}` | `mkdtempSync` creates a temp dir; multiple `git init/config/add/commit/diff/checkout` mutate that repo; **temp dir is created here** |
| `recordFor(fx)` | internal | construct the calibration record (id/repo/`repo_local`/`base_sha`/`problem_statement`/`accepted_diff`/`fail_to_pass`) for the fixture | `fx` (from `makeFixture`) | returns a plain record object | none (pure constructor) |
| `legs(behavioralFn, semanticSupported, semanticSource)` | internal (arrow) | build the `{behavioralFn, semanticFn, referenceFn}` leg bundle; leg B/C are deterministic stubs | params | returns leg object | none (closures only) |
| `main()` | cli-entry (async) | orchestrate the three `scoreAttempt` runs, compute the four checks, print, and exit | `process.platform`; the backend; the fixture | `console.log`/`console.error` (human + JSON); `process.exit(0/1/2)` | creates the sandbox backend (attests containment), spawns leg-A contained runs, **removes the temp dir** (`fs.rmSync` best-effort in a `try/catch`), sets the process exit code |

- **File-level notes** — Clean fail-closed/cleanup discipline: a `try/catch` wraps the `rmSync`. The `main().catch(...)` at the bottom guards an unhandled rejection (exit 1). One subtlety: the cleanup `rmSync` runs only on the happy path BEFORE the checks — if any of the three `scoreAttempt` awaits throws, control jumps to `main().catch`, the temp dir is NOT removed (a `/tmp` leak on the error path). Contrast with `trajectory-friction-dogfood.js`, which correctly uses `try/finally` for the same concern (and cites a CodeRabbit Minor for it). The dogfood's assertion `tamper.behavioral.tamper_flags.includes('test-tree-mutated')` (line 85) is a TEST membership check, not an authorization post-condition — acceptable here, but worth noting it is the same `.includes` SHAPE the repo elsewhere forbids for security gates.

### `dogfood-derive-sample.js`

- **Purpose** — The Rule-2a-corollary dogfood driver for the v3.11 W1 lesson-derive leg. Runs the REAL `claude -p` derive leg end-to-end through `runCaptureRerun` on ONE synthetic-but-realistic `(candidate, accepted)` windowed-boundary bug, into a temp recall-graph + sidecar store, then reports counters, loads the minted node back, and verifies the candidate patch is byte-recoverable from the sidecar. The file's own header is honest: this is an N=1 SMOKE that the real path EXECUTES and produces a valid leak-free floor-keyed lesson — NOT corpus-level evidence.
- **Imports / consumes** — `fs`, `os`, `path`; `runCaptureRerun` from `./lesson-capture-rerun`; `loadNode` from `../../attribution/recall-graph-store`; `readCandidate` from `../../attribution/candidate-sidecar`. Reads `process.argv` (the `--dry` flag is consumed downstream by `runCaptureRerun`). Hard-codes the `candidate_patch`, `accepted_diff`, and `ITEM` fixtures inline.
- **Consumers** — None programmatic. Referenced by `DOGFOOD-SAMPLE.md` and `docs/SIGNPOST.md`. Run manually (`node ...`, or `--dry` for no-network).
- **Functions**

| name | kind | purpose | consumes (params, files read) | writes (files, refs, stdout/JSON) | state changes / side effects |
|---|---|---|---|---|---|
| `main()` | cli-entry (async) | mkdtemp a base store dir; call `runCaptureRerun([ITEM], opts)`; print counters; if a node minted, `loadNode` it back and `readCandidate` the patch and print verify booleans | `ITEM` const; `os.tmpdir()`; the minted node + sidecar via the store reads | `process.stdout.write` (human + JSON counters + lesson fields); writes a recall-graph node + sidecar + report.json under the temp `base` dir (via `runCaptureRerun`) | `mkdtempSync` creates a temp dir; spawns a real `claude -p` child (unless `--dry`); **writes store records to disk**; **NEVER removes the temp dir** (no `rmSync`); no `process.exit` |

- **File-level notes** — Two real fragilities. (1) **No temp-dir cleanup at all** — `mkdtempSync` at line 65 has no matching `rmSync` anywhere; every run leaks a `w1-dogfood-*` dir under `os.tmpdir()` containing the recall-graph + sidecar + report. (2) **`main()` is invoked bare as `main();` (line 82) with no `.catch(...)`** — unlike the other three files, an async rejection here surfaces as an unhandled promise rejection (a noisy crash, and in modern Node a non-zero exit, but with no diagnostic framing). The `--dry`-vs-network branch is delegated entirely to `runCaptureRerun`, which re-reads `process.argv` — so passing `opts.dry` is not threaded here; the file relies on the global argv scan.

### `lesson-capture-rerun.js`

- **Purpose** — The capture re-run SPIKE: the IMPURE real-`claude -p` lesson-derive leg (`makeLessonDeriver`) plus a manual driver (`runCaptureRerun`). The pure machinery it drives (`captureLessons`, `deriveLesson`) is CI-tested with mocks; THIS is the real contrast leg. `makeLessonDeriver` builds the `deriveFn` that `captureLessons` expects: a `claude -p` contrast call that classifies a fixed bug into the FROZEN floor taxonomy and writes a leak-free `lesson_body`. The claude call mirrors `calibration-issue-run.js`'s pattern: STDIN prompt (ARG_MAX-robust), whole-output fence-strip (anchored), `--model` pinned, fail-closed to null on any refuse. The W3 trap seam: when an item carries a `failed_patch` (a verified-wrong attempt), it rides into the prompt as the primary contrast.
- **Imports / consumes** — `fs`, `os`, `path`, `child_process.spawnSync`; `TRIGGER_CLASS`/`GOTCHA_CLASS`/`CORRECTIVE_CLASS` from `../lesson-signature`; `captureLessons` from `../lesson-capture`. Reads env indirectly (`os.homedir()` for the claude fallback path), `process.argv` (`--dry`). Module constant `JUDGE_MODEL = 'claude-sonnet-4-6'`.
- **Consumers** — `dogfood-derive-sample.js` (`runCaptureRerun`) and `packages/lab/issue-corpus/_spike/corpus-build/bootcamp-capture.js` (`runCaptureRerun`). Mentioned in `lesson-derive.js` / `lesson-capture.js` header comments and the v3.11 W3 plan. No CI/unit consumer.
- **Functions**

| name | kind | purpose | consumes (params, files read) | writes (files, refs, stdout/JSON) | state changes / side effects |
|---|---|---|---|---|---|
| `resolveClaude()` | internal | locate the `claude` binary via `command -v` then `~/.local/bin/claude` fallback | `command -v claude` (a bash subshell); `fs.existsSync` on the fallback path | returns the bin path or `null` | spawns a `command` child via `spawnSync` with `shell: '/bin/bash'`; reads the FS |
| `claudeOnce(bin, prompt, timeout)` | internal | run a single `claude -p --model <pinned>` call over a STDIN prompt; strip an anchored code fence; parse strict JSON | `bin`, `prompt`, `timeout`; spawns the child | nothing to disk; returns `{ok:true, obj}` or `{ok:false, reason}` | spawns a real LLM child (`spawnSync`, `maxBuffer 4MiB`); `try/catch` around spawn; classifies ETIMEDOUT/non-zero/empty/parse-failure as fail-closed |
| `makeLessonDeriver({bin, timeout})` | exported (factory) | build the `deriveFn` that classifies a bug into the frozen floor + writes a leak-free body; wires the W3 `failed_patch` trap contrast | `contrastInput` (problem digest, candidate_patch, accepted_diff, failed_patch); the frozen-class enums | returns `{trigger_class, gotcha_class, corrective_class, lesson_body}` or `null` | calls `claudeOnce` (a real LLM call); coerces inputs through `String(...)`; fail-closed to `null` on any non-OK / non-object result |

| (module) `module.exports` | exported | `{ makeLessonDeriver, runCaptureRerun, resolveClaude }` | — | — | — |

- **File-level notes** — Solid fail-closed discipline (every claude failure mode -> `null` -> `deriveLesson` records a `harness_fallback`, never a fabricated classification). The leak-guard is in `deriveLesson` (scans the OUTPUT body against `accepted_diff` only), so quoting the UNSEALED `failed_patch` in the prompt is safe by construction (verified: `lesson-derive.js:60` scans body-vs-accepted regardless). Note the two `resolveClaude`/`claudeOnce` implementations: a near-duplicate of these exists in `trajectory-friction-run.js` (and the `JUDGE_MODEL`/`DEFAULT_MODEL` constant) — a DRY smell across the spike tier (acceptable for throwaway probes, but real duplication). The fence regex `^\`\`\`[a-zA-Z0-9]*\r?\n([\s\S]*?)\r?\n?\`\`\`$` is anchored (whole-output), stricter and safer than the embedded-match fence regex in `trajectory-friction-run.js:114`.

### `trajectory-friction-dogfood.js`

- **Purpose** — A macOS verification probe for the v3.9 W3 trajectory/friction paths. Drives (1) a REAL top-level `claude -p --output-format stream-json` capture on a benign fixture repo through `parseTrajectory`/`computeProcessGraph` to sane non-empty Layer-1 metrics; (2) the recall-smell DISCRIMINATES — a planted recall-shaped trajectory (low loop + relevant file UNREAD + reached) FIRES, a localization-heavy one does NOT; (3) `validateRecallSmellAgainstControls` reports a THREE-valued verdict over a labeled clean + neg-control set. It also drives the LLM friction labeler ONCE, best-effort (non-deterministic, NOT gated).
- **Imports / consumes** — `fs`, `os`, `path`, `child_process.execFileSync`; `parseTrajectory`/`computeProcessGraph`/`detectRecallSmell`/`validateRecallSmellAgainstControls` from `../trajectory-friction`; `runActorTrajectory`/`makeFrictionLabeler`/`resolveClaude` from `../trajectory-friction-run`. Reads `os.tmpdir()`, the `claude` bin (via `resolveClaude`).
- **Consumers** — None programmatic. Referenced in `docs/ACTIVATION-LEDGER.md`, `docs/SIGNPOST.md`, and the v3.9 W3 plan (D4). Run manually.
- **Functions**

| name | kind | purpose | consumes (params, files read) | writes (files, refs, stdout/JSON) | state changes / side effects |
|---|---|---|---|---|---|
| `git(args, cwd)` | internal | `execFileSync('git', ...)` -> stdout string | `args[]`, `cwd`; git binary | git side effects per `args` | spawns a git child synchronously; throws on non-zero |
| `makeFixture()` | internal | build a benign temp git repo: a buggy `calc.py` (subtracts) + a failing `test_calc.py` | `os.tmpdir()` | `calc.py`, `test_calc.py`; one git commit; returns `{dir, base_sha}` | `mkdtempSync` creates a temp dir; git init/config/add/commit/rev-parse mutate it |
| `recordFor(fx)` | internal | construct the calibration-record for the fixture | `fx` | returns a record object | none |
| `plantedRecallEvents()` | internal | hand-build a RECALL-shaped stream-json event list (one confident Edit, no Read of `calc.py`) | none | returns an events array | none (pure) |
| `plantedHonestEvents()` | internal | hand-build a LOCALIZATION-heavy event list (Read `calc.py` then Edit) | none | returns an events array | none (pure) |
| `main()` | cli-entry (sync) | run the three checks (+ best-effort labeler), print, and exit | the claude bin; the fixture; the planted events; the labeled set | `console.log` (human + JSON); `process.exit(0/1)` | conditionally creates a fixture (real capture) inside `try/finally`; **removes the temp dir in `finally`**; spawns a real `claude -p` actor + labeler when the bin exists; sets exit code |

- **File-level notes** — The cleanest of the four on cleanup: the real-capture block is wrapped in `try { ... } finally { rmSync(...) }`, so the temp dir is removed even if the capture/parse throws (the header cites the CodeRabbit Minor that drove it). The `bin`-absent branch correctly DEGRADES the two gated real-capture checks to `true` (lines 98-99) so the probe doesn't false-FAIL on a machine without `claude`; the labeler is best-effort and never gated. The check `pg.localization_reads.some((p) => p.endsWith('/calc.py') || p === 'calc.py')` (line 90) deliberately re-implements the basename fallback to assert the F4 fix holds on real absolute paths. One coupling fragility: `main()` is synchronous and calls `runActorTrajectory` (which is synchronous `spawnSync`) — correct, but it means a 180s actor timeout blocks the whole probe synchronously. `main();` is invoked bare with no `.catch` — acceptable because `main` is sync (a throw propagates as a normal uncaught exception, exit 1), unlike the async `dogfood-derive-sample.js`.
