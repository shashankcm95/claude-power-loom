## вњ… Phase 0 вЂ” Workspace Restructure

*Plan: [`packages/specs/plans/2026-05-25-phase-0-workspace-restructure.md`](../packages/specs/plans/2026-05-25-phase-0-workspace-restructure.md) В· ADR-0008 В· PR #158.*

A careful, mostly-mechanical repository restructure that gave the three-layer architecture a **filesystem boundary**. It established internal package boundaries via pnpm workspaces вЂ” `kernel / runtime / lab / skills / specs` вЂ” aligning the previously flat repo (`hooks/`, `scripts/`, `swarm/`, вЂ¦) with the kernel/runtime/lab model.

Why it mattered:

- It gave v3.0-alpha real directories to add ~900вЂ“1,300 LoC of kernel code into, and gave the K12 lint actual layer boundaries to check.
- **Move-don't-change**, with one acknowledged exception: introducing `packages/kernel/_lib/` and rewriting ~20 `require` statements to fix a genuine `kernel в†’ runtime` back-edge (kernel hooks had been importing `scripts/agent-team/_lib/*`). That refactor is semantic, not pure `git mv` вЂ” honestly re-estimated from ~4вЂ“5h to ~8вЂ“15h.
- Non-goals: no physical repo split, no plugin-distribution change, no public-API renames.
