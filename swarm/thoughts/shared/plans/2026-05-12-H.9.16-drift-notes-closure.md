**B.2 Downstream parse safe-pattern** (~40 sites):

Per compound test, suffix `|| true` on `T_X_Y=$(echo "$T_X" | python3 -c "..." 2>/dev/null)` lines. Without this, source-level safe-pattern doesn't prevent downstream cmd-sub failures from triggering set -e on empty input.

Example (Test 69):

```bash
# BEFORE:
T69_OUT=$(node ... 2>/dev/null)
T69_VERDICT=$(echo "$T69_OUT" | python3 -c "import json,sys; print(json.load(sys.stdin).get('verdict', ''))")
T69_A4_STATUS=$(echo "$T69_OUT" | python3 -c "...")

# AFTER:
T69_EXIT=0
T69_OUT=$(node ... 2>/dev/null) || T69_EXIT=$?
T69_VERDICT=$(echo "$T69_OUT" | python3 -c "import json,sys; print(json.load(sys.stdin).get('verdict', ''))" 2>/dev/null) || true
T69_A4_STATUS=$(echo "$T69_OUT" | python3 -c "..." 2>/dev/null) || true
```

Test logic unchanged: if source cmd fails, `$T69_OUT=""` → downstream parses fail-soft → `$T69_VERDICT=""` → existing `if [ "$T69_VERDICT" = "pass" ]` reports FAIL with empty diagnostic. Behavior: FAIL with diagnostic instead of silent exit.

**Net LoC delta**: ~+50 (25 source sites x 2 lines + ~40 downstream `|| true` suffix lines = ~90 line additions; some compounds share init line; net ~+50 effective).
