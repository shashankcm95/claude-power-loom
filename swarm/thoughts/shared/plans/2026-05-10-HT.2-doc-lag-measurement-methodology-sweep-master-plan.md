### Part A — drift-note 75 fix: `_lib/lock.js withLock` fail-fast on absent parent dir

Current behavior: `withLock(path, fn, {maxWaitMs})` fails opaquely with a 3-sec timeout when `path`'s parent directory does not exist. The fcntl-style lock primitive cannot create the lockfile, so it spin-retries until timeout.

Two options (architect to evaluate during pre-approval):
- **Option A — auto-mkdir parent**: `mkdirSync(dirname(path), {recursive: true})` before acquisition. Resilient but masks misconfigured callers.
- **Option B — fail-fast ENOENT**: detect parent-dir-missing on first attempt and throw `ENOENT` immediately. Clear error; calls fail visibly.

**Preliminary lean**: Option B (fail-fast ENOENT). Reasons:
1. Lock-file parent dir is part of the caller's contract — if `~/.claude/checkpoints/` doesn't exist, the caller has a misconfiguration.
2. Auto-mkdir masks bugs that should surface (caller may not intend a lock at that path).
3. Aligns with ADR-0001 fail-open invariant 2 (hooks fail-soft on errors) — fail-fast error is preferable to opaque 3-sec timeout that LOOKS like contention.

Pre-approval gate resolves the option choice. Sub-plan documents the chosen option with rationale absorbed from architect feedback.

### Part B — drift-note 67 fix: `session-end-nudge.js` lock-primitive migration

Current state: `session-end-nudge.js` (a Stop hook) uses inline lock primitive (open + flock + close manually) rather than `_lib/lock.js withLock()`. Reason: different fail-soft contract — session-end-nudge fails-soft on lock contention (silently skip nudge); `_lib/lock.js withLock()` THROWS on timeout.

Two options (architect to evaluate during pre-approval):
- **Option A — extend `_lib/lock.js withLock()`** with `{failSoftOnTimeout: true}` option to support both semantics
- **Option B — keep inline primitive** but document the deliberate divergence in a comment + add to a hooks-discipline ADR

**Preliminary lean**: Option A (extend `_lib/lock.js` with fail-soft option). Reasons:
1. DRY discipline — same lock-acquisition logic in two places will drift over time
2. Per-call configuration is the right pattern; HT.1.8 established that `{maxWaitMs}` per-call works well
3. ADR-0001 forbids hook crashes — fail-soft default is hook-appropriate; opt-in for substrate scripts

Pre-approval gate resolves the option choice. Note: if pre-approval surfaces a NEW ADR requirement for Part B Option B path, that would expand HT.2 scope (no new ADR currently anticipated; Part B Option A keeps substrate's institutional ledger at 4 ADRs post-HT.2 per the convention-vs-commitment discriminator).
