## 1. Architecture Overview — Transactional Agent Runtime

The substrate is a **deterministic envelope around byzantine processes**. Agents are treated as untrusted, non-deterministic execution units; the substrate guarantees behavioral correctness at the interface boundary, not in the agent's internals.

This is the same architectural pattern as:
- Database transaction managers wrapping non-deterministic application code
- CI gates wrapping untrusted developer commits
- BFT consensus wrapping unreliable distributed nodes
- iOS sandbox wrapping arbitrary app code

The substrate's job is **not** to make LLMs deterministic. It is to make **state transitions** deterministic given non-deterministic LLM trajectories.

```
        ORCHESTRATOR (depth-0; deterministic kernel)
                       │
       ┌───────────────┴───────────────┐
       │                               │
   user-space spawn               user-space spawn
   (byzantine, LLM)               (byzantine, LLM)
       │                               │
       │ writes deltas to              │ writes deltas to
       │ isolated worktree             │ isolated worktree
       ▼                               ▼
   ┌─────────────────────────────────────────┐
   │       INTERFACE BOUNDARY (kernel)         │
   │  - Deterministic gates (tests, lint,      │
   │    schema, contract-verifier)             │
   │  - Capability validators                  │
   │  - Path-rewriting / cherrypick            │
   │  - Spawn-record envelope capture          │
   │  - Negative-attestation extraction        │
   └─────────────────────────────────────────┘
                       │
                       ▼
           HOST CODEBASE (validated state)
```

## 2. The Five Axioms

### Axiom 1 — Transactional Determinism

> Validated filesystem deltas (or contract-conformant text outputs) are the substrate's unit of truth. LLM trajectories are non-deterministic by construction and recoverable-by-resampling.

**Forbids**: storing LLM reasoning traces as ground truth; treating agent self-reports as authoritative; implicit trust in agent claims of completion.

**Implies**: every spawn's output must be verifiable by deterministic gates; failures invalidate the spawn's deltas atomically; replay-from-state is supported, replay-from-decisions is not.

### Axiom 2 — Kernel / User-Space / Interface Boundary

> Kernel = hooks + GC + validators + recall-CLI + contract-verifier (pure functions, versioned, testable, deterministic). User-space = agent spawns (non-deterministic, ephemeral). Interface = filesystem deltas + contract-conformant text outputs (the verification surface).

**Forbids**: LLMs writing to kernel paths (`hooks/`, `scripts/kernel/`, `.claude-plugin/`); kernel code calling LLMs in the verification path; agents bypassing the interface to directly mutate host state.

**Implies**: kernel additions require TDD + versioning; user-space additions can iterate freely; the boundary IS the verification surface.

### Axiom 3 — Verification Gates as Pure Functions

> Promotion / merge / cherrypick gates must be runnable as pure functions. LLM-as-judge is NOT a verification primitive at the gating layer.

**Forbids**: "ask claude-judge whether this diff is good" as a merge condition; gate decisions that depend on LLM interpretation; non-deterministic verification.

**Permits**: LLM-as-judge for *advisory* verification (architect review, honesty-auditor findings, code-reviewer pair-runs) that produces audit trails but does NOT block merges. This is the split that accommodates existing HETS practice without weakening the gating-determinism guarantee.

### Axiom 4 — Algorithmic Discipline is Kernel Responsibility

> When an operation requires deterministic logic (sorting, subset-check, traversal, budget arithmetic), it lives in kernel code with unit tests — not as prose discipline in persona prompts.

**Forbids**: "iterate through children in topological order" in a persona contract (LLM may choose slightly-wrong order); embedded pseudocode that the LLM is expected to mentally execute; algorithmic correctness depending on agent compliance.

**Implies**: kernel grows a `scripts/kernel/algorithms/` library (topological_sort, subset_check, budget_allocate, tree_traversal, leaf_criteria validators, path_canonicalize, trait_resolve). Persona contracts REFERENCE algorithms by name; kernel runs them; LLM provides judgment inputs.

This is the **template method pattern**: kernel owns the control flow, LLM fills the judgment slots, verification gates check the outputs.

### Axiom 5 — Substrate Evolution is First-Class

> The substrate measures its own components' quality (personas, tests, decompositions, recalls) and uses those measurements to evolve. Components below quality thresholds are demoted/culled; components above thresholds become defaults.

**Forbids**: static persona contracts that never update; reputation tracking that doesn't feed back into spawn-routing; KB that never grows from substrate experience.

**Implies**: reputation per-persona × per-task-type × per-quality-axis; attribution graph for tracing bugs to test-gaps; evolve/forge triggers from reputation decay; convergence metrics as Phase 3 validation criterion.

## 3. Primitives

### 3.1 Isolation

| Primitive | Mechanism | Source |
|---|---|---|
| Worktree-bounded sandbox per spawn | Declare `isolation: "worktree"` in persona contract; Claude Code auto-allocates worktree + auto-cleans if no changes | Anthropic native — `code.claude.com/docs/en/worktrees` |
| Capability injection at spawn-init | `PreToolUse(Agent)` hook with `hookSpecificOutput.updatedInput` rewrite | Anthropic native — `code.claude.com/docs/en/hooks` §5 |
| Worktree-scoped writes | Enforced by Claude Code's worktree primitive; no kernel work needed | Anthropic native |

**Substrate contribution**: declarative configuration in persona contracts; the heavy lifting is Anthropic-native.

### 3.2 Capabilities (Dual-Axis)

| Axis | Scope | Enforcement |
|---|---|---|
| Read | Broad (project files, libraries, recall results, documentation, prior spawn outputs) | Post-hoc audit on spawn close (writes outside scope rejected by promote-deltas) |
| Write | Narrow (sandbox/worktree only) | Worktree filesystem scope (Anthropic native) |
| Network | Explicit allowlist | Declared in contract; ADVISORY in Phase 2 (enforcement deferred — capability gates not natively per-spawn) |
| Subprocess | Scoped command whitelist | Settings.json `permissions.allow` patterns |

**Capability traits as mixins** (composition over inheritance):

```yaml
# Atomic traits (declared once)
traits:
  worktree_writable: { isolation: worktree, write: ["sandbox://**"] }
  network_anthropic: { network: ["api.anthropic.com"] }
  bash_test_runner: { subprocess: ["npm test", "vitest", "pytest", "tsc --noEmit"] }
  recall_global: { read_recall: ["@library/*", "@thoughts/*"] }

# Personas COMPOSE traits
06-node-backend:
  traits: [worktree_writable, network_anthropic, bash_test_runner, recall_global]
```

### 3.3 Persona Contracts (Two-Tier)

```yaml
04-architect:
  interface:                            # what all instances inherit
    declared_capabilities: { ... }
    declared_scope: ["architectural decisions", "pressure-testing", "RFC drafting"]
    output_schema: { type: "design_artifact", required: ["decision", "rationale"] }
    decomposition_discipline:
      primary: "spec-driven"
      fallback_when_code_producing: "tdd"
  defaults:                             # used when no spawn-time override
    timeout: 1800
    retention: { ... }
```

**Liskov substitution constraint**: any instance's `capabilities_used ⊆ declared_capabilities`, `task_scope ⊆ declared_scope`, `output ⊨ output_schema`. Enforced at spawn-init and spawn-verify.

### 3.4 Decomposition

**The merge-sort-for-tasks principle**: recursive atomization until each leaf satisfies five criteria.

| Criterion | Operational check |
|---|---|
| Cost-justified | Expected work ≥ ~500 tokens / ~30s wallclock |
| Semantically cohesive | One clear purpose, not a bundle |
| Interface-clean | Focused input; structured output |
| Validation-supported | Deterministic gate exists for this leaf type |
| Resource-bounded | High confidence completes without internal sub-spawning |

**Leaves at domain-artifact granularity, not operation granularity**: "add a migration" is a leaf; "add a semicolon" is inline noise.

**Atomicity cutoff is empirically tuned**, not hardcoded. Telemetry: `decomposition_metrics` emitted by spawn-record (parent_spawn_id, leaf_count, cost_per_leaf, validation_pass_rate). Self-improve loop aggregates and recommends cutoff adjustments.

**Decomposition disciplines (per-persona, configurable)**:

| Discipline | When to use | Verification gate |
|---|---|---|
| `tdd` | Code-producing leaves (add migration, implement endpoint, refactor) | Test suite passes |
| `spec-driven` | Configuration / structural work | Schema validate |
| `exploratory` | Research / subjective writing / design docs | Structural check (required sections present) |

**Two trampoline patterns** (for arbitrary logical depth despite Anthropic's depth-1 spawn constraint):

| Pattern | Use case | Implementation |
|---|---|---|
| **Pattern A — Persona-internal trampoline** | Intra-persona decomposition (same cognitive domain, sequential acceptable) | TodoWrite tree + folder hierarchy + save states (within ONE Anthropic spawn). ~30 LoC contract validator. |
| **Pattern B — Multi-spawn trampoline** | Inter-persona orchestration (cross-domain, parallel desired) | Orchestrator queue + continuations + parent_state_id chain. ~300-400 LoC kernel. Deferred to Phase 3 unless real depth-3+ workflow appears. |

Personas use Pattern A internally; the orchestrator uses Pattern B across personas. They compose (orchestrator → mira via Pattern B; mira splits into sub-tasks via Pattern A).

**Persona-as-decomposer (fractal HETS)**: the existing 16 domain personas (architect, ml-engineer, node-backend, etc.) ARE the decomposers. No new "decomposer persona" needed. Each persona owns its sub-tree opaquely; orchestrator sees flat persona-result. Borrows OTP-style hierarchical supervision.

**Negative attestation as Class-4 artifact**: failed spawns produce SCOPED constraint records that later spawns must respect. Recall-CLI retrieves them at spawn-init; route-decide considers them for spawn-or-not.

**TodoWrite as checkpoint primitive**: monotonically-updated task list serves as the persona's checkpoint trail. Pattern A trampoline uses this natively.

**Verification gates** (pure-function pipeline, deterministic):
- Schema validation (JSON / YAML frontmatter / contract-verifier)
- Test suite execution (test-runner adapters for jest, vitest, pytest, etc.)
- Lint + type-check
- Capability subset check (child ⊆ parent at spawn-init)
- Path-canonicalization check (no `..` / absolute / symlink-escape at promote-deltas)
- Budget envelope check (tokens / wallclock / recursion depth at spawn-init)

### 3.6 Recall + Memory

**Recall-CLI tri-signal ranker** (already shipped Phase 1):
```
score = 0.5 * keyword_jaccard + 0.3 * tag_overlap + 0.2 * surface_overlap
```
Deterministic, zero-LLM, reproducible. Top-K=3 over `~/.claude/library/sections/`. Phase 2 extends scope to include L_persona + L_spawn tiers.

**KB structure**:
```
~/.claude/library/sections/
  toolkit/
    stacks/session-snapshots/volumes/    # session ship records (existing)
    research/                            # external research distillations (NEW)
    craft/tdd/                           # TDD craft KB (NEW; seed + self-authored)
    craft/decomposition/                 # decomposition patterns (NEW; self-authored)
```

**Reference test suites** (cold-start calibration): hand-authored test suites per task-type that new personas must satisfy BEFORE getting real work. Standard ML validation-set practice.

### 3.7 Budget + Resource Discipline

**Budget envelope propagation** (across recursion depth):

```yaml
budget:
  token_limit: 50000          # inherited fraction of parent's remaining
  wallclock_limit: 1800       # inherited
  recursion_depth: 2          # decrements per sub-spawn
  on_exhaustion: "abort" | "escalate-to-parent"
```

Persona's budget envelope covers ENTIRE subtree, not just its own work. Route-decide gate enforces.

**Escape hatch**: `LOOM_DISABLE_WORKTREE=1` env var bypasses spawn-init to v2.9.1 semantics. Must be designed in from Wave 0.

**Circuit-breaker** (deferred to Phase 3 from blind-spot list): `max_denials_per_minute` terminates misbehaving persona.

### 3.8 Evolution Wiring (Phase 3)

| Component | Purpose |
|---|---|
| Reputation extension | Per-persona × per-task-type × per-quality-axis EWMA (test-writing quality, decomposition quality, capability-declaration quality) |
| Attribution graph | Spawn-records form DAG via parent_state_id. Bug surfaced → graph query identifies which spawn(s) + which missed test(s) |
| TDD-craft KB | Seed knowledge (hand-authored) + self-authored craft (from negative attestations) |
| Cross-persona test review | Peer persona scores test quality independently of code-pass-rate. Anti-gaming. |
| Convergence metrics | `ConvergenceRate(t) = first_attempt_pass_rate(t) / first_attempt_pass_rate(t=0)` |
| Evolve/forge triggers | Reputation < threshold N → propose evolve cycle. Below threshold M → propose forge replacement. |

### 3.9 Kernel Algorithm Library

`scripts/kernel/algorithms/` (~400-500 LoC across these files):
- `topological_sort.js` — dependency-ordered traversal
- `subset_check.js` — capability inheritance Liskov check
- `budget_allocate.js` — fractional budget propagation
- `tree_traversal.js` — TodoWrite tree walking
- `leaf_criteria.js` — 5 sub-validators for atomicity
- `path_canonicalize.js` — promote-deltas path-rewriting
- `trait_resolve.js` — capability trait composition

Each algorithm: pure function, unit-tested, invoked from hook scripts when corresponding decisions need to be made. LLM never asked to execute deterministic logic.

**What we have that's genuinely novel** (cross-checked against all above + Letta/mem0/MemPalace/LangGraph/statewright/Voyager/Reflexion/MetaGPT/CrewAI):

1. **Negative attestation as first-class Class-4 artifact** — failed spawns producing scoped constraint records that constrain future spawns. Nobody else treats failure as positive substrate-typed artifact.
2. **Budget envelope across recursion depth** — others have per-step or session-cumulative budgets, not recursion-aware envelope propagation.
3. **Merge-sort-for-tasks with 5-criteria leaf rule** — formal decomposition stopping criterion. Others leave to LLM judgment.
4. **Persona × identity × task-type two-axis reputation** — WBFT literature is single-axis; ours splits role-shape from concrete instance.
5. **Filesystem-delta-as-truth axiom over LLM-state** — LangGraph uses JSON state; ours is adversarially robust against agent self-reporting.
6. **Persona-as-decomposer (fractal HETS)** — MetaGPT is linear assembly-line; CrewAI is shallow hierarchy; recursive persona-decomposition is a real generalization.
7. **Decomposition disciplines (tdd / spec-driven / exploratory)** as persona-contract-declared and substrate-enforced — no other system surfaces this as a first-class field.

**What we're aligned with (validated patterns)**:
- Determinism axiom matches LangGraph's design philosophy
- Reputation tracking matches WBFT-for-LLMs literature convergence (arxiv 2507.14928)
- Recall-CLI structurally matches Voyager's skill library (deterministic variant)
- Substrate axioms read like a direct AutoGPT/BabyAGI postmortem response

**Blind spots surfaced**:
1. **No hash-chained tamper-evidence** (Bernstein + det-acp have this). ADR needed.
2. **No liveness/stuck-spawn detection** (Gas Town has Witness/Deacon). First swarm to wedge in production = painful lesson.
3. **No per-tool-call gate after spawn** (OAP per-call enforcement has adversarial-testbed validation).
4. **No declarative workflow artifact** (Conductor YAML; Bernstein TOML Formulas; Archon DAGs). Limits workflow reuse + auditability.
5. **No circuit-breaker on denial/error rate** (det-acp `max_denials`).
6. **No transitive revocation lineage** (seL4 capability literature).
7. **No global trajectory hash** for replay-equivalence checking (LangGraph).
8. **No explicit sibling-fate strategy declaration** (OTP `one_for_one` / `one_for_all` / `rest_for_one`).
9. **Possible serializable-snapshot-isolation gap** — recurring `DRIFT-test3-020` looks like SSI failure on derived view (PR base).
