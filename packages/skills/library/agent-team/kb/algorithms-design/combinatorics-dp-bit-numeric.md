## Summary

**Concept**: Backtracking (permutations, combinations, power set, Sudoku / Algorithm X exact cover), two-pointer / sliding-window techniques, dynamic programming (0/1 knapsack, Kadane's max-subarray, edit distance), number theory (GCD, Roman/Arabic, money-to-words), and bit/hash/encoding tricks.
**Key APIs**: Heap's algorithm for permutations; include/exclude power-set recursion; Knuth Dancing Links (DLX); sliding-window `Map`; `BigInteger.gcd`; `Integer.numberOfTrailingZeros`; folding hash; byte[]↔hex (7 techniques → `java.util.HexFormat`).
**Gotcha**: `TwoSum.twoSumSlow` inner loop starts at `j=1` not `j=i+1` (pairs elements with themselves); `SmallestMissingPositiveInteger` boolean variant silently ignores negatives; `KadaneAlgorithm` logger is misnamed.
**2026-currency**: `javax.xml.bind.DatatypeConverter` hex (removed Java 11) → `java.util.HexFormat` (Java 17); algorithms otherwise evergreen.
**Sources**: Baeldung `algorithms-miscellaneous-2/3/4/5`.

## Quick Reference

**Backtracking**:
- **Permutations** — Heap's algorithm (recursive + iterative), lexicographic, random.
- **k-combinations** and **power set** (include/exclude recursion).
- **Sudoku** — constraint backtracking, plus Knuth's **Dancing Links / Algorithm X** exact-cover formulation.

**Two-pointer / sliding window**:
- TwoSum on a sorted array.
- Array rotation via triple-reverse.
- Longest substring without repeats (sliding-window `Map`).
- Palindromic substrings (center-expansion, brute, Manacher).

**Dynamic programming**:

| Problem | Naive | DP |
|---|---|---|
| 0/1 knapsack | recursion | bottom-up table |
| Maximum subarray | brute O(n^2) | Kadane O(n) |
| Edit distance | recursion | 2D table (see `algorithms-design/searching-and-string-matching`) |

**Number theory / numerics**: GCD & coprimality (iterative / recursive / `BigInteger.gcd`); Roman ↔ Arabic (enum symbols, greedy subtractive); money-to-words (hand-rolled + Tradukisto); binary gap (`Integer.numberOfTrailingZeros` + shifts); smallest missing positive int; merge two sorted arrays.

**Hashing / encoding**: a folding hash (group + concat ASCII + mod); byte[]↔hex shown **seven ways**; a Big-O demonstrator (one loop shape per complexity class).

**Top gotchas**:
- `TwoSum.twoSumSlow` inner loop starts at `j=1` instead of `j=i+1`, pairing an element with itself.
- `SmallestMissingPositiveInteger` boolean-array variant silently ignores negatives.
- `KadaneAlgorithm`'s logger is named with `BruteForceAlgorithm.class` (copy-paste bug).

**Current (mid-2026)**: the seven hand-rolled hex techniques and any `javax.xml.bind.DatatypeConverter` use (removed at Java 11, JEP 320) collapse into **`java.util.HexFormat`** (Java 17). The algorithms themselves are pure-JDK and evergreen.

## Full content

This section gathers the combinatorial, DP, numeric, and bit/encoding algorithms spread across `algorithms-miscellaneous-2/3/4/5`.

### Backtracking and exact cover

Permutations are generated by Heap's algorithm (recursive and iterative), plus lexicographic and random orderings. k-combinations and the power set use include/exclude recursion. Sudoku is solved by constraint backtracking and, more interestingly, by Knuth's Dancing Links (Algorithm X) exact-cover reduction — the canonical "model as exact cover, then DLX" technique.

### Two-pointer and sliding window

TwoSum on a sorted array, array rotation via triple-reverse, longest-substring-without-repeats via a sliding-window map, and palindromic-substring counting (center-expansion, brute, and Manacher's algorithm) make up the two-pointer family.

### Dynamic programming

The DP set is the standard teaching trio: 0/1 knapsack (naive recursion + bottom-up table), maximum subarray (brute O(n^2) vs Kadane's O(n)), and edit distance (also covered under string matching). Each shows the naive-then-tabulated progression.

### Number theory and numerics

GCD and coprimality are shown iteratively, recursively, and via `BigInteger.gcd`. Roman ↔ Arabic conversion uses enum symbols with a greedy subtractive rule. Money-to-words is hand-rolled and also via the Tradukisto library. The binary gap uses `Integer.numberOfTrailingZeros` plus shifts. Smallest-missing-positive-int and merge-two-sorted-arrays round out the set.

### Hashing and encoding

A folding hash groups, concatenates ASCII codes, and mods. byte[]↔hex is shown seven ways — the prime modernization target. A Big-O demonstrator pairs one loop shape with each complexity class for teaching.

### 2026 currency

- **Hex conversion: `javax.xml.bind.DatatypeConverter` → `java.util.HexFormat` (Java 17).** `DatatypeConverter` was removed from the JDK at Java 11 (JEP 320). All seven hand-rolled hex techniques now collapse into `HexFormat.of()` / `ofDelimiter()` with `formatHex` / `parseHex` / `toHexDigits`. [HexFormat — Oracle docs](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/util/HexFormat.html) · [Introduction to HexFormat | Baeldung](https://www.baeldung.com/java-hexformat)
- **Tradukisto** (money-to-words) is a niche library — fine if already used, but not a default reach. It is actively maintained (the 4.x line is current on Maven Central, spanning major versions 0 through 4).
- The combinatorics / DP / number-theory / bit algorithms are pure-JDK 8 and evergreen; only the hex-encoding helper changed.
