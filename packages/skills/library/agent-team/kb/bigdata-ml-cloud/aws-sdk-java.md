## Summary

**Concept**: The AWS SDK for Java across the core services (S3, EC2, SQS, RDS, DynamoDB) — the v1 `com.amazonaws` builder/mutable surface that dominates the corpus, plus the v1→v2 generational contrast.
**Key APIs**: `AmazonS3ClientBuilder.standard().withCredentials(...).withRegion(...)`; `TransferManagerBuilder` (multipart); `DynamoDBMapper` + `@DynamoDBTable`/`@DynamoDBHashKey`; SQS standard vs FIFO (`FifoQueue=true`, `MessageGroupId`, `RedrivePolicy` DLQ); `RunInstancesRequest`; `createDBInstance`.
**Gotcha**: hardcoded `BasicAWSCredentials("<accesskey>","<secretkey>")` placeholders; `main`s provision real billable EC2/SQS/RDS resources; DynamoDB `findAll()` full-table `scan` warns against production use.
**2026-currency**: AWS SDK v1 reached **end-of-support Dec 31, 2025** — migrate to **v2** (`software.amazon.awssdk`); `DynamoDBMapper` → DynamoDB Enhanced Client; v1 `TransferManager` → S3 Transfer Manager v2.
**Sources**: Baeldung `aws` module; AWS v1 end-of-support announcement.

## Quick Reference

**Client construction (v1 pattern, constructor-inject for testability)**:
- `AmazonS3 s3 = AmazonS3ClientBuilder.standard().withCredentials(provider).withRegion(region).build()`
- Inject the `AmazonS3` into your service rather than building it inline.

**S3**: bucket/object CRUD (`createBucket`, `putObject`, `getObject`, `copyObject`, `deleteObjects`). Multipart via `TransferManager`: `TransferManagerBuilder` with `withMultipartUploadThreshold`, `upload()` → `Upload.waitForCompletion()`, a `ProgressListener`.

**EC2**: security groups (`IpPermission`/`IpRange` CIDR), key pairs (private key returned once), `runInstances` (`RunInstancesRequest`, `t2.micro`), start/stop/reboot/monitor/describe.

**SQS**: standard vs FIFO (`FifoQueue=true`, `ContentBasedDeduplication`, `MessageGroupId`), DLQ via `RedrivePolicy`, single/batch send, message attributes, `withDelaySeconds`, long polling (`withWaitTimeSeconds`).

**RDS**: `createDBInstance` (postgres, `gp2`), `describe`/`delete`, then raw JDBC against the endpoint.

**DynamoDB**:
- ORM: `DynamoDBMapper` over `@DynamoDBTable`/`@DynamoDBHashKey`/`@DynamoDBAutoGeneratedKey`/`@DynamoDBAttribute` POJOs; `save`/`load`/`scan`; `generateCreateTableRequest`
- Document API: `DynamoDB`, `Item`, `PutItemSpec`, `getTable`
- Local testing: embedded `ServerRunner`/`DynamoDBProxyServer`, JUnit `@ClassRule extends ExternalResource`, needs sqlite4java native libs

**v1↔v2 contrast**: v1 = mutable builders, sync; v2 (`software.amazon.awssdk`, the `aws-reactive` module) = builder/immutable, async-capable, `S3AsyncClient` over Netty NIO.

**Top gotchas**:
- Hardcoded `BasicAWSCredentials("<accesskey>","<secretkey>")` placeholders — use IAM roles / `DefaultCredentialsProvider`, never commit keys.
- The EC2/SQS/RDS `main`s provision **real billable** cloud resources.
- `AbstractRepository.findAll()` (DynamoDB full-table `scan`) explicitly warns against production use.
- `setEndpoint(...)` is deprecated (use `withEndpointConfiguration`).

**Current (mid-2026)**: AWS SDK **v1 reached end-of-support Dec 31, 2025** (maintenance since Jul 31, 2024). Migrate everything to **v2** (`software.amazon.awssdk`). `DynamoDBMapper`/document API → **DynamoDB Enhanced Client**; hand-rolled v1 `TransferManager` → **S3 Transfer Manager v2** (GA Dec 2022, CRT-based).

## Full content

The Baeldung `aws` module is the densest sub-area of the domain, covering S3, EC2, SQS, RDS, DynamoDB, and the abandoned JetS3t client — almost all using **AWS SDK for Java v1** (`com.amazonaws`). The idiomatic shape is the v1 mutable builder: `AmazonS3ClientBuilder.standard().withCredentials(...).withRegion(...).build()`, with the resulting client constructor-injected into a service class for testability.

**S3** covers bucket/object CRUD plus multipart upload. Multipart is delegated to the high-level `TransferManager` (`TransferManagerBuilder` with `withMultipartUploadThreshold`, `upload().waitForCompletion()`, a `ProgressListener`), which handles part splitting and parallelism.

**EC2** demonstrates security groups (CIDR via `IpPermission`/`IpRange`), key-pair creation (the private key is returned exactly once), and the instance lifecycle (`runInstances` with a `RunInstancesRequest` for a `t2.micro`, then start/stop/reboot/monitor/describe).

**SQS** is the most feature-complete service example: standard vs FIFO queues (`FifoQueue=true`, `ContentBasedDeduplication`, `MessageGroupId`), dead-letter queues via `RedrivePolicy`, single and batch sends, message attributes, delayed delivery (`withDelaySeconds`), and long polling (`withWaitTimeSeconds`).

**RDS** provisions a Postgres instance (`createDBInstance`, `gp2` storage), then connects with raw JDBC against the endpoint. **DynamoDB** appears three ways: a `DynamoDBMapper` ORM over annotated POJOs (`@DynamoDBTable`, `@DynamoDBHashKey`, `@DynamoDBAttribute`) with a reflection-based generic `AbstractRepository<T,ID>`; the lower-level document API (`DynamoDB`, `Item`, `PutItemSpec`); and DynamoDB Local for integration testing (embedded via `ServerRunner`/`DynamoDBProxyServer`, wired with a JUnit `@ClassRule`).

The whole module carries operational hazards: credentials are hardcoded as placeholders, several `main`s actually create billable cloud resources, `setEndpoint(...)` is deprecated, and the generic `findAll()` does a full-table `scan` its own Javadoc warns against in production.

The genuine teaching asset is the **v1↔v2 generational contrast**: this `aws` module (and `aws-lambda`) use the mutable, synchronous v1 surface, while the `aws-reactive` module demonstrates the modern v2 (`software.amazon.awssdk`) builder/immutable, async-capable surface (`S3AsyncClient` over Netty NIO) — covered in the reactive-cloud-integration sibling.

### 2026 currency

**AWS SDK for Java v1 is dead.** v1.x entered maintenance mode July 31, 2024 and reached **end-of-support December 31, 2025**; existing apps keep working and prior releases stay on Maven, but there is no new service support ([Announcing end-of-support for AWS SDK for Java v1.x (AWS Developer Tools Blog)](https://aws.amazon.com/blogs/developer/announcing-end-of-support-for-aws-sdk-for-java-v1-x-on-december-31-2025/) · [Announcement: End-of-Support as of December 31, 2025 (aws/aws-sdk-java#3195)](https://github.com/aws/aws-sdk-java/issues/3195)). Migrate to **v2** (`software.amazon.awssdk`, builder/immutable/async).

Specific replacements: `DynamoDBMapper` and the document API → the **DynamoDB Enhanced Client** (the v2 high-level mapper) ([Enhanced Client README (aws-sdk-java-v2)](https://github.com/aws/aws-sdk-java-v2/blob/master/services-custom/dynamodb-enhanced/README.md)); the v1 `TransferManager` hand-rolled multipart → **S3 Transfer Manager v2** (GA Dec 19, 2022, CRT-based, with directory transfer + pause/resume + progress) ([S3 Transfer Manager GA (AWS)](https://aws.amazon.com/about-aws/whats-new/2022/12/amazon-s3-transfer-manager-aws-sdk-java-2-x)). On credentials, hardcoded keys → IAM roles / `DefaultCredentialsProvider` is confirmed current guidance.
