# RFC v3.11 — The Recall-Graph Experience Layer

> **Revision note (2026-06-15):** a contract-driven VERIFY board (architect + honesty-auditor + code-reviewer)
> returned NEEDS-REVISION and was folded firsthand. The load-bearing corrections: the 11 seed nodes are **not**
> derivation-ready (the candidate patch is a dead digest — Sec 8); `friction_signature_ref` is **4-null / 7
> friction-bearing**, not null-on-every-node (Sec 1/8); the lesson key is a **new** `lessonClusterKey`, never an
> extension of the frozen `frictionClusterKey` (Sec 5/6); the confirmation gate needs a **cross-run join**
> (`fail_to_pass` is not on the node — Sec 5). The full board record is in Sec 12.

## 0. North-star (the first principle): a client-owned, model-agnostic substrate

Power Loom inverts the control polarity of a naive agent. The model is **not** the authority; it is a
**swappable carrier of capability** — like the connectivity a client rides, never the policy. Authority,
intent, memory, and trust are **client-owned and local**.

| The mental model | Power Loom |
|---|---|
| Connectivity / signal (commodity, swappable) | the LLM (Claude) — the active reasoner, but demoted to a carrier |
| Router + firewall + same-origin policy (enforced, deterministic) | the **kernel** — the only tier that can say *no* |
| Tabs / configured profiles | runtime / HETS personas |
| History, bookmarks, cache, learned preferences (owned, audited) | the **lab** — recall graph, reputation, **this experience layer** |
| The human choosing where to go | the **merge gate** |

The metaphor is **normative, not descriptive**: the model is really an *active reasoner*, closer to untrusted
remote code the kernel sandboxes than to dumb transport. We deliberately run a deterministic layer over a
stochastic signal — TCP-over-lossy-link: the kernel does the ordering / idempotency / content-addressing /
**confirmation** so a noisy signal yields reliable, audited artifacts. (One honest edge on the metaphor: the
kernel's *enforced* surface is narrower than "firewall" implies — the delta-promote path is shadow-default
`LOOM_RESOLVER_ENFORCE`-off; the hard "no" today is the K5 validators + K7/K13, not the promote gate. The
experience layer never touches the promote path.)

**The first principle this gives every design question:** whenever we ask *"should this live signal-side or
client-side?"*, the answer is **client-side and model-agnostic**. The experience layer is the part of the client
that **learns its preferences from its own audited history** rather than having them configured.

### 0.1 The capability ceiling (the honest edge)

A client can only **narrow** the signal; it can never **lift** its raw capability. The experience layer makes the
client **use** the signal better (route away from bad spawns, surface the relevant lesson) — it does **not** make
the actor smarter. This is the README's standing disavowal and **OQ-NS-6** restated: *the client narrows, the
world hardens, the signal does the cognitive work*. The metaphor's failure mode is the inverse temptation —
"model is a commodity" seducing us into under-investing in the actor when the signal is still the dominant quality
lever. Do not.

### 0.2 Two guardrails the metaphor surfaces (named residuals)

- **G-AGNOSTIC — "model-agnostic" is an invariant we ENFORCE, not a free property.** The *signature key* is
  model-agnostic by construction (a closed-enum tuple, client-side). But the lesson *content* (gotcha text,
  corrective principle) is derived by a specific model and can smuggle a model-specific assumption. "Artifacts
  survive an ISP change" is the property we *test*, not assume. (Likely a portability check, not a build, for v3.11.)
- **G-OWNERSHIP — the ownership boundary is more entangled than "your browsing history."** A bookmark is cleanly
  yours; a *lesson* is derived by the signal **reading sealed content** (the accepted diff it must never quote).
  That entanglement is the entire reason the **leak-guard** exists — the seam where client-owned artifact and
  signal-side content touch.

### 0.3 The symmetry: un-learning is as client-owned as learning

The confirmation gate (promote) and retire-as-view + contradiction-invalidation (demote) are **one** mechanism —
the client's authority over its own preferences runs both directions. That is what keeps "earned preferences" from
curdling into confidently-wrong preferences that mis-steer the next actor (**false >> missing**).

## 1. Problem

The recall graph today is an **action-log of behavioral-pass "trophy" nodes**. Each stores **digests** (an opaque
sha256 of the problem statement + a **truncated 16-char** `candidate_patch_ref` that is a *dead* digest, not a
recoverable pointer) and a `reference_divergence` scalar. Firsthand census of the 11 seed nodes
(`~/.claude/lab-state/recall-graph-backtest/`, 2026-06-15): **4 carry `friction_signature_ref: null`, 7 carry a
real signature** in the *failure* taxonomy (`incorrect-implementation|editing|semantic-lens` etc.); **8 carry a
`reference_divergence` scalar, 3 null**; **none** carry a *lesson* (`trigger|gotcha|corrective`) signature; **none**
persist the candidate patch content or `fail_to_pass`. The graph is edgeless bar one authorship edge (v3.10), is
retrieved lexically, and v3.10 closed with **"mechanism proven, DATA is the gap."**

The trophy case is the wrong artifact. The file delta is already in git. Re-storing it is the **too-literal**
failure mode.

## 2. The reframe (lessons, not actions)

A recall node should be a **derived lesson**: a TRIGGER (the situation it recognizes), a GOTCHA (the non-obvious
trap), a corrective PRINCIPLE, and a WEIGHT (recurrence). **Store surprise; compress expectation.** **git is the
answer key; memory is the margin-notes** (a one-way dependency, memory derived from delta). The graph's first real
edge: `(failure-context, lesson) --confirmed-by--> (delta-ref)`.

## 3. Pinned decisions

**Pinned by the USER, 2026-06-15:**

- **D1 (FROZEN taxonomy) — a distinct trigger-recognizer axis:** `trigger-class | gotcha-class | corrective-class`
  (three fresh, frozen closed-enums; the friction axes describe *what went wrong*, the retriever needs *what
  situation you are in*). Highest cost-to-reverse — **the freeze deliverable (Sec 6) is larger than "pick values":
  it also produces the composite-key namespacing + an each-value-assignable audit, all frozen together.**
- **D2 (REVISED post-board) — build the organ as machinery + a small capture re-run; the failed-attempt (trap)
  seam is later.** The 11 seed nodes cannot be derived from (candidate patch is gone — Sec 8). So W1 builds the
  schema + derivation leg + `lessonClusterKey` + leak-guard + gate as **machinery (TDD on fixtures)**, and a small
  bootcamp re-run **persists the now-missing fields** (candidate patch + accepted diff + `fail_to_pass` + friction
  block) to mint the first real lessons. Gate-first preserved; the trap seam (capturing *failed* attempts) stays a
  later wave.
- **D3 (confirmation grain) — strict same-`fail_to_pass`.** A lesson promotes only when the *same* issue's test-set
  passes again. Because `fail_to_pass` is not stored on a node, the gate is a **cross-run join** between the new
  run's record (which carries the corpus `fail_to_pass`) and the candidate node by `issue_id` — spec'd in Sec 5.

**KISS defaults (reversible; the board did not contest them):**

- **DEF-1** — unconfirmed candidates decay **from the retriever's working-set only** (a suppression view), never deleted.
- **DEF-2** — expected baseline = a **bare class-level counter** (the epsilon-floor vs survivorship-of-surprise).
- **DEF-3** — **confirmed-collisions only** for the trust-bearing weight, **plus a read-only raw-collision
  diagnostic** (kept because D1 adds the trigger axis — it catches the new taxonomy under-separating). **The
  diagnostic + the recurrence weight need a named storage seam** (neither is in the node schema today) — a
  `consolidation-report.json` under `lab-state/` or a CLI-only report; W1 names it.

## 4. The 7 borrowed mechanisms (survived the research cut)

Fit-lineage: **cognitive-architecture + RL (ACT-R / Soar / CBR / prioritized experience replay)** — structured
keys, gated promotion — **not** the LLM-embedding-memory lineage (embeddings + no confirmation gate: the two lines
we do not cross).

1. **A1 — Tag at a boundary the system already emits** (SWR fires at the pause, ~33% selective).
2. **A2 — Store surprise, compress expectation** (prioritized experience replay; keep a class-level baseline counter).
3. **A3 — Identity = a content-addressed structured signature, never an embedding** (pattern-separation + ACT-R/CBR symbolic keys).
4. **A4 — Recurrence is the weight** (a raw confirmed-collision count — explicitly NOT ACT-R base-level activation; the decay term is deferred; we cite the idea, not the formula).
5. **A5 — Provisional -> confirmed, keyed to the SAME signature** (synaptic tagging & capture; STC's *promiscuous capture* is the bug our addressing prevents — the one load-bearing refinement).
6. **A6 — The node is a sparse index/digest, not content** (hippocampal index theory) — already shipped.
7. **A7 — Forget at the read layer, never delete** (active forgetting + retire-as-view); biological erasure rejected (fights INV-22 + audit).

## 5. Architecture (kernel -> lab)

Hard rule: **kernel is the only enforced tier; lab is shadow/advisory; the dependency points inward; the ONLY lab
-> kernel bridge that can touch a live spawn is the A6 reputation snapshot.** ADR-0012 makes per-spawn lesson
injection structurally impossible (a free safety property). No LLM step is ever a kernel primitive.

**Persona-agnostic key contract (forward, board-Q5):** the lesson signature and the recurrence weight are
**persona-agnostic** — built only from the D1 axes. v3.10's `built_by`/`graded_by` remain **unauthenticated
provenance metadata, never a key or weight input** (else we re-import the v3.10 C2 key-fragmentation laundering
lever). W2's gate must not key on persona.

**The vertical tie (scope-honest):** the kernel emits ground truth (K2 spawn-records, the content-addressed
`record-store` keyed by `post_state_hash` = the `delta-ref` the edge points at). The lab reads it, derives + gates
lessons. v3.11 is **closed kernel -> lab (read) + lab-internal derive/gate**; the lab -> spawn **A6 lesson-bridge
is the v-next consumer (deferred, Sec 9)** — not built this phase (the graph must have edges first).

## 6. The frozen signature taxonomy (PROPOSED — the W1 freeze deliverable)

The lesson signature = `lesson:trigger-class | gotcha-class | corrective-class` (key-space-prefixed). The **W1
freeze deliverable is three artifacts, frozen together:** (1) the final enum values, (2) the composite-key
constructor + namespacing vs `frictionClusterKey`, (3) an **each-value-assignable audit** — every value must be
assignable to a *resolved-issue fix attempt* (drop process-only tokens that can never fire), and the values must
**reconcile against the 7 existing friction-bearing nodes** (the freeze is not against a blank slate). Cardinality
~8-11/axis: enough orthogonal cells that distinct traps land apart (DEF-3 watches for collisions), few enough to
stay hashable + auditable. **`friction_phase` is NOT a key axis** (YAGNI — no consumer query filters on it; keep it
a top-level diagnostic field only).

**`trigger-class`** (the situation recognizer): `env-setup` · `boundary-contract` · `state-mutation` ·
`auth-or-gate` · `concurrency` · `external-io` · `data-parse` · `api-shape` · `test-harness` · `path-or-fs`

**`gotcha-class`** (the trap): `mock-not-real` · `subset-not-exact-set` · `read-path-not-frozen` ·
`path-not-canonicalized` · `silent-coercion` · `stale-premise` · `harness-capability-assumed` ·
`ordering-dependency` · `partial-fix-basename` · `content-not-verified-on-read`

**`corrective-class`** (the principle): `probe-the-real-path` · `exact-set-equality` · `freeze-deep` ·
`canonicalize-before-check` · `verify-content-on-read` · `fail-closed` · `premise-probe-the-mitigation` ·
`narrow-not-block`

(Trimmed per the board's each-value-assignable note: the process-only tokens `survivorship-bias`,
`branch-off-main-first`, `dogfood-before-claim` were removed — they describe how the *team* fails, not how a *fix
attempt* fails, so they would be permanent dead frozen cells. W1 finalizes the audit.)

## 7. Wave plan (shadow-first; each exit is an internal-N discrimination claim)

- **W1 — the organ as machinery + the capture re-run.** Ratify + FREEZE the taxonomy triplet (Sec 6). Build the
  lesson-node schema (persisting candidate patch + accepted diff ref + `fail_to_pass` + friction block); the
  derivation leg (advisory LLM contrast on the re-run's persisted candidate vs accepted content); the
  string-variant leak-guard; `lessonClusterKey`; exact-match merge + raw recurrence count + the DEF-3 storage seam.
  Run a **small capture re-run** over the seed corpus issues to mint the first real lessons. **Exit:**
  `derive -> lessonClusterKey -> leak-guard -> store` runs end-to-end on the re-run's captured nodes; the leak-guard
  never emits a >=12-char run shared with a sealed diff; two same-signature lessons merge into one weighted node.
  (TDD the machinery on fixtures first; the re-run proves it on real captured data.)
- **W2 — the gate + the first edge.** The same-`fail_to_pass` confirmation gate (the cross-run join, Sec 5) + the
  `(failure-context) --confirmed-by--> (delta-ref)` edge, composed at the call site (not merged into the
  contamination gate). **Exit:** a lesson *provably cannot* enter the predictor lane without a same-requirement
  confirming delta; unconfirmed sits in a hazard lane.
- **W3 — data-gated (the payoff + the trap seam).** A re-run that also **captures FAILED attempts**
  (`failed_attempt_ref`) so `contrast(wrong-diff, accepted-fix)` is computable -> consolidation pass +
  trigger-retrieval (signature-match). **Exit:** trigger-retrieval *measurably* beats the lexical floor on a corpus
  with real same-signature collisions present (an N-floor precondition, not a noise margin). Signature-retrieval
  cannot precede the lesson-signature-bearing nodes the re-runs produce — do not front-load the measurement.

## 8. Honest data-readiness (the recurring trap, named — firsthand-corrected)

Firsthand census of `~/.claude/lab-state/recall-graph-backtest/` (2026-06-15, all 11 nodes tallied, not sampled):

- **The candidate patch is GONE (the board CRITICAL).** `candidate_patch_ref` is `sha256(candidate).slice(0,16)`
  — a dead 16-char digest with no inverse; the candidate patch content was never persisted (no sidecar; `find`
  returns nothing). The node stores only `{issue_id, repo, problem_statement_digest, candidate_patch_ref,
  behavioral_verdict, reference_divergence, contamination_tier}`. The `accepted_diff` IS recoverable (the corpus
  `seed-manifest.json`). **So the 11 seed nodes cannot be derived into lessons** — the *attempt* whose divergence
  *is* the lesson is unrecoverable. This is why D2 was revised to a **capture re-run** (persist candidate +
  accepted + `fail_to_pass` + friction) rather than "derive from the existing 11."
- **Signatures:** 4 null, 7 friction-bearing (the *failure* taxonomy, not lesson) — the freeze reconciles against
  those 7, not a blank slate. **Divergence:** 8 scalar, 3 null. A divergence *scalar* is not the attempt-vs-accepted
  *content* a lesson needs regardless.
- **11 < the floor of 20** (INSUFFICIENT-N). The build is the easy half; *confirming a lesson discriminates* needs
  N beyond the seed. Wave exits are discrimination claims so we cannot declare done on machinery alone. **"Data is
  the gap" recurs, and more acutely than first stated** — not just thin, but the derivation substrate for the
  existing nodes is absent. Named risk, not a surprise.

## 9. NOT building (YAGNI / explicit rejections)

- **No embedding as the identity key.** **No second physical "slow store"** (CLS motivates two *write policies* =
  the gate, not a second store; we have no interference problem). **No kernel record deletion** (fights INV-22).
  **No bending the live K4 kernel recall-CLI** (MAJOR-protected; the retriever stays in lab). **No LLM-discretionary
  self-editing memory** (borrow the ADD/UPDATE/NOOP vocabulary; make the decision deterministic/gated). **No
  embedding-keyed PPR / spreading-activation retriever** (runs on topology that does not exist yet — build the first
  edge first). **No auto-firing graduation** (manual, threshold 5). **No A6 lesson-bridge to a live spawn this
  phase** (the v-next consumer; the graph must have edges first).

## 10. Open residuals / carries

- **G-AGNOSTIC** — a model-portability check for lesson *content*. Likely a test, not a build.
- **The trap-seam re-run** (W3) — captures *failed* attempts; the only piece needing genuinely new data beyond the
  W1 capture re-run. USER to decide later-in-v3.11 vs v3.12. Timeouts are a *complexity hazard* sub-type, not a trap.
- **The live retriever / OQ-7** — any future *live* retriever must be empirically PROVEN to filter
  `recall-graph-backtest/` by provenance (the Runtime-Claim Probe), never assumed.
- **`clusterFriction` positional-index caveat** (board M3) — the consolidation pass must pass the full
  unsorted/unfiltered list + build the parallel `refs` array first.
- **DEF-1/2/3** — reversible.
