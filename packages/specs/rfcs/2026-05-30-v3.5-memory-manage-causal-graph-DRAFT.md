# DRAFT RFC: v3.5 Memory ‚Äî Manage Layer + Typed Causal-Edge Schema (post-pair-review v2)

> **Status discipline.** DRAFT for a scope decision. Amends nothing. v6 is LOCKED.
> The two deliverables ‚Äî the manage-ops-as-transactions table (¬ß3) and the typed causal-edge
> schema (¬ß4) ‚Äî exist so we can decide whether this is v3.5 scope before any line is committed.
> **v2 absorbs a 3-lens pair review (¬ß10); the security lens changed the design materially.**

## 0. What this is / is NOT

**IS:** a re-grounding of the v3.2 causal-recall RFC's two most-deferred ideas ‚Äî the "manage" phase
(its dream cycles) and the cross-file causal graph ‚Äî onto v6's transaction substrate + the verified
2026 memory literature.

**IS NOT:** an implementation spec, a v6 amendment, or a claim anything ships before v3.3. No new
kernel primitive, axiom, or invariant. (It DOES add **schema-additive fields** on the existing
¬ß4.2 record type, and proposes **new derived lifecycle states** ‚Äî both additive per
INV-K2-SchemaForwardCompat / ¬ß5a.1's "states are projections, not stored.")

**Load-bearing reframe:** in v6 the chain is canonical and every state change is a transaction
(A8 + ¬ß5a.1). So the manage layer is **a new class of transactions + policies**, and a *semantic*
causal edge is a canonical record ‚Äî needing no new primitive. **Correction from v1 (honesty HIGH-1):**
this does NOT mean dedup is free. v6's `idempotency_key` includes `prev_state_hash`, so it is
**transaction-replay** dedup (no-op on identical retry), NOT content-dedup of equivalent blocks
authored by different spawns. Content-exact-dedup is a v3.5 design item, not an existing capability.

## 1. Organizing principle ‚Äî deterministic manage vs stochastic manage

| | **Deterministic manage** | **Stochastic manage** |
|---|---|---|
| Decided by | pure chain-walk / policy predicate | LLM judgment (advisory, A3b) |
| Trust | attestation-grade | stochastic sample ‚Äî evidence-anchored, **never read as fact** |
| Driver | kernel write-path / GC sweep | dream-cycle (E13) |
| Examples | evidence-driven staleness, retention-archive (both as **projections**) | near-dup merge, contradiction detection, abstraction-typing |
| Failure mode | none (re-derivable) | experience-following self-reinforcement ‚Üí needs out-of-band ground truth (v6 OQ-12) |

**Honest bound (honesty MEDIUM-3):** the deterministic column is **real but narrow** ‚Äî it covers
exactly the ops whose triggering invalidation is *already a recorded kernel transaction*. Today that
is essentially evidence-driven staleness (a `SUPERSEDE` chain-walk). It does **not** generalize to
content-dedup or to externally-triggered staleness the substrate never witnessed. The strategy is
still "push what you can left, ruthlessly evidence-anchor what stays right" ‚Äî just thinner than it
first looked.

## 2. Field-integrity & authorization rules (the security spine ‚Äî NEW v2)

These govern BOTH deliverables. They exist because the v1 draft optimized the *write*-side and left
the *authority* model open (security F3/F4).

> **Closure-honesty preamble (2nd security pass, 2026-05-30).** These rules CLOSE the integrity
> threats at the **design layer**; their **enforcement** is v3.5 work tracked in ¬ß6‚Äì¬ß8. R3/R4 are
> K9-pre-commit-checkable with existing fields and add no primitive. **R1's enforcement** (kernel
> overwrite of trust-bearing fields) and **R2's authority gate** are *not* free: R2 presupposes a
> **kernel-attested writer identity** that the LOCKED v6 substrate does not yet have (see OQ-E) ‚Äî so
> the "none is a new primitive" claim is scoped to R3/R4 (and R1's *principle*), NOT to R2's
> enforcement. Where a rule's parenthetical below says "Mitigates" or "design-closes," read it as
> *narrowed to a fail-closed default pending the named OQ*, not settled. See OQ-E for the load-bearing
> open question.

- **R1 ‚Äî Trust-bearing fields are kernel-derived, never LLM-set.** `source_origin`, `assertion_class`,
  `abstraction_level`, `faithfulness_status`, and computed `confidence` MUST be derived by the kernel
  from the evidence chain at commit time, overwriting any persona-supplied value. A persona's proposed
  value is stored separately as `claimed_*` and is advisory-only. `source_origin` is derived from the
  A10-protected evidence chain (genuinely forgery-resistant). **Caveat (2nd security pass): two clauses
  have a latent dependency on `writer_persona_id`, which is itself NOT kernel-attested in the LOCKED
  envelope nor in the shipped recorder** (`spawn-record.js` derives the persona label from input ‚Äî
  `toolInput.subagent_type || toolInput.subagent || toolInput.type` ‚Äî while `spawn_id` is kernel-minted):
  (a) `assertion_class`'s `kernel‚Üíattestation, else stochastic_sample` discrimination keys off
  `writer_persona_id`; (b) any kernel-vs-stochastic test does. **Fail-closed interim default until OQ-E
  resolves kernel-attested writer identity:** `assertion_class` derived from an un-attested persona label
  MUST default to `stochastic_sample` (never auto-promote to `attestation` on an unverified writer), and
  `faithfulness_status` on any stochastic-source CREATE MUST default to `unvalidated` ‚Äî no status upgrade
  without a kernel-attested transaction (self-cert via persona-issued SUPERSEDE is rejected). The stored
  `abstraction_level` is the persona's CLAIMED value off the promotion path; the kernel re-derives it for
  every trust decision (promotion, ranking, display). (**Design-closes F3** ‚Äî the rule + the
  `source_origin`-from-evidence-chain derivation close source_origin spoofing in principle; K9-overwrite
  *enforcement* and the `writer_persona_id` attestation it leans on are deferred to v3.5 K9/K2 per ¬ß6 +
  OQ-E.)
- **R2 ‚Äî Destructive manage ops are authority-gated.** A `TOMBSTONE` or `SUPERSEDE` against an
  `active` canonical record is admissible ONLY when `writer_persona_id ‚àà {kernel:gc-sweep,
  kernel:retention, kernel:source-trust}` OR the record carries an `operator-signed` authority token.
  A non-kernel persona's destructive op against an active record is REJECTED pre-commit
  (`unauthorized-destructive-manage`). A10's evidence-*existence* check is necessary but NOT
  sufficient to authorize deletion. **Critical gap (2nd security pass): R2 presupposes a primitive the
  LOCKED substrate lacks.** Both branches are un-enforceable in isolation today ‚Äî `writer_persona_id` is
  persona-supplied (R1 caveat above; conspicuously ABSENT from R1's kernel-derived list), so K9
  pre-commit *can* compare the string to the kernel set but has no *trustworthy* string to compare; and
  the `operator-signed` token's issuer/verifier is undefined (OQ-E). Field-stuffing `writer_persona_id:
  kernel:gc-sweep` reproduces the F4 threat with extra steps. R2 therefore needs **kernel-attested
  writer identity** ‚Äî a spawn-init roster check, OR signing of the `writer_persona_id` claim, OR a
  kernel-exclusive WAL-write path for the three `kernel:*` writers ‚Äî which IS a new primitive/discipline
  the v3.5 spec must add (deferred to OQ-E). **Fail-closed interim default:** until OQ-E resolves, the
  `operator-signed-token` path is treated as UNAVAILABLE (unspecified ‚áí disabled), collapsing R2 to a
  kernel-personas-only destroy gate; and because the persona label is not yet attested, a destructive op
  is admitted ONLY if the surrounding execution context independently corroborates a `kernel:*` writer
  (the K9 string-set check is the field-value predicate, NOT an authentication). (**Mitigates
  manage-as-censorship to a fail-closed default, security F4 CRITICAL ‚Äî the most dangerous gap: erasure
  leaves audit-clean history. NARROWED, not closed, until OQ-E specs the token issuer/verifier AND the
  kernel-attested-writer-identity primitive.**)
- **R2-note (quarantine ‚Äî intentionally outside R2).** Quarantine (¬ß3 table) is retrieval-suppression,
  not deletion; the record stays WAL-retained. It is deliberately Lab-advisory (A3b), NOT a kernel
  write-path op (OQ-D), to keep the LOCKED A10 surface untouched. The **kernel-derived taint signal**
  required in its `evidence_refs` is the equivalent authority gate ‚Äî Lab advisory may PROPOSE the
  quarantine marker but the marker is inadmissible at K9 pre-commit without that kernel-derived
  evidence (so a dream-cycle cannot suppress an arbitrary record on its own say-so). **Un-quarantine is
  available to kernel-authority writers.** This is an honest architectural choice, not an R2 oversight;
  full read-side-suppression authority unification with R2 is folded into OQ-E.
- **R3 ‚Äî Read-tier gate on causal edges.** Edges with `faithfulness_status ‚àà {unvalidated,
  surface_overlap_only}` are **AUDIT-ONLY** ‚Äî invisible to read-side context-assembly (OQ-27 walk).
  Only `{advisory_llm_checked, human_confirmed}` edges are **traversal-eligible**. The gate is a
  deterministic enum-membership check (never LLM-graded). **AUDIT-ONLY applies to ALL read paths**, not
  only the OQ-27 walk: an AUDIT-ONLY edge MUST NOT appear in any K4 recall-CLI result set, MUST NOT be
  snapshotted to axiom-class via A6 (its trust-class is Stochastic Sample, not Axiom), and MUST be
  excluded from any future traversal seed. **Self-cert closed (2nd security pass):** the upgrade from
  `unvalidated`/`surface_overlap_only` to `advisory_llm_checked` requires a **kernel-attested**
  transaction (R1) ‚Äî a stochastic-sample emitter (dream-cycle) CANNOT promote its own edge via a
  persona-issued SUPERSEDE/APPEND. **Enforcement contingency:** the gate's consumer ‚Äî the read-side
  context-assembly walker ‚Äî is v6 OQ-27 ("the deepest under-specified concern"), not yet built. R3 is
  therefore the **invariant the OQ-27 implementer MUST honor**, enforced once that wiring lands.
  **Fail-closed default until OQ-27 resolves: causal-edge traversal is PROHIBITED in production read
  paths** (no walker ‚áí no edge is traversal-eligible, regardless of `faithfulness_status`). (**Design-
  closes trust laundering, security F1 CRITICAL** ‚Äî the AUDIT-ONLY invariant is sound and
  read-side-checkable with existing fields; live enforcement rides on OQ-27 / OQ-F.)
- **R4 ‚Äî Closed enums + canonicalization.** `relation` and `conflict_type` are **new edge-specific**
  closed enums introduced additively at ¬ß4 and validated at K5; `operation_class` is the **existing
  v6-LOCKED enum** = {`CREATE`, `APPEND`, `SUPERSEDE`, `TOMBSTONE`, `DERIVED-VIEW-INVALIDATE`} ‚Äî
  R4 validates membership against it and does **NOT** extend it (¬ß3 mints no new `operation_class`;
  reading ¬ß3 as a v6 amendment would be wrong). `mem://` URIs and `surface` paths are K7-canonicalized
  before any equality/overlap check. **K5 enum validation MUST normalize to ASCII/NFC before the
  membership check** (the closed enums contain ASCII strings only; any non-ASCII byte sequence in an
  enum field is K5-rejected) ‚Äî otherwise a homoglyph/case bypass (e.g. Cyrillic `–µ` in `caused_by`)
  passes naive string-equality. (Closes CWE-20/CWE-22, security F6.)

## 3. DELIVERABLE 1 ‚Äî Manage operations as transactions

Columns: v6 `operation_class` it emits (or "projection" = no record), `evidence_refs` (A10),
lifecycle transition (¬ß5a.1; `*`=net-new state, see ¬ß3.1), who may author it (R2), det/stochastic.

| Manage op | emits | `evidence_refs` | lifecycle | who authors | det? |
|---|---|---|---|---|---|
| **content-dedup** | `SUPERSEDE` | `[dupe_a, dupe_b]` | dupe ‚Üí `superseded` | kernel sweep (R2) | ‚úó (near-dup is LLM) |
| **merge / consolidate** | `CREATE` summary, `status: candidate`, stochastic-tagged | `[raw_1..raw_n]` | dream-cycle | mixed | ‚úó |
| **supersede (new info)** | `SUPERSEDE` | `[old, justifying_record]` | old ‚Üí `superseded` | kernel or operator-signed (R2) | mixed |
| **flag-conflict** | `CREATE` causal-edge (`relation=contradicts`) | `[x, y, conflict_evidence]` | x,y ‚Üí `conflicted`* | dream-cycle | ‚úó |
| **mark-stale** | **projection ‚Äî NO record** | (none; derived) | `active` ‚Üí `stale`* | n/a (computed) | ‚úÖ |
| **retention-archive** | **projection over a retention-policy record** | policy record only | `active` ‚Üí `archived`* | kernel sweep | ‚úÖ |
| **cull / tombstone** | `TOMBSTONE` | `[justification]` | ‚Üí `tombstoned` (WAL-retained) | kernel/operator-signed (R2) | mixed |
| **quarantine** | `CREATE` marker (suppress-from-retrieval) | `[kernel-derived taint signal]` | `active` ‚Üí `quarantined`* | Lab advisory (A3b) | mixed |
| **promote (cross-scope)** | `CREATE` at higher scope | `[lower-scope block]` | `candidate` ‚Üí `active` | dream-cycle, **gated** | ‚úó |
| **demote / reduce-trust** | projection (ranking) | `[utility record]` | confidence ‚Üì | Evolution-Lab (v3.4) | ‚úÖ |

**Corrections absorbed (architect F1/F2):** `mark-stale` and `retention-archive` emit **no record** ‚Äî
they are pure chain-walk projections (a block is `stale` when a later COMMITTED `SUPERSEDE` invalidates
a record in its transitive `evidence_refs`; `archived` when age > policy AND no superseder/tombstone).
v1's `DERIVED-VIEW-INVALIDATE`/`TOMBSTONE` mappings were category errors ‚Äî `DERIVED-VIEW-INVALIDATE`
is a *cache-clear for derived siblings* (¬ß4.2/¬ß5a.1), and `archived ‚âÝ tombstoned` (audit distinction:
aged-out-but-valid vs deleted-because-wrong). `merge` starts `candidate`, never `active` (architect F3).
**"stochastic-tagged" operationalized (2nd security pass):** every CREATE from a stochastic-sample
source (dream-cycle, Lab advisory ‚Äî `merge`/`flag-conflict`/`quarantine`-marker) starts with
`faithfulness_status=unvalidated` per R1's fail-closed default, regardless of the emitter's self-
assessment at creation time. Upgrading requires a kernel-attested transaction (closes the merge-record
self-cert path alongside R3's).

### 3.1 Proposed net-new lifecycle states (additive projections; NOT in v6 today)

`stale`, `archived`, `conflicted`, `quarantined` are **new derived states** to add to the ¬ß5a.1 table.
v6 ¬ß5a.1 says lifecycle states are pure projections (not stored), so adding them is additive-by-
construction ‚Äî but they are flagged net-new here so this is read as *extending* the enum, not assuming it.

**Read-side policy for each net-new state (2nd security pass ‚Äî prevents non-R2 retrieval-layer
censorship):** `quarantined` is retrieval-suppressed (its whole purpose; gated by the R2-note kernel-
derived taint signal). `conflicted` records **remain retrieval-eligible** ‚Äî the state is an annotation
for human review, NOT a suppression; a `flag-conflict` (dream-cycle, not R2-gated) MUST NOT remove
either record from the OQ-27 walk (else a spurious contradiction buries two legitimate records without
passing R2). `stale`/`archived` are de-prioritized, not excluded. This keeps every retrieval-affecting
suppression either R2-gated or kernel-taint-gated.

**Promotion gate (the MTL operationalization ‚Äî a pure predicate, kernel-verified per R1):**
```
admit_promotion(block, target_scope) :=
      kernel_abstraction_level(block) ‚àà allowed_for(target_scope)   # R1: kernel-derived, NOT self-labeled
  AND count(distinct source_blocks in evidence_refs) ‚â• insight_floor(target_scope)   # ‚â•3 for `insight`
  AND lifecycle(block) == active   # i.e. not stale / quarantined / superseded
  AND no in-flight quarantine PENDING for block   # predicate-at-evaluation; race mitigation ‚Üí OQ-G
```

**Promote‚Üîquarantine race (2nd security pass ‚Äî downgraded from "closed"):** the predicate is correct
*at the instant it is evaluated*, but K13 serializes **spawns**, not Lab-advisory quarantine markers
(an async write outside the spawn-boundary), so K13 does not by itself close this race (sec F5a). The
honest mechanism is deferred to **OQ-G** with a fail-closed default (reject the promote if an atomic
check-and-commit against the block's lifecycle cannot be guaranteed).

**Forget-vs-audit knot is already untied (verified):** v6 ¬ß5a.1 ‚Äî `TOMBSTONE` removes from canonical
*view*, retains in WAL for audit. No new mechanism. **Caveat:** `merge` reduces *retrieval surface*,
not storage ‚Äî raw blocks are retained to avoid summary drift (our design choice for auditability).

## 4. DELIVERABLE 2 ‚Äî Typed causal-edge schema (A‚Ä≤ hybrid)

**Architect's design correction (adopted).** There are two edge classes, and only one is a record:

- **Provenance edges** (kernel-asserted) are NOT minted as records ‚Äî they ARE the existing
  `evidence_refs` / `prev_state_hash` / `parent_state_id` chain, surfaced as a **derived view**
  (pure projection, trustworthy by construction).
- **Semantic edges** (LLM-asserted) ARE canonical `CREATE` records (`node_type=causal-edge`), but
  start `candidate`, carry stochastic-content tagging, are subject to the promotion/read-tier gates,
  and **age out by retention** if never validated (so the WAL doesn't fill with permanent guesses).

This makes "an edge is just a canonical record" *true for the half that needs durability* and
correctly *false for the provenance half already in the chain* (architect Alternative A‚Ä≤).

A semantic-edge record = the ¬ß4.2 transaction envelope (inherited verbatim ‚Äî do NOT re-declare
`transaction_id`/`prev_state_hash`/`writer_persona_id`/`operation_class`/`affected_records`/
`evidence_refs`/`intent_recorded_at`/`commit_outcome`/`idempotency_key`) **plus** these edge-specific
extension fields ONLY (architect F5 ‚Äî DRY; ~6 new fields, not a shadow schema):

```yaml
# affected_records := [source_block, target_block]   (inherited; not re-declared)
relation:            caused_by | depends_on | validated_by | contradicts | supersedes
                     | regressed_after | fixed_by | reviewed_by | blocked_by   # closed enum (R4)
conflict_type:       temporal | factual | contextual | conditional | null      # iff relation=contradicts
source_origin:       KERNEL-DERIVED from evidence chain (R1); persona value stored as claimed_source_origin
assertion_class:     DERIVED from writer_persona_id (kernel‚Üíattestation, else stochastic_sample) ‚Äî not stored (R1/architect F4)
faithfulness_status: unvalidated | surface_overlap_only | advisory_llm_checked | human_confirmed
abstraction_level:   raw_trace | workflow | summary | insight   # kernel-verified when gating promotion (R1)
# lifecycle (candidate/active/stale/conflicted/...): DERIVED projection per ¬ß5a.1 ‚Äî NOT a stored field
# confidence: DERIVED (evidence-strength only at v3.5; √ó utility once Evolution-Lab v3.4 lands) ‚Äî never trust-by-frequency (¬ß0a.3.1)
```

**`assertion_class` is the load-bearing field (architect F4 ‚Äî the draft's best idea).** It makes v6's
¬ß4.1 content/emission split *structural*: a provenance edge's emission AND content are kernel
attestations; a semantic edge's emission is an attestation but its *claim* is a Stochastic Sample ‚Äî
**never read as fact while not `advisory_llm_checked`+** (R3).

**The OQ-21 faithfulness ladder (renamed per security F2 ‚Äî rung 1 was security theater):**
1. `surface_overlap_only` (deterministic): `source_block.surface ‚à© evidence.delta.paths ‚âÝ ‚àÖ` after
   K7-canonicalization. Confirms the edge references the same files as its evidence. It does **NOT**
   validate causal direction, temporal order, or semantic content ‚Äî so it is **AUDIT-ONLY (R3)**,
   never routing-eligible.
2. `advisory_llm_checked` (A3b, non-blocking): LLM judges claim‚Üîevidence faithfulness; **minimum bar
   for traversal eligibility.**
3. `human_confirmed`: required for any edge that gates routing at L_persona/L_global.

## 5. Worked example (post-review)

```
PROVENANCE edge (derived view; kernel-asserted; trustworthy): R1.evidence_refs ‚Üí [R3,R4]
SEMANTIC edge (canonical CREATE; LLM-asserted; stochastic):

```

## 6. Honest scope ‚Äî cheap-and-near vs hard-and-deferred

**Cheap / near (schema-additive, dormant; could land v3.1‚Äìv3.3, NOT v3.0-alpha):**
- `source_origin` as a **kernel-derived FACT** on records (R1). Cheap, but note: deriving it touches
  K9/K2 (the kernel surface), so even this reservation is not free of LOCK-adjacent review.
- Evidence-driven staleness as a chain-walk projection ‚Äî **rides on the OQ-27 traversal infra** (it's
  a narrower query than full context-assembly, so cheaper, but not standalone-trivial).

**Hard / deferred (v3.5+; the actual research frontier):**
- Contradiction *detection* ‚Äî STALE/MemConflict say best-model 55.2%; ours is LLM-judged + evidence-
  anchored, **honestly weak**.
- `faithfulness_status` rungs 2‚Äì3 (OQ-21 semantic admission) ‚Äî the gate for any routing use.
- The read-side traversal algorithm (OQ-27 ‚Äî v6's "deepest under-specified concern").
- Utility-linked confidence (needs the Evolution-Lab outcome loop, v3.4).
- The R2 authority model + R1 kernel-derivation (the security spine) ‚Äî design, not yet specced.

## 7. What this explicitly does NOT solve (expanded post-review)

- **Does not provide content-exact-dedup** ‚Äî v6 idempotency is transaction-replay only (honesty HIGH-1).
- **Does not solve STALE's hardest case** ‚Äî only catches invalidations the substrate witnessed as a
  transaction; external-world staleness is open.
- **Does not make LLM causal claims trustworthy** ‚Äî only quarantinable, evidence-anchored, never-read-
  as-fact-until-validated, and (R2) not usable to erase true records. Containment, not correctness.
- **Does not make causal edges safe to gate routing** ‚Äî only `surface_overlap_only` is deterministic,
  and it's AUDIT-ONLY; routing needs rungs 2‚Äì3 (OQ-21, unsolved).
- **Does not reduce storage** ‚Äî WAL still grows; v6 OQ-24 (compaction) is the relevant lever.
- **Adds no kernel primitive/axiom/invariant** ‚Äî but adds ~6 schema fields + 4 net-new derived
  lifecycle states + the R1‚ÄìR4 rules (which want invariants to be *enforceable*; that's v3.5 work).

## 8. Open questions (with post-review resolutions)

- **OQ-A (v6 OQ-21):** faithfulness ‚Äî resolved direction: `surface_overlap_only` is AUDIT-ONLY;
  `advisory_llm_checked` is the minimum routing gate (security F1/F2 + R3).
- **OQ-B (v6 OQ-27):** read-side seed + edge-walk policy ‚Äî still open; K4 tri-signal ranker is the
  v3.0-alpha starting point and addresses neither. **Fail-closed default:** until OQ-27 lands a walker
