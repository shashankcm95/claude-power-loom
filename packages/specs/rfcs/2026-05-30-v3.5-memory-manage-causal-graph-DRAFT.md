# DRAFT RFC: v3.5 Memory — Manage Layer + Typed Causal-Edge Schema (post-pair-review v2)

> **Status discipline.** DRAFT for a scope decision. Amends nothing. v6 is LOCKED.
> The two deliverables — the manage-ops-as-transactions table (§3) and the typed causal-edge
> schema (§4) — exist so we can decide whether this is v3.5 scope before any line is committed.
> **v2 absorbs a 3-lens pair review (§10); the security lens changed the design materially.**

## 0. What this is / is NOT

**IS:** a re-grounding of the v3.2 causal-recall RFC's two most-deferred ideas — the "manage" phase
(its dream cycles) and the cross-file causal graph — onto v6's transaction substrate + the verified
2026 memory literature.

**IS NOT:** an implementation spec, a v6 amendment, or a claim anything ships before v3.3. No new
kernel primitive, axiom, or invariant. (It DOES add **schema-additive fields** on the existing
§4.2 record type, and proposes **new derived lifecycle states** — both additive per
INV-K2-SchemaForwardCompat / §5a.1's "states are projections, not stored.")

**Load-bearing reframe:** in v6 the chain is canonical and every state change is a transaction
(A8 + §5a.1). So the manage layer is **a new class of transactions + policies**, and a *semantic*
causal edge is a canonical record — needing no new primitive. **Correction from v1 (honesty HIGH-1):**
this does NOT mean dedup is free. v6's `idempotency_key` includes `prev_state_hash`, so it is
**transaction-replay** dedup (no-op on identical retry), NOT content-dedup of equivalent blocks
authored by different spawns. Content-exact-dedup is a v3.5 design item, not an existing capability.

## 1. Organizing principle — deterministic manage vs stochastic manage

| | **Deterministic manage** | **Stochastic manage** |
|---|---|---|
| Decided by | pure chain-walk / policy predicate | LLM judgment (advisory, A3b) |
| Trust | attestation-grade | stochastic sample — evidence-anchored, **never read as fact** |
| Driver | kernel write-path / GC sweep | dream-cycle (E13) |
| Examples | evidence-driven staleness, retention-archive (both as **projections**) | near-dup merge, contradiction detection, abstraction-typing |
| Failure mode | none (re-derivable) | experience-following self-reinforcement → needs out-of-band ground truth (v6 OQ-12) |

**Honest bound (honesty MEDIUM-3):** the deterministic column is **real but narrow** — it covers
exactly the ops whose triggering invalidation is *already a recorded kernel transaction*. Today that
is essentially evidence-driven staleness (a `SUPERSEDE` chain-walk). It does **not** generalize to
content-dedup or to externally-triggered staleness the substrate never witnessed. The strategy is
still "push what you can left, ruthlessly evidence-anchor what stays right" — just thinner than it
first looked.

## 2. Field-integrity & authorization rules (the security spine — NEW v2)

These govern BOTH deliverables. They exist because the v1 draft optimized the *write*-side and left
the *authority* model open (security F3/F4). All are K9-pre-commit-checkable; none is a new primitive.

- **R1 — Trust-bearing fields are kernel-derived, never LLM-set.** `source_origin`, `assertion_class`,
  `abstraction_level` (when used to gate promotion), and computed `confidence` MUST be derived by the
  kernel from the evidence chain at commit time, overwriting any persona-supplied value. A persona's
  proposed value is stored separately as `claimed_*` and is advisory-only. (Closes source_origin
  spoofing, security F3 CRITICAL.)
- **R2 — Destructive manage ops are authority-gated.** A `TOMBSTONE` or `SUPERSEDE` against an
  `active` canonical record is admissible ONLY when `writer_persona_id ∈ {kernel:gc-sweep,
  kernel:retention, kernel:source-trust}` OR the record carries an `operator-signed` authority token.
  A non-kernel persona's destructive op against an active record is REJECTED pre-commit
  (`unauthorized-destructive-manage`). A10's evidence-*existence* check is necessary but NOT
  sufficient to authorize deletion. (Closes manage-as-censorship, security F4 CRITICAL — the most
  dangerous gap: erasure leaves audit-clean history.)
- **R3 — Read-tier gate on causal edges.** Edges with `faithfulness_status ∈ {unvalidated,
  surface_overlap_only}` are **AUDIT-ONLY** — invisible to read-side context-assembly (OQ-27 walk).
  Only `{advisory_llm_checked, human_confirmed}` edges are **traversal-eligible**. (Closes trust
  laundering, security F1 CRITICAL.)
- **R4 — Closed enums + canonicalization.** `relation`, `conflict_type`, `operation_class` are
  closed enums validated at K5; `mem://` URIs and `surface` paths are K7-canonicalized before any
  equality/overlap check. (Closes CWE-20/CWE-22, security F6.)

## 3. DELIVERABLE 1 — Manage operations as transactions

Columns: v6 `operation_class` it emits (or "projection" = no record), `evidence_refs` (A10),
lifecycle transition (§5a.1; `*`=net-new state, see §3.1), who may author it (R2), det/stochastic.

| Manage op | emits | `evidence_refs` | lifecycle | who authors | det? |
|---|---|---|---|---|---|
| **content-dedup** | `SUPERSEDE` | `[dupe_a, dupe_b]` | dupe → `superseded` | kernel sweep (R2) | ✗ (near-dup is LLM) |
| **merge / consolidate** | `CREATE` summary, `status: candidate`, stochastic-tagged | `[raw_1..raw_n]` | dream-cycle | mixed | ✗ |
| **supersede (new info)** | `SUPERSEDE` | `[old, justifying_record]` | old → `superseded` | kernel or operator-signed (R2) | mixed |
| **flag-conflict** | `CREATE` causal-edge (`relation=contradicts`) | `[x, y, conflict_evidence]` | x,y → `conflicted`* | dream-cycle | ✗ |
| **mark-stale** | **projection — NO record** | (none; derived) | `active` → `stale`* | n/a (computed) | ✅ |
| **retention-archive** | **projection over a retention-policy record** | policy record only | `active` → `archived`* | kernel sweep | ✅ |
| **cull / tombstone** | `TOMBSTONE` | `[justification]` | → `tombstoned` (WAL-retained) | kernel/operator-signed (R2) | mixed |
| **quarantine** | `CREATE` marker (suppress-from-retrieval) | `[kernel-derived taint signal]` | `active` → `quarantined`* | Lab advisory (A3b) | mixed |
| **promote (cross-scope)** | `CREATE` at higher scope | `[lower-scope block]` | `candidate` → `active` | dream-cycle, **gated** | ✗ |
| **demote / reduce-trust** | projection (ranking) | `[utility record]` | confidence ↓ | Evolution-Lab (v3.4) | ✅ |

**Corrections absorbed (architect F1/F2):** `mark-stale` and `retention-archive` emit **no record** —
they are pure chain-walk projections (a block is `stale` when a later COMMITTED `SUPERSEDE` invalidates
a record in its transitive `evidence_refs`; `archived` when age > policy AND no superseder/tombstone).
v1's `DERIVED-VIEW-INVALIDATE`/`TOMBSTONE` mappings were category errors — `DERIVED-VIEW-INVALIDATE`
is a *cache-clear for derived siblings* (§4.2/§5a.1), and `archived ≠ tombstoned` (audit distinction:
aged-out-but-valid vs deleted-because-wrong). `merge` starts `candidate`, never `active` (architect F3).

### 3.1 Proposed net-new lifecycle states (additive projections; NOT in v6 today)

`stale`, `archived`, `conflicted`, `quarantined` are **new derived states** to add to the §5a.1 table.
v6 §5a.1 says lifecycle states are pure projections (not stored), so adding them is additive-by-
construction — but they are flagged net-new here so this is read as *extending* the enum, not assuming it.

**Promotion gate (the MTL operationalization — a pure predicate, kernel-verified per R1):**
```
admit_promotion(block, target_scope) :=
      kernel_abstraction_level(block) ∈ allowed_for(target_scope)   # R1: kernel-derived, NOT self-labeled
  AND count(distinct source_blocks in evidence_refs) ≥ insight_floor(target_scope)   # ≥3 for `insight`
  AND lifecycle(block) == active   # i.e. not stale / quarantined / superseded
  AND no in-flight quarantine PENDING for block   # R2/K13 serialization — closes promote↔quarantine race (sec F5a)
```

**Forget-vs-audit knot is already untied (verified):** v6 §5a.1 — `TOMBSTONE` removes from canonical
*view*, retains in WAL for audit. No new mechanism. **Caveat:** `merge` reduces *retrieval surface*,
not storage — raw blocks are retained to avoid summary drift (our design choice for auditability).

## 4. DELIVERABLE 2 — Typed causal-edge schema (A′ hybrid)

**Architect's design correction (adopted).** There are two edge classes, and only one is a record:

- **Provenance edges** (kernel-asserted) are NOT minted as records — they ARE the existing
  `evidence_refs` / `prev_state_hash` / `parent_state_id` chain, surfaced as a **derived view**
  (pure projection, trustworthy by construction).
- **Semantic edges** (LLM-asserted) ARE canonical `CREATE` records (`node_type=causal-edge`), but
  start `candidate`, carry stochastic-content tagging, are subject to the promotion/read-tier gates,
  and **age out by retention** if never validated (so the WAL doesn't fill with permanent guesses).

This makes "an edge is just a canonical record" *true for the half that needs durability* and
correctly *false for the provenance half already in the chain* (architect Alternative A′).

A semantic-edge record = the §4.2 transaction envelope (inherited verbatim — do NOT re-declare
`transaction_id`/`prev_state_hash`/`writer_persona_id`/`operation_class`/`affected_records`/
`evidence_refs`/`intent_recorded_at`/`commit_outcome`/`idempotency_key`) **plus** these edge-specific
extension fields ONLY (architect F5 — DRY; ~6 new fields, not a shadow schema):

```yaml
# affected_records := [source_block, target_block]   (inherited; not re-declared)
relation:            caused_by | depends_on | validated_by | contradicts | supersedes
                     | regressed_after | fixed_by | reviewed_by | blocked_by   # closed enum (R4)
conflict_type:       temporal | factual | contextual | conditional | null      # iff relation=contradicts
source_origin:       KERNEL-DERIVED from evidence chain (R1); persona value stored as claimed_source_origin
assertion_class:     DERIVED from writer_persona_id (kernel→attestation, else stochastic_sample) — not stored (R1/architect F4)
faithfulness_status: unvalidated | surface_overlap_only | advisory_llm_checked | human_confirmed
abstraction_level:   raw_trace | workflow | summary | insight   # kernel-verified when gating promotion (R1)
# lifecycle (candidate/active/stale/conflicted/...): DERIVED projection per §5a.1 — NOT a stored field
# confidence: DERIVED (evidence-strength only at v3.5; × utility once Evolution-Lab v3.4 lands) — never trust-by-frequency (§0a.3.1)
```

**`assertion_class` is the load-bearing field (architect F4 — the draft's best idea).** It makes v6's
§4.1 content/emission split *structural*: a provenance edge's emission AND content are kernel
attestations; a semantic edge's emission is an attestation but its *claim* is a Stochastic Sample —
**never read as fact while not `advisory_llm_checked`+** (R3).

**The OQ-21 faithfulness ladder (renamed per security F2 — rung 1 was security theater):**
1. `surface_overlap_only` (deterministic): `source_block.surface ∩ evidence.delta.paths ≠ ∅` after
   K7-canonicalization. Confirms the edge references the same files as its evidence. It does **NOT**
   validate causal direction, temporal order, or semantic content — so it is **AUDIT-ONLY (R3)**,
   never routing-eligible.
2. `advisory_llm_checked` (A3b, non-blocking): LLM judges claim↔evidence faithfulness; **minimum bar
   for traversal eligibility.**
3. `human_confirmed`: required for any edge that gates routing at L_persona/L_global.

## 5. Worked example (post-review)

```
R1 mem://backend.auth.refresh-token   (block; surface=src/auth/refresh.ts)
R2 mem://frontend.auth.token-storage  (block; surface=web/auth/store.ts)
R3 failure.auth.expired-token         (K9/K14 ABORTED attestation — kernel-emitted)
R4 test.auth-refresh-regression       (COMMITTED test result — kernel-emitted)

PROVENANCE edge (derived view; kernel-asserted; trustworthy): R1.evidence_refs → [R3,R4]
SEMANTIC edge (canonical CREATE; LLM-asserted; stochastic):
  affected_records=[R1,R2], relation=caused_by, evidence_refs=[R3,R4],
  source_origin=KERNEL-DERIVED(test_harness from R4), assertion_class=stochastic_sample,
  faithfulness_status=surface_overlap_only → AUDIT-ONLY (R3): NOT returned to any spawn's context
  until advisory_llm_checked. abstraction_level=workflow, status=candidate.

STALENESS (deterministic, no record): if a later COMMITTED SUPERSEDE invalidates R3/R4,
  R1 projects `stale` via chain-walk — no LLM, no embedding, no emitted record.
```

## 6. Honest scope — cheap-and-near vs hard-and-deferred

**Cheap / near (schema-additive, dormant; could land v3.1–v3.3, NOT v3.0-alpha):**
- `source_origin` as a **kernel-derived FACT** on records (R1). Cheap, but note: deriving it touches
  K9/K2 (the kernel surface), so even this reservation is not free of LOCK-adjacent review.
- Evidence-driven staleness as a chain-walk projection — **rides on the OQ-27 traversal infra** (it's
  a narrower query than full context-assembly, so cheaper, but not standalone-trivial).

**Hard / deferred (v3.5+; the actual research frontier):**
- Contradiction *detection* — STALE/MemConflict say best-model 55.2%; ours is LLM-judged + evidence-
  anchored, **honestly weak**.
- `faithfulness_status` rungs 2–3 (OQ-21 semantic admission) — the gate for any routing use.
- The read-side traversal algorithm (OQ-27 — v6's "deepest under-specified concern").
- Utility-linked confidence (needs the Evolution-Lab outcome loop, v3.4).
- The R2 authority model + R1 kernel-derivation (the security spine) — design, not yet specced.

## 7. What this explicitly does NOT solve (expanded post-review)

- **Does not provide content-exact-dedup** — v6 idempotency is transaction-replay only (honesty HIGH-1).
- **Does not solve STALE's hardest case** — only catches invalidations the substrate witnessed as a
  transaction; external-world staleness is open.
- **Does not make LLM causal claims trustworthy** — only quarantinable, evidence-anchored, never-read-
  as-fact-until-validated, and (R2) not usable to erase true records. Containment, not correctness.
- **Does not make causal edges safe to gate routing** — only `surface_overlap_only` is deterministic,
  and it's AUDIT-ONLY; routing needs rungs 2–3 (OQ-21, unsolved).
- **Does not reduce storage** — WAL still grows; v6 OQ-24 (compaction) is the relevant lever.
- **Adds no kernel primitive/axiom/invariant** — but adds ~6 schema fields + 4 net-new derived
  lifecycle states + the R1–R4 rules (which want invariants to be *enforceable*; that's v3.5 work).

## 8. Open questions (with post-review resolutions)

- **OQ-A (v6 OQ-21):** faithfulness — resolved direction: `surface_overlap_only` is AUDIT-ONLY;
  `advisory_llm_checked` is the minimum routing gate (security F1/F2 + R3).
- **OQ-B (v6 OQ-27):** read-side seed + edge-walk policy — still open; K4 tri-signal ranker is the
  v3.0-alpha starting point and addresses neither.
- **OQ-C (merge shape):** resolved direction — CREATE-as-candidate-stochastic-sibling (NOT pure
  derived view, which would be non-pure and violate §0a.3.1) (architect F3).
- **OQ-D (quarantine placement):** resolved direction — **Lab + advisory (A3b)**, NOT kernel
  write-path (keeps the LOCKED A10 surface untouched; honors v6's advisory-before-blocking
  sequencing) (architect F7).
- **OQ-E (NEW, security F4):** the R2 authority model — kernel-only vs operator-signed-token for
  destructive manage; what issues/verifies the token? Highest-priority unresolved design question.

## 9. Scope recommendation

**This is v3.5 scope, and the pair review says it is *worth* pursuing — but only with the security
spine (§2 R1–R4) treated as load-bearing, not an afterthought.** The architecture's instincts are
sound across all three lenses (manage-as-transactions, the det/stochastic split, `assertion_class`);
the holes are in the *authority/integrity* model and have low-LoC fixes that need **no new kernel
primitive and do not amend v6**. Two cheap reservations remain worth pulling earlier (`source_origin`
kernel-fact; evidence-driven-staleness query), with the caveat that both touch K9/K2. The
differentiator that survives all three reviews is the §1 left column: deterministic, evidence-driven
manage the chat-memory field cannot do — now honestly bounded to the cases the substrate witnessed.
