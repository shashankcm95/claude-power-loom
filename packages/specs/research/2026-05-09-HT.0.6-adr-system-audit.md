### ADR-0001 invariants (reference)

Per `swarm/adrs/0001-substrate-fail-open-hook-discipline.md:24-27`, four invariants:

1. Every hook has a top-level try/catch that prevents uncaught exceptions
2. Hooks that fail return `decision: approve` (PreToolUse) or exit cleanly (other lifecycles); never block on hook errors
3. Every fail-open path goes through `logger('error', ...)` so the failure is observable in `~/.claude/logs/<hook>.log`
4. Hard-block decisions (`PreToolUse decision: block`) are reserved for genuine policy violations, not hook errors

## Bar D — Optimization (static-only)

### D.1 — Per-call full-tree read on every Edit/Write event (joins HT.0.2 finding)

The hook chain is: PreToolUse:Edit|Write → `validate-adr-drift.js` → subprocess `adr.js touched-by <file>` → `loadAllAdrs()` → `listAdrFiles().map(readAdr)`. Every Edit/Write triggers a full re-read of every ADR file. With 1 ADR currently (`0001-...md`, 124 LoC), the cost per Edit/Write is one Node startup + one file scan + one ~3KB read. Linear scaling: cost grows with ADR count.

`optimization-observation:scripts/agent-team/adr.js:67-69` (`loadAllAdrs`) — per-call full-tree read; same shape as HT.0.2 follow-up item "adr.js per-call full-tree read: ~400 file reads per session via validate-adr-drift.js Edits". Caching opportunity once ADR count climbs (currently no pressure with 1 ADR).

### D.2 — Subprocess spawn per Edit/Write event

`hooks/scripts/validators/validate-adr-drift.js:98` spawns the `adr.js` subprocess with a 3-second timeout for every matching event. Inside-hook subprocess pattern. Cost: Node startup (~50-100ms) + file scan (~1ms × ADR count) per Edit/Write.

`optimization-observation:hooks/scripts/validators/validate-adr-drift.js:98` — subprocess fired per matching Edit/Write event. Static observation; cost is bounded (3-second timeout) and bounded by H.8.4 safe-exec discipline (array-form arg passing).

### D.3 — `loadAllAdrs().filter(isActive)` duplicated across `cmdActive` + `cmdTouchedBy`
