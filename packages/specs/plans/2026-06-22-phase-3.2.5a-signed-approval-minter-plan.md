## PR-1 — what ships (this is the buildable unit)

### M1 — the body-threading seam extension (`packages/kernel/_lib/edge-attestation.js`)

`signRecordId(recordId, opts)` currently calls `signer(recordId)` (1-arg, line ~118). Extend to
`signRecordId(recordId, opts = {}, body)` -> `signer(recordId, body)`. The default closure (resolveSigner) accepts
+ IGNORES `body` (the env-PEM signer needs no recompute). Backward-compatible: every existing 2-arg call site is
unaffected (body=undefined). The injected broker signer (PR-2) USES body for the recompute-bind. The OUTPUT gate
(canonical-base64 + 64-byte) is unchanged.

### M2 — the approval becomes a SIGNED artifact (`packages/kernel/egress/approval.js`)

- `verifyApproval({ fileBytes, requestedHash, now, ttlMs, verifyKeyPem })` -> additionally require a valid `sig`:
  after the #273 body-rehash + TTL + nonce checks, `verifyRecordSig(body.hash, body.sig, { publicKeyPem:
  verifyKeyPem })` MUST be true (fail-closed: absent/invalid sig -> `{ ok:false, reason:'sig-invalid' }`). The
  signature is over the SAME `hash` the body re-derives (binds the sig to the exact emission). `key_id` is carried
  for the PR-2 key-set resolver (recorded, not yet branched on).
- `verifyKeyPem` is resolved by the STORE/caller from a PINNED location (PR-1: `opts`/`LOOM_BROKER_VERIFY_KEY`);
  absent -> fail-closed (no ambient default — mirror `loadPublicKey`).

### M3 — the mint signs (`packages/kernel/egress/approval-store.js`)

- `recordApproval(dir, draft, { now, nonce, selfUid, signFn })` -> compute `hash`; obtain `sig = signFn(hash,
  emissionAxiom(draft))` (the body threaded for PR-2's recompute-bind); REFUSE to write if `signFn` returns no
  canonical 64-byte sig (fail-closed — never write an unsigned approval once signing is required). Write
  `{ hash, emission, approvedAt, nonce, sig, key_id }` via the existing `wx` exclusive writer.
- `signFn` default = `(h, body) => signRecordId(h, { /* env/opts key */ }, body)` (Option-A env key — PR-2 injects
  the cross-uid broker signer). Tests inject an in-process keypair signer.

### M4 — the recompute-bind HELPER (reused, not re-implemented)

The broker (PR-2) recompute-binds via the EXISTING pure `computeEmissionHash(body)` (approval.js) — `computed ===
claimedHash` else refuse. PR-1 exports nothing new here; it just confirms `computeEmissionHash` is the shared
re-derivation both the gate and the (PR-2) broker call (DRY — the #273 verify-the-body discipline, now signed).

### gate wiring (`emit-pr.js`)

`readVerifiedApproval` / the emitPR 4th-AND already calls `verifyApproval`; thread `verifyKeyPem` (custody, via
opts/env) through. No new AND — the sig requirement lives INSIDE `verifyApproval` (a stronger approved-check).
armedEmit STILL throws; EC6 (no child_process) HOLDS.
