# v3.8a W4 — un-darken the verdict→E4→A6→E11 routine (the producer step + first real volume)

## Context

Every component of the advisory trust loop is LIVE: `recordVerdict`/`record-review` (the
W1 store + CLI, with batch + auto-enrich), the spawn-state enricher (agentId → kernel txid),
E4 reputation (projection + `materialize` + the A6 snapshot read), and E11 (the breaker's
DEFAULT `verdict-fail` source). The CONSUMER side is conventioned
(`agent-identity-reputation.md`: `reputation snapshot --personas` + breaker `check
--persona` before delegated spawns). But the loop is VOLUME-STARVED because the **producer
step is codified nowhere**: `grep recordVerdict packages/skills/rules/` → only the
reputation pattern doc; the per-wave workflow (`workflow.md`'s persona-selection block)
says NOTHING about recording the VALIDATE board's verdicts. Six lens boards ran in this
session alone; zero attestations were recorded. The ledger's E11 row says it honestly:
"trips once delegated builds accumulate fails (**0 at close**)".

The structural constraint (canonical — [[verdict-attestation-subject-is-builder-spawn]]):
the subject `agentId` is the DELEGATED, delta-bearing BUILDER spawn — only delegated builds
are legal subjects. Root-orchestrated builds (this arc's W1-W3) have NO legal subject and
must record NOTHING. So the un-darkening is two-part: **codify the producer step** (fires
exactly when a build was delegated), and **put the first REAL volume through the loop** by
dogfooding one genuinely useful delegated build end-to-end.
