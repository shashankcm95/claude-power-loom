## What ships (modules)

### M1 ‚Äî `packages/kernel/egress/approval.js` (PURE ‚Äî no I/O; mirrors manage-op-record.js)

- `emissionAxiom(draft)` -> the MINIMAL canonical binding set `{ repo: <.git-stripped, lowercased owner/name>,
  issueRef: Number, diff: <the scrubbed diff> }` (F4: NOT title/touched_paths ‚Äî derived).
- `computeEmissionHash(draft)` = `sha256(canonicalJsonSerialize(emissionAxiom(draft)))`. REUSE
  `kernel/_lib/canonical-json.js` (depth+node bounded ‚Äî INV). Mirrors the manage-promote
  `approvalAxiomHash = sha256(canonicalJsonSerialize(<approved>))` USER_INTENT_AXIOM.
- `verifyApproval({ fileBytes, requestedHash, now, ttlMs })` -> `{ ok, reason }` (PURE, fail-closed):
  parse JSON `{ hash, emission, approvedAt, nonce }`; require `computeEmissionHash(emission) === body.hash ===
  requestedHash` (#273 verify-the-BODY-not-the-key) AND `now - approvedAt <= ttlMs` (H4 TTL) AND a non-empty
  nonce. Any parse-fail / mismatch / stale -> `{ ok:false }`.

### M2 ‚Äî `packages/kernel/egress/approval-store.js` (fail-closed I/O over M1)

- `assertCustodyApprovalsDir(dir, selfUid)`: `lstatSync(dir)` must be a directory, NOT a symlink, owned by
  `selfUid` (skip uid on Windows `getuid===undefined`). Foreign/symlink/absent -> throw (fail-closed). (H3/F5)
- `readVerifiedApproval(dir, hash, { now, ttlMs, selfUid })` -> `{ ok, reason }`: assert the dir; open
  `path.join(dir, hash + '.approved')` with `O_RDONLY | O_NOFOLLOW` (refuses a symlinked FINAL component
  atomically ‚Äî no lstat->open TOCTOU); `fstatSync(fd)` must be a regular file owned by `selfUid` (reject dir
  [EISDIR-class] / foreign-uid); read the bytes via the fd; close; hand to `verifyApproval`. ANY error ->
  `{ ok:false }`.
- `recordApproval(dir, draft, { now, nonce, selfUid })` -> `{ hash, path }`: assert the dir; `hash =
  computeEmissionHash(draft)`; write `{ hash, emission: emissionAxiom(draft), approvedAt: now, nonce }` to
  `<hash>.approved` with `fs.writeFileSync(.., { flag: 'wx' })` (exclusive create -> EEXIST on a pre-seeded
  symlink/file; fail-closed; matches the reader's no-follow). (F3)
- `consumeApproval(dir, hash)` -> unlink `<hash>.approved` (one-shot; best-effort, errors swallowed ‚Äî a
  failed consume must not un-emit). (H4)

### M3 ‚Äî emitPR wiring (`packages/kernel/egress/emit-pr.js`)

- Build the draft with `draft.repo = <normalized canonical repo>` (H5 ‚Äî same field the axiom hashes + the
  ‚ë¢.2.5 target reads).
- After the draft, `approvalHash = computeEmissionHash(draft)`; include `approvalHash` in EVERY non-error return.
- Gate (step 6): `if (disposition.mode === 'live' && token && !killswitchOn)` THEN
  `const appr = readVerifiedApproval(opts.custodyApprovalsDir, approvalHash, { now, ttlMs, selfUid })`;
  if `!appr.ok` -> `return { ok:true, emitted:false, reason:'awaiting-approval', approvalHash, draft }`
  (the EXPECTED pending state, NOT an error). Else -> the emit-then-record path:
  `const res = armedEmitFn({...})` (I2 ‚Äî `opts.armedEmitFn` default the module `armedEmit`, which THROWS this
  wave) THEN on success `recordEmit` + `recordEmitted` + `consumeApproval` + `return { ok:true, emitted:true,
  pr:res, approvalHash, draft }`. A throw -> the outer catch -> fail-closed, cap+ledger+approval UNCHANGED.
- `DISPOSITION_KEYS` += the full approval vocabulary (H7). `ttlMs`/`now`/`selfUid` come from opts (custody),
  never `data`.

## Forward-Contract ‚Äî what ‚ë¢.2.5 MUST honor for THIS gate to be sound (the load-bearing section)

1. **Bind-input == emit-input (F1/H1/honesty-H1).** ‚ë¢.2.5 emits EXACTLY the approved scrubbed draft ‚Äî never
   re-scrubs, never emits the raw diff, never emits blobs that diverge from the approved bytes. At emit time it
   RE-DERIVES `computeEmissionHash` over the ACTUAL emission payload and REFUSES on mismatch.
2. **Authenticated minter before ARMING (H2/honesty-H3).** Do NOT arm until an approval has an authenticated
   minter the actor / any same-uid host process cannot forge ‚Äî a detached signature over the canonical emission
   bytes verified against an operator public key the kernel pins, OR a kernel-owned writer the caller cannot
   invoke. Until then the gate is INTEGRITY-only and must not authorize a live emission.
3. **Full envelope (honesty-H2).** ‚ë¢.2.5 adds base-branch / head-ref / body to `emissionAxiom` (the gh-REST PR
   envelope) and re-gates; this wave binds only `{repo, issueRef, diff}`.
4. **The approve-CLI (H6).** ‚ë¢.2.5 ships the human-gate CLI: read the pending artifact ONCE, hash those exact
   bytes, DISPLAY the digest + byte-count + "you are approving N bytes you have not read", and on `--confirm
   <hash>` store the SAME in-memory bytes (never re-read); refuse an actor-writable artifact path.
5. **Blob-source** ‚Äî the original ‚ë¢.2.5 design Q (full-file-contents vs apply-patch-from-the-scrubbed-diff);
   #1 makes apply-patch-from-the-approved-scrubbed-draft the natural resolution.
6. **reserve->rollback** for cap/ledger (vs this wave's emit-then-record) IF the emit-success-then-record-fail
   duplicate risk is judged real once armedEmit can succeed.

## Trust model (honest ‚Äî integrity, NOT provenance, this wave)

`verifyApproval` + content-verify-on-read closes INTEGRITY (the approval is self-consistent + bound to the
draft). Custody-uid-ownership + container isolation closes the SANDBOXED-ACTOR forge (the actor on a different
uid cannot write a host-uid approval). It does NOT close a same-uid HOST forge: `computeEmissionHash` is
exported, so any host-uid process can mint a byte-valid approval. Per #273 / security.md, a trust input that
GATES AN ACTION needs an AUTHENTICATED minter ‚Äî so signed approvals are a HARD ‚ë¢.2.5 ARMING precondition
(Forward-Contract #2). This wave the gate is INERT (armedEmit throws), so the residual is non-exploitable now.
