## Design

`packages/runtime/test-runners/` — a registry + adapters. Two **distinct** enums (do not conflate):

- **`RUNNER_KINDS`** (R12-internal): `['node','jest','vitest','pytest']` — WHICH test runner. Frozen.
- **`verifier_kind`** (ADR-0015, R11's gate-kind axis): R12 owns the `test-run` + `registry-lookup` members. R12 does NOT add members — it confirms these two are right and removes "provisional."

**Files:**

1. **`registry.js`** — the adapter registry (the surface R9 #4 + R11 consume):
   - `registerAdapter(adapter)` / built-ins registered at module load (Open/Closed via a `Map`, mirroring the `validators={}` idiom).
   - `getAdapter(kind)` → adapter | null; `listRunnerKinds()` → frozen array.
   - `resolveAdapter(ctx)` → the first registered adapter whose `appliesTo(ctx)` is true, else null (detection).
   - `isVerificationSupported(ctx)` → boolean — **the R9 #4 `validation-supported` lookup** (does ANY adapter apply?).
   - An **adapter contract** (duck-typed, ISP-narrow): `{ kind, appliesTo(ctx)->bool, buildCommand(ctx)->{cmd,args,cwd}, run(ctx)->{passed,exitCode,stdout,stderr,timedOut} }`.
2. **`node-runner.js`** — the REQUIRED, fully-functional adapter:
   - `kind:'node'`; `appliesTo(ctx)` true when `ctx.testFile` ends `.test.js` (the substrate convention) and node is the declared runner.
   - `run(ctx)` → `execFileSync('node', [testFile], {cwd, timeout, encoding})`; **NO shell** (execFile + args array — no command injection); maps a non-zero exit / throw to `{passed:false,...}` reading `err.status`/`err.stdout`/`err.stderr` (the execFileSync gotcha); a timeout → `{timedOut:true,passed:false}`.
   - **Security (hacker surface):** the test path is validated **before** exec — reject absolute-escape / `..`-traversal outside `ctx.cwd` via `path-canonicalize` (`checkWithinRoot`); `cwd` must be a real dir; a hard `timeout` bounds a runaway/hanging test. Pure/stateless (no run-state writes).
3. **`jest.js` / `vitest.js` / `pytest.js`** — thin **shims** (detect + command-build, NOT executed — the substrate has no such project; KISS/YAGNI floor):
   - `kind`; `appliesTo(ctx)` detects an ecosystem marker (jest/vitest config; `pytest.ini`/`conftest.py`/`pyproject [tool.pytest]`); `buildCommand(ctx)` → the argv. `run(ctx)` MAY throw `not-exercised-in-substrate` (honest) or delegate to a generic spawn — **decide in VERIFY** (Q3 below).
4. **`index.js`** — barrel: exports the registry with built-ins registered + the adapter-kind constants.

**CI change** (`.github/workflows/ci.yml`, the `runtime-contracts-tests` job): widen find root `tests/unit/runtime/contracts` → `tests/unit/runtime` (recursive; symmetric with the kernel job at :145), rename job/step "Runtime contracts tests" → "Runtime tests", update the PR-1-specific comment to "all runtime unit-test dirs (contracts + test-runners + …)". **Fixture-discovery trap (designed-around):** the node-runner unit test must run the adapter against real script files. If those fixtures are named `*.test.js`, the **widened CI glob `find tests/unit/runtime -name '*.test.js'` would ALSO run them directly as suites** — a deliberately-failing fixture would then fail CI. **Mitigation:** name run-fixtures `*.fixture.js` (e.g. `fixtures/pass.fixture.js`, `fixtures/fail.fixture.js`); `appliesTo`/detection tests use string paths (no execution). The glob never matches `*.fixture.js`.

## Principle Audit

- **SRP** — each adapter owns one runner; the registry owns resolution/lookup; R12 owns "can this leaf be verified" — NOT "verify it" (that's R11). One reason to change per file.
- **OCP** — new runners register into the `Map` without editing existing adapters (mirrors the `validators={}` registry); R9/R11 depend on the registry interface, not concrete adapters (DIP).
- **ISP** — the adapter contract is narrow (`appliesTo`/`buildCommand`/`run`); R9 #4 needs only `isVerificationSupported` and is not forced to depend on `run`.
- **KISS/YAGNI** — node-runner is fully built (dogfood need); jest/vitest/pytest are detect+command shims, NOT live executors (no such project exists yet) — build the executor when a consumer appears. Don't gold-plate.
- **DRY** — reuse `path-canonicalize` (path guard), the house execFile idiom, the test-harness idiom; do NOT re-implement a runner framework.
- **Immutability** — `RUNNER_KINDS` frozen; `run()` returns a fresh result object; ctx is read, never mutated.

### Residual risks TRACKED to the ContainerAdapter-tier (v3.5; user review 2026-06-03)

These are the R12-layer best-effort defenses whose COMPLETE closure is the ContainerAdapter's job — recorded here so that work inherits them as explicit acceptance criteria:

1. **(H1) Host-path traversal must be eliminated by true sandboxing.** R12's `lstat`-reject narrows the TOCTOU symlink window to sub-microsecond but does not close it, and an *absolute-path write* by the test still escapes `cwd` entirely (the standing `p-writescope` reality — `cwd` is not a chroot). When R12 execution moves into the ContainerAdapter, the execution context MUST be a real filesystem sandbox (namespace/chroot/container) so neither the residual race NOR absolute-path escape can reach host paths. Acceptance: a test that `open('/etc/...')` or symlink-swaps cannot read/write outside the sandboxed root.
2. **(C2) No indefinite hang — VERIFIED at the R12 layer; carry the DoS-bound to the container.** Firsthand-probed (and now a regression test, `node-runner.test.js` "C2"): a pipe-pressured child that fills the OS pipe and hangs is bounded — `spawnSync` drains the pipe concurrently AND the wall-clock `timeout`→SIGKILL (or `maxBuffer`→ENOBUFS→SIGKILL) fires regardless; there is no deadlock path. The ContainerAdapter still owns the HARD output-DoS bound (a flood faster than the drain) + process-GROUP reaping of detached grandchildren that outlive SIGKILL. Acceptance: an output flood and a fork-bomb are both bounded by the container's resource limits, not just the per-process timeout.
