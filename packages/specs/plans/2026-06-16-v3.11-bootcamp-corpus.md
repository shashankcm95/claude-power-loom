# Phase 1 — the N>=20 collision corpus (the discrimination-measurement precondition)

USER chose the FULL bootcamp re-run. Phase 0 (real-leg no-hang) + Phase 2a (the trap-seam pipeline on
real data — verify-the-failure + a real insightful `claude` lesson) are GREEN. The remaining blocker to
the discrimination measurement (Phase 3) is the CORPUS: today there is exactly **1 real issue**
(more-itertools; the seed-manifest's 3 are synthetic unit fixtures). The measurement needs **N>=20 real
lesson-bearing nodes with same-signature collisions** against the 24-cell frozen floor.

## Honest framing (set once)

- **OQ-NS-6:** even a completed discrimination measurement NARROWS only (diagnostic) — it never HARDENS
  trust. This corpus is a measurement instrument, not a trust gate.
- **Multi-session:** sourcing + sandbox-verifying 20+ real recent OSS bugfixes is the v3.9 W0 effort at
  5x scale. Realistic cadence: a few verified issues per session. Checkpoint per issue; never block on a
  hang (every clone/sandbox/claude leg is timeout-wrapped + run in the background).
- **Anti-contamination:** issues must be resolved AFTER the model cutoff (Jan 2026) -> `contamination_tier
  = clean-pending-probe`. The recall graph only admits CLEAN_FOR_RETRIEVAL tiers.

## The per-issue gate (quality bar — every issue clears this before it counts)

A staged record `{ id, repo, base_sha, problem_statement, accepted_diff, test_patch, fail_to_pass[],
pass_to_pass[], resolved_at, contamination_tier }` is VERIFIED iff, in the sandbox:
1. **fail-before:** clone@base_sha + apply `test_patch` (no fix) -> `fail_to_pass` tests FAIL.
2. **pass-after:** + apply `accepted_diff` -> `fail_to_pass` PASS and `pass_to_pass` stay PASS.
Both legs via `makeBehavioralFn` + the sandbox-exec backend (the same path Phase 2a proved). A record
that can't be made to fail-before/pass-after (deps, test-discovery, flaky) is REJECTED, not forced.

## Collision strategy (so the measurement is possible, not just N>=20)

Source in CLASS CLUSTERS so >=2 issues plausibly land in one (trigger|gotcha|corrective) cell. Anchor
cluster = the more-itertools lesson `api-shape|unguarded-edge-case|handle-edge-explicitly`; seed more
edge-case / wrong-guard / off-by-one bugs to collide. Target >=3 clusters of >=3 each. The derive leg
assigns the actual signature at capture (Phase 2b); under-separation (DEF-3 raw diagnostic) is the
expected near-term signal and is itself informative.

## Harness (this branch)

- `_spike/corpus-build/verify-record.js` — the per-issue gate (fail-before/pass-after in the sandbox;
  watchdog backstop; background-run). REUSABLE per issue.
- `_spike/corpus-build/staged/<id>.json` — one staged record per candidate (pre-verify).
- `_spike/corpus-build/bootcamp-manifest.json` — the accreting VERIFIED corpus (the Phase 2b input).
- Sourcing: `gh search prs --merged --created '>2026-01-01' --language Python` -> `gh pr diff` +
  `gh pr view --json baseRefOid,title,mergedAt,body` -> split source/test files -> extract candidate
  `fail_to_pass` (added `def test_*` in the test files) -> stage -> verify.

## Cadence / exit

Accrete to N>=20 verified WITH >=1 real same-signature collision cluster confirmed at Phase 2b capture.
Each session: source a cluster, verify, checkpoint the manifest, report `verified/20`. Phase 2b (capture)
+ Phase 3 (measure) + Phase 4 (phase-close) follow once the corpus clears the floor.
