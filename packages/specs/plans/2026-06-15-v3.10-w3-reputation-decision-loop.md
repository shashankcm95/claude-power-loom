## The design — a PURE advisory consumer (`reputation-gate.js`)

`recommendNarrowing(candidates, reputation, breakerOf, opts)` -> a per-candidate ADVISORY (never a hard gate).
**VERIFY load-bearing fold: the inputs are THREE INDEPENDENT axes — evaluated SEPARATELY, fail-safe to
NO-SIGNAL per-axis, combined by MOST-restrictive. NOT a single short-circuit ladder** (a ladder let one axis's
"insufficient" swallow another's real signal: a thin persona under a tripped breaker proceeded; a starved
source neutralized the whole advisory).

- Inputs: `candidates` (persona keys), `reputation` (a `projectReputation` output), `breakerOf` (candidate -> a
  breaker `evaluate()` decision OBJECT or null — it **MUST NOT throw**: the HARNESS wraps `evaluate()`, the
  consumer NEVER passes `requireLive`, and a defensive try/catch treats a throw as null), `opts {minEvidence,
  passFloor, now}`.
- **Structural MIS-WIRE guard FIRST (NOT cryptographic auth — VALIDATE-hacker MED honesty):** if
  `reputation.source !== 'verdict-attestation'` (the EXPORTED `SOURCE` constant) OR `reputation.personas` is not
  an array -> EVERY candidate `proceed`, `reason:'unauthenticated-lane'`. This catches an ACCIDENTAL mirror
  pass-through (a careless caller wiring the wrong object); it does NOT cryptographically prove the rows came
  from the kernel-attested store (a caller controls its own input -> `source` is self-asserted). The REAL
  authentication is UPSTREAM (the verdict-attestation store the harness reads). Defense-in-depth FAILS TOWARD
  NARROWING: a malformed/inconsistent row (`total !== sum(by_verdict)`, non-int/negative total, a DUPLICATE
  persona key) -> `down-weight` (`unreadable-distribution`/`duplicate-row`), never laundered to proceed.
- Index rows by a **`Map`** (persona -> row), never a plain object (a `__proto__` persona key would launder —
  hacker MED; mirrors `circuit-breaker/project.js:305` + `reputation/project.js:79`).
- **Per candidate, evaluate each axis INDEPENDENTLY:**
  - **AXIS A — reputation distribution (the down-weight axis):** a signal ONLY when `row` exists AND `total >=
    minEvidence`. Then if `by_verdict` is readable (`pass`/`partial`/`fail` are integers, `total>0`):
    `pass_ratio = pass/total` (**`partial` counts as NON-passing** — pinned, architect LOW) -> `< passFloor`
    => `down-weight`. An UNREADABLE `by_verdict` that ALREADY cleared `minEvidence` => `down-weight` reason
    `unreadable-distribution` (NEVER proceed — a stripped/NaN distribution must not launder a real-total
    persona; hacker HIGH). Thin (`!row` or `total < minEvidence`) => NO down-weight signal (but this does NOT
    suppress AXIS B).
  - **AXIS B — breaker (the reroute axis), INDEPENDENT of reputation volume:** a signal ONLY when `brk` exists
    AND `brk.source_starved !== true` (a starved source contributes NOTHING — but does NOT suppress AXIS A;
    hacker HIGH: else one env var neutralizes the whole advisory). Then `brk.tripped ||
    brk.global_tripped || brk.persona_tripped` => `reroute`.
  - **COMBINE by MOST-restrictive:** `reroute` > `down-weight` > `proceed`. (A thin-reputation persona under a
    TRIPPED live breaker => `reroute`, NOT proceed — the architect-HIGH ordering fix.)
- Output: `{ candidate, recommendation: 'proceed'|'down-weight'|'reroute', reason, evidence:{ total, pass_ratio,
  readable, breaker_tripped, source_starved } }`. **No `exclude`** — the worst advisory is `reroute` (shadow +
  the "NOT a quality score" self-label). PURE (no I/O); the harness reads the real stores and pins `breakerOf`
  to the LIVE default source `verdict-fail` (non-starved; architect MED — the SAME kernel-attested store feeds
  BOTH axes, so both read the authenticated lane; the W2 entry criterion holds end-to-end).
