# ③.2.1a — Containment + grade-integrity (build wave)

## Frame (sharpened by the VERIFY board)

**Grade-integrity = "the automated grade faithfully reflects the sealed-test outcome; it cannot be FORGED to report
PASS when the sealed tests, run under a pinned harness, actually reported FAIL."** It is NOT "the candidate really
fixed the bug" — that is *proof-of-fix*, which is (a) visible in the human-reviewed diff under Path-1 and (b) the
fundamental limit OQ-NS-6 already names (a backtest grade only NARROWS; only a world-anchored merge HARDENS). This
boundary is load-bearing: it tells us which vectors the gate MUST close (grade-channel forgery) vs which are the
human-review/OQ-NS-6 residual (a source edit that genuinely makes the sealed test pass).

Derives from the master plan's ③.2.1a (`2026-06-19-phase-3.2-live-external-pr-beta-plan.md:348`). Trust-independent
(Path-1): a forged grade makes the HUMAN push a bad PR regardless of any lab-weight gating, so this is a hard
first-PR gate even though every lab weight stays SHADOW. **③.2.2's live DRAFT loop drives `real-solve.js` over
stranger repos — this wave must hold before that.**

## The forgery vectors (what the gate MUST close) vs the residual

The board built live pytest exploits. Grade-channel **forgeries** (gate MUST close):
1. **Test-tree mutation** — candidate edits a graded test to assert True. (tree-hash signal exists; not gated on `mapBehavioral`.)
2. **Sentinel forgery** — candidate `atexit`/late-write of a second `__LOOM_TEST_RESULT__{...:'pass'}` line (last-line-wins). **Bypasses all original closes.**
3. **Plugin/conftest hijack** — a `pytest11` entry-point plugin (incl. a committed `*.egg-info/entry_points.txt` on `sys.path`) or an in-tree `conftest.py` monkeypatches pytest's report. (autoload-disable + diff-scope + tree-restore.)
4. **`.gitattributes` smuggle** — candidate ships a `.gitattributes` filter/eol/ident attr applied under non-pristine grader config.

**Residual (OUT of grade-integrity scope; documented, NOT closed by hashing):** a candidate edit to an ordinary
source module that the sealed test legitimately imports, which makes the sealed test *genuinely* pass (the grade is
HONEST — the tests really passed). This is proof-of-fix, bounded by Path-1 human diff-review + OQ-NS-6. The narrow
sub-residual the board named (a BASE-repo `conftest.py` that already imports a mutable non-test helper the candidate
then edits) is caught by diff visibility under human review; we document it explicitly rather than over-claim a
hashing close.
