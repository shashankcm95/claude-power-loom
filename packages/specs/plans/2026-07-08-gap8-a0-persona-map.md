## Security invariants

1. **owns-basis re-derive on read + fail-soft** ([[scars-toolkit#scar-41]], F6) — node_id = hash(repo[folded], pr_number) re-derived + rejected on mismatch; content_hash seals the persona value (a tamper is rejected — tamper-EVIDENT). Reads are **fail-soft: null on any defect, never throw** — a poisoned map record relocates that PR to the sentinel plane (under-attribution, baseline-safe), it does NOT abort the breaker projection (a per-record DoS lever, closed).
2. **consumer groups by the store's full identity** ([[scars-toolkit#scar-42]]) — the join keys by the FULL `(repo[folded], pr_number)`, never a sub-field; `new Map()` + composite string keys (the store admits a `__proto__` repo segment). The **relocate defense is the store's one-record-per-PR + `{flag:'wx'}` first-write-wins + explicit-field conflict-reject** — the `(persona,id)` `dedupBySubject` port is DROPPED (F2: vacuous here — `blockedPrs` already collapses to one denial per `(repo,pr)`, and a sub-field key would UNDER-count).
3. **persona value validated by roster membership** (F5) — `canonicalPersonaKey(persona) === persona` rejects the `kernel:`-prefixed IDOR class (colon fails `BARE_SHAPE`) AND the reserved `changes-requested` sentinel (not an `agents/*.md` member) at the store boundary, consistent with the upstream `classifyIssue` D2 gate.
4. **self-asserted persona value = the arming-time TRUST residual** — the persona is `classifyIssue` output, not authenticated; content-addressing makes it tamper-EVIDENT, NOT provenance-authentic. Two faces, both NAMED + arming-deferred: (a) an attacker steers `classifyIssue` (over stranger issue-text) to a CHOSEN persona (the byzantine-classifier surface); (b) a same-uid **first-write-wins pre-seed** pins an attacker-chosen persona for a PR, and the conflict-reject then makes the honest producer's later write REJECT (an un-correctable pin; operator repair = delete-and-rewrite). Both bounded by halt-only NARROWS (§0a.3.1 — over/under-halt, never grants) + the C1 insider `author_association` gate upstream; the authenticated minter (PACT's cross-uid broker, wired at arming) closes the provenance residual. SHADOW-tolerable.
5. **dam-preserving** — the breaker joins on `(repo,pr_number)` from records it already reads; ZERO kernel join-key read (the 2-reader allowlist is untouched). The persona-map is SHADOW (its only reader is the starved breaker source).
