# HT.2.2 — `parseFrontmatter` YAML 1.2 inline-comment strip

## Context

Per HT.2 master plan §"Sub-phase 2 — HT.2.2 parser-discipline-edge sweep" + drift-note 73 (HT.1.12 surface): `scripts/agent-team/_lib/frontmatter.js parseFrontmatter` does NOT strip YAML inline `#` comments per YAML 1.2 spec. Surfaced when an HT.1.12 frontmatter authoring attempt used inline comment annotation (`  - architecture/ai-systems/agent-design  # planned — not yet authored`) and the roundtrip test contaminated the ref-string. HT.1.12 pivoted to body-section migration; the parser limitation was captured as drift-note 73 for HT.2.

YAML 1.2 spec §9.1.6: "Comments are always indented and ignored... A comment may appear at the end of any line, separated from any preceding non-comment content by white space." Behavior to implement:
- `#` preceded by whitespace (or at trimmed-value start) → comment start; truncate
- `#` inside single-quoted or double-quoted scalar → literal (preserved)
- `#` not preceded by whitespace (bare scalar like `a#b`) → literal (preserved)
- Trailing whitespace after comment removal stripped

**Why now (HT.2.2 timing)**: parser-discipline-edge is HT.2's second sub-phase per master plan; mechanical YAML 1.2 spec compliance with chaos-test mitigation against ~78 existing frontmatter sources. No new institutional commitment.

**Why sub-plan-only methodology** (per master plan methodology table line 328): "Mechanical YAML 1.2 spec compliance; no fresh design surface; chaos-test mitigates regression risk." No option-axis fork (the algorithm is unambiguous per YAML 1.2 spec); no schema change (parser BEHAVIOR change, not API surface change); no institutional discipline encoding. Matches HT.1.2/HT.1.4/HT.1.6/HT.1.8/HT.1.9 sub-plan-only precedents. HIGH-class risk surface: regression on existing frontmatter — mitigated by chaos-test across all 4 frontmatter source cohorts.

## Implementation strategy

### Helper function: `_stripInlineComment(val)`

```javascript
/**
 * Strip YAML 1.2 inline comment from a value string.
 *
 * Per YAML 1.2 spec §9.1.6, '#' starts a comment when preceded by
 * whitespace (or at start of the trimmed value — the original
 * leading whitespace was consumed by the regex's \s* group). Inside
 * single-quoted or double-quoted scalars, '#' is literal.
 *
 * Edge cases:
 *   - val starts with '#' → entire value is comment; return ''
 *   - '#' not preceded by whitespace (bare scalar like 'a#b') → literal
 *   - '#' inside "..." or '...' → literal
 *   - Trailing whitespace after comment removal stripped
 *   - Comments inside inline arrays ([a, b # c]) follow spec: '#' preceded
 *     by whitespace truncates at that point. Document gotcha; quoted form
 *     ([a, "b # c"]) preserves the '#' as literal.
 *
 * @param {string} val - trimmed value string (post-`m[2].trim()`)
 * @returns {string} value with inline comment + trailing whitespace stripped
 */
function _stripInlineComment(val) {
  if (val.startsWith('#')) return '';
  let inSingle = false;
  let inDouble = false;
  for (let i = 0; i < val.length; i++) {
    const ch = val[i];
    if (inDouble) {
      if (ch === '"') inDouble = false;
      continue;
    }
    if (inSingle) {
      if (ch === "'") inSingle = false;
      continue;
    }
    if (ch === '"') { inDouble = true; continue; }
    if (ch === "'") { inSingle = true; continue; }
    if (ch === '#' && /\s/.test(val[i - 1])) {
      return val.slice(0, i).replace(/\s+$/, '');
    }
  }
  return val;
}
```

// and quote-strip. Comments at the value position (`key: # foo`) collapse to
// empty value → block-list start. Comments inside quoted scalars are

**Site 2: Block-list item extraction (line 45)**

```javascript
if (line.match(/^\s+- /)) {
  if (currentListKey) {
    let item = line.replace(/^\s+- /, '').trim();
    // YAML 1.2: strip inline '#' comment (quote-aware) BEFORE quote-strip
    // so quote-protected '#' survives (e.g. - "value with # inside" → 'value with # inside').
    item = _stripInlineComment(item);
    item = item.replace(/^["']|["']$/g, '');
    if (!Array.isArray(fm[currentListKey])) fm[currentListKey] = [];
    fm[currentListKey].push(item);
  }
  continue;
}
```

### Edge cases handled vs deferred

**Handled**:
1. Bare scalar with trailing `# comment` → strip
2. Inline array with trailing `# comment` AFTER closing `]` → strip preserves array shape
3. `#` inside double-quoted scalar → preserve (e.g., `key: "value with # inside"`)
4. `#` inside single-quoted scalar → preserve
5. Block-list item with trailing `# comment` → strip
6. `key: # comment` (entire value is comment) → val='' → triggers existing block-list-start branch
7. `#` not preceded by whitespace (`key: a#b`) → preserved as literal `a#b`
8. Comment-free frontmatter unchanged (regression guard)

**Deferred / documented as gotcha**:
- Inline-array elements containing unquoted `# comment`: per YAML 1.2 spec, `# ` truncates the array prematurely. Quoted form (`[a, "b # c", d]`) is the spec-compliant way to embed `#`. Documented in helper-function JSDoc.
- Block-folded scalars (`key: >` / `key: |`) — not parsed by current parseFrontmatter; out of scope.
- Multi-line escaped strings — current parseFrontmatter doesn't support; out of scope.
