# ③.2.1b — The PR-egress kernel (the single `emitPR` chokepoint)

The fail-closed barrier between "draft artifact" and "anything leaves the box." Needed only before anything LEAVES
(③.2.3), so it is built + hacker-re-probed now WITHOUT a real emission — `emitPR` defaults to **dry-run + killswitch-on**
(no resolvable token), so the whole wave is testable with zero bytes ever leaving. Kernel/security tier → full 3-lens
VERIFY + VALIDATE.

## Frame — custody, not interception (ADR-0012)

The killswitch must REMOVE the emission **capability**, not intercept a command. A `PreToolUse:Bash` hook on
`gh`/`git push` is advisory/evadable for a direct subprocess (the lesson `network-egress-audit.js` already encodes:
"pattern-DENY is evadable theater — DETECT + advise, never block"). So the gate is **architectural**: a single
in-process `emitPR(...)` is the SOLE holder of the GitHub token and the SOLE place the network is touched. With the
killswitch set there is **no resolvable token** → the capability is *gone*, not *blocked*. "Fail-closed by
construction."

## Design (VERIFY-folded) — `packages/kernel/egress/emit-pr.js`

**The reshape:** the live emission is **ABSENT this wave** — PR-A builds the GATE (custody env-sanitization,
killswitch, disposition, validation, egress-path-scope, serialization), and the actual PR-creation network call is a seam
deferred to ③.2.3. So "cannot emit" is true *by construction* (no emission code exists), the strongest guarantee
(honesty F-scope). The env-sanitization IS testable without emitting — a read-only `gh auth status` in the scrubbed
env asserts the ambient capability is gone.

1. **`emitPR(data, { now })` — untrusted DATA separated from trusted POLICY (architect F2 / hacker F-input).** `data`
   carries ONLY the bounded sandboxed candidate diff + the issue/repo ref (actor-influenceable). **Disposition + token
   come from CUSTODY, never from `data`.** If `data` contains ANY disposition-shaped key → **fail-closed reject** (loud,
   not silent-ignore); no `{...defaults, ...data}` spread / `??` gap-fill (the #273 exact-set lesson). Kernel-tier
   (node core + `kernel/_lib` only; the diff is data, so kernel imports nothing).
2. **Env-sanitization IS the killswitch (architect+hacker CRITICAL).** "No token in my custody path" is NOT
   capability-removal — `gh`/`git` resolve ambient creds (`GITHUB_TOKEN`/`GH_TOKEN`, `gh` keychain + `~/.config/gh/hosts.yml`,
   `git credential.helper`). `buildEmitEnv()` constructs the subprocess env FROM SCRATCH (allowlist, not inherit): drop
   `GITHUB_TOKEN`/`GH_TOKEN`/`GH_*`/`GIT_ASKPASS`; `GH_CONFIG_DIR` → an empty custody-owned dir (no inherited
   hosts.yml/keychain); `git -c credential.helper=` (disable ALL helpers) + `GIT_TERMINAL_PROMPT=0`. The token reaches
   the tool ONLY via the custody injection. **Killswitch ON ⇒ custody returns null ⇒ token never injected ⇒ the
   sanitized subprocess CANNOT authenticate even though the host is `gh auth`'d.** Reuse the `buildGitEnv()` scrub list
   as the precedent.
3. **Build the PR via the gh REST API from the diff-as-DATA — NEVER a `git push` from the candidate clone (hacker
   CRITICAL).** A push from the attacker-influenced `.git` runs pre-push hooks / `credential.helper` / `insteadOf` /
   submodules. The emission (deferred to ③.2.3) constructs the PR from the bounded diff via the REST API
   (blob→tree→commit→ref→pull) in the sanitized env — no local push, no candidate `.git` ever executes.
4. **Input-shape validation BEFORE any field reaches argv (hacker F-input).** `repo` must match
   `^[A-Za-z0-9_.-]+/[A-Za-z0-9_.-]+$` AND its host resolve to the github.com allowlist (reuse `resolveHostAllowlist` /
   `DEFAULT_REPO_HOST_ALLOWLIST` from `_clone-lifecycle`); issue/branch refs shape-validated; reject (fail-closed) before emit.
5. **Egress-time diff path-scope (hacker F-egress-scope) — distinct from the grading-time scope.** Reject (fail-closed)
   a candidate diff touching `.github/`, `.git*` (`.gitmodules`/`.gitattributes`/`.gitignore`), CI/workflow files, or any
   path outside an allowlisted code subtree, BEFORE building the PR.
6. **Killswitch re-check + token re-resolve TOGETHER, under one held lock; fail-closed everywhere (architect+hacker
   MEDIUMs).** `LOOM_BETA_KILLSWITCH` is FILE-backed (re-stat under lock). Serialize with `withLockSoft`; a
   lock-unavailable acquisition → **refuse the emit** (fail-closed, zero bytes), NOT k13 age-reap-admit; no
   `process.exit`. Any internal error (parse/validate/scrub/lock) → **zero bytes leave** (the catch-all never falls
   through to a real call). **Honest residual:** the re-check honors any killswitch flip that COMMITS before the network
   syscall; the in-flight-syscall window is irreducible (an irreversible egress is not a two-phase transaction) — named, not hidden.
7. **Sole-chokepoint = a LINT + a CUSTODY env-isolation TEST (honesty F-EC2).** The lint asserts no production module
   (outside `emitPR`) spawns `gh`/`git push` or reads the token, matching the CAPABILITY not the string, with an
   enumerated `_spike/` dev-only allowlist (asserted unreachable from production entrypoints). The custody test —
   **the fail-closed control for the actor/clone boundary the lint can't see** — asserts the token/credential is absent
   from the actor/grader/clone env.
8. **Secret-scrub = PR-B, strictly defense-in-depth (architect+hacker LOW).** Over the bounded diff only; a clean scrub
   NEVER unlocks a real push (the killswitch + dry-run default + human gate are the only unlock). "0 leaks on fixtures"
   proves only ANTICIPATED classes — CUSTODY is what HARDENS.

## The split

- **PR-A (custody GATE):** `data`/policy separation + disposition deny-by-default + `buildEmitEnv()` env-sanitization
  (the killswitch) + input-shape validation + egress-time path-scope + the file-backed killswitch + `withLockSoft`
  fail-closed serialization + fail-closed-everywhere + the sole-chokepoint lint + the custody env-isolation test. The
  REST-emission call is a clearly-marked seam that THROWS `not-armed-until-③.2.3` (no live network code).
- **PR-B (scrub + policy):** the secret-scrub (split-secret / git-object fixtures), a per-window cap, 429 backpressure,
  and the etiquette gate.

(③.2.3 arms the REST emission + is gated on BOTH PRs; nothing emits before then by construction.)

## Exit criteria (each proven INTERNALLY — the live emission is ABSENT this wave)

- **EC1b.1 [PR-A]** — **env-sanitization removes the ambient capability:** with the killswitch ON (custody null), a
  read-only `gh auth status` run in `buildEmitEnv()`'s scrubbed env reports NOT-authenticated **even though the host is
  `gh auth`'d** (proves the capability is gone, not just my-handle-gone). `buildEmitEnv()` is also unit-asserted to drop
  `GITHUB_TOKEN`/`GH_TOKEN`/`GH_*` + set `GH_CONFIG_DIR`/`credential.helper=`/`GIT_TERMINAL_PROMPT=0`.
- **EC1b.2 [PR-A]** — split honestly: (a) a LINT proves no production module reads the token or spawns `gh`/`git push`
  outside `emitPR`; (b) a CUSTODY env-isolation test proves the token is absent from the actor/grader/clone env — and
  the EC states (b), not the lint, is the fail-closed control for the actor/clone boundary.
- **EC1b.3 [PR-A]** — disposition is computed SOLELY from custody; a poisoned-`data` fixture (a disposition-shaped key)
  → **fail-closed reject (error)**, never silent-ignore, never a flip to real/live.
- **EC1b.4 [PR-A]** — input-shape validation rejects a non-`owner/repo` / non-github.com / argv-injection ref; the
  egress-time path-scope rejects a `.github/` / `.git*` / CI-file diff; any internal error → zero bytes / no seam call.
- **EC1b.5 [PR-A]** — the REST-emission seam THROWS `not-armed` (no live network this wave — the construction guarantee).
- **EC1b.6 [PR-B]** — the scrub catches the base64/high-entropy/SPLIT-secret/git-object FIXTURES (anticipated instances,
  per Design 8); the per-window cap + 429 backpressure halt on threshold; the etiquette gate rejects a 2nd PR/issue.

## VALIDATE result — PR-A (2026-06-21 board, folded)

3-lens on the BUILT gate. **Core custody design HELD under every probe** — the hacker's 40+ live probes
confirmed `buildEmitEnv` drops EVERY credential channel (GH_TOKEN/GITHUB_TOKEN/GH_ENTERPRISE_TOKEN/
GIT_ASKPASS/SSH_AUTH_SOCK/GIT_CONFIG_*/proxies/LD_PRELOAD), `emitted:true` is structurally impossible,
`armedEmit` fail-closes on every live-disposition path, and disposition/token come SOLELY from custody
(data-side keys inert). Honesty confirmed "cannot emit" is true by construction (no child_process/http/
fetch in the diff). Folded:
- **HIGH (blocking) — egress case-bypass:** `isEgressDeniedPath` was case-sensitive for `.github`/`.git`
  (`.GITHUB/`, `.GIT/config` bypassed end-to-end). Now `/i` on all checks + a `"`-quote reject (the
  c-quoting bypass) + a control-char reject (NUL/newline paths) + a `MAX_DIFF_BYTES` bound. Regression-tested.
- **argv-injection (LOW):** the repo validator now rejects a dash-leading SECOND segment (`owner/--upload-file`).
- **deny-list robustness (LOW):** `assertDataIsPolicyFree` is now CASE-FOLDED over all `data` keys (+ the
  prototype keys) so `Live`/`DRY_RUN`/`__proto__` (JSON-parsed) can't slip — though structurally inert
  (policy is read only from custody).
- **lint breadth (MEDIUM):** the sole-chokepoint CAP broadened to spawn/exec/execSync + bracket-env; the
  comment now states the custody env-isolation test is the AUTHORITATIVE control the lint can't fully see.
- **honesty:** "EMPIRICALLY PROVEN" down-scoped to "host-verified; the LIVE EC1b.1 skips in CI"; the
  "unreachable" comment corrected to "reachable on disarm → armedEmit throws → fail-closed"; an
  SSL_CERT TODO for ③.2.3 (Linux/Docker TLS).
