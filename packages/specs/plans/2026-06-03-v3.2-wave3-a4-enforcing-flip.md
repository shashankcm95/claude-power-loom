## The design fork (the core decision ‚Äî needs architect VERIFY + USER ratify)

`planned[]` names R9/R11 as *"MUST land as a tested kernel algorithm."* But Wave 2
built them in `packages/runtime/`, not `kernel/algorithms/`. Two readings, with
conflicting authoritative sources:

- **Option A ‚Äî KERNELIZE** (honor scope-doc line 214 "enforces over R9/R11's
  cores"): extract R9/R11 deterministic cores into `kernel/algorithms/*.js`,
  register + test, move `planned[]`‚Üí`algorithms[]`, then flip. The gate then
  literally binds on R9/R11.
- **Option B ‚Äî RECLASSIFY** (honor the Wave-1 boundary RULE, newer + more
  specific): R9's criteria are threshold/presence/membership checks and R11
  runs subprocesses + routes via a static map ‚Äî **none is "non-trivial derivation
  logic"**; per the boundary rule they are runtime constants that MUST NOT be
  registered. Remove the two `planned[]` entries with a rationale, then flip over
  the drained watchlist.
- **Option C ‚Äî HYBRID** (extract only genuine derivation sub-cores): collapses to
  A or B because R9/R11 contain *no* non-trivial derivation logic to extract.

### Recommendation: **Option B (reclassify)** ‚Äî pending architect + USER

Evidence weight favors B:
1. **The Wave-1 boundary rule (June 3) is newer + more specific** than the scope
   doc's line-214 summary phrase (June 2, written before the rule crystallized).
   It exists precisely to stop "trivial checks cargo-culted into the manifest."
2. **R9's own Wave-2 scope caveat** frames R9 as a declaration-conformance gate
   and anticipates the flip happening *around* it, not by kernelizing it.
3. **A4's actual spirit is "tested function, not prose"** (scope doc lines 33‚Äì34,
   82, 92). R9/R11 are tested functions ‚Äî the spirit is already satisfied.
4. **KISS / YAGNI / don't-undo-Wave-2**: B is a data + rationale change; A is a
   refactor of just-shipped, fully-tested runtime code into a shape the boundary
   rule forbids.

**Honesty caveat (for the honesty-auditor lens):** B drains `planned[]` to empty,
so post-flip the watchlist has 0 entries. Is "A4 binding" then meaningful? YES, but
state it honestly: the gate hard-enforces (a) structural integrity of every
`algorithms[]` member (route-decide: file+export+test), (b) the unregistered-`.js`
scan over `kernel/algorithms/`, and (c) **no-park-and-forget** ‚Äî any FUTURE
`planned[]` entry now fails CI rather than nagging forever. The exit criterion
"algorithm library shipped + gate enforces" is met; the "over R9/R11's cores"
clause is satisfied in spirit (their determinism is in tested code) but NOT by
manifest membership ‚Äî this reframing must be explicit, not silent.
