# ③.2.2b — the Docker actor-write-sandbox + cost-guard (the contained-actor primitive)

> Phase ③.2 (live external-PR beta) · sub-wave ③.2.2b · trust-ZERO · emits NOTHING · gates NOTHING.
> Build + PROVE the primitive that the ③.2.2c semantic loop will run its actor through: a `claude -p`
> actor executing INSIDE a Docker container over a bind-mounted throwaway clone, WRITE-confined to the
> clone (an out-of-clone write hits the ephemeral container FS, never the host), network-ON (the actor
> needs egress to the Anthropic API — the one place this sandbox is deliberately weaker than the
> `--network none` GRADE sandbox), the API key injected as a container env var (never argv, never
> logged), and per-run `total_cost_usd` captured into a fail-closed cost-ledger. No issue loop, no
> grading, no DRAFT — those are ③.2.2c. This wave is the containment + cost FLOOR, proven by a
> deterministic containment spike + ONE real in-container dogfood.

## Decision trail (why a separate sandbox wave, why Docker)

The ③.2.2b → ③.2.2c split came out of an ADR-0012 probe that falsified the load-bearing premise of the
prior plan:

- **The host-side actor is NOT write-confined (firsthand-probed).** `runActorTrajectory`
  ([trajectory-friction-run.js:73-86]) spawns `claude -p` with `--allowedTools` ONLY — no `--add-dir`,
  no `--permission-mode`, no sandbox — in the clone `cwd` on the HOST. A prompt-injected stranger repo
  can `Write` to a host path OUTSIDE the throwaway clone (p-writescope), and the host-side
  `captureActorDiff` ([_clone-lifecycle.js:297]) only ever sees the in-tree `git diff --cached`. The
  probe: an INNOCENT out-of-`--add-dir` write SUCCEEDED; the model only DECLINED the obvious-escape
  attempts — a soft, probabilistic defense, never a boundary.
- **USER decision: contain the actor in Docker** (not a macOS seatbelt profile — `claude` needs the
  real HOME for auth + writes to `~/.claude/` during a run, so a seatbelt allow/deny list within
  `~/.claude` is fiddly; a container gives a clean ephemeral HOME).
- **Auth is CLOSED (firsthand, this arc):** a Max-subscription `setup-token` is rejected as a raw
  bearer inside a clean container (401); an **Anthropic API key** (`sk-ant-*`) VERIFIED authenticates
  `claude -p` in-container. Key at `~/.config/loom/anthropic-api-key` (gitignored, chmod 600, OUTSIDE
  the repo). Re-confirmed at this wave's dogfood against the actual built actor image.
- **Cost is METERED on the API key** (separate from the flat Max subscription) → a fail-closed local
  budget guard is a hard requirement, not a nicety. USER's outer backstop = prepaid Console credits,
  no auto-reload; this ledger is the inner, code-level guard.

The GRADE Docker backend already exists ([docker-backend.js], #346) but is the WRONG shape for the
actor: it is `python3 + pytest` ONLY ([Dockerfile]) and `--network none` ([docker-backend.js:48,117]).
The actor needs `node + claude` AND network. So ③.2.2b is a NEW, sibling backend that REUSES
docker-backend's proven hardening helpers (`assertSafeMountPath` / `assertSafeName` / the cap-drop +
read-only + tmpfs + non-root posture) but inverts the network stance and swaps the image + command.

## Scope (③.2.2b — the primitive, attested + dogfooded; NO loop)

**IN scope:**
- **`docker-backend.js` refactor (additive; the grade path unchanged) — the DRY/reaper close (architect
  F1+F5):**
  - Extract a `dockerHardeningFlags({ memory, pidsLimit, cpus, tmpfsSize, user, name, ownerPid })` →
    the SHARED posture array (`--cap-drop ALL`, `--security-opt no-new-privileges`, `--read-only`,
    `--tmpfs /tmp`, `--pids-limit`, `--memory`/`--memory-swap`, `--cpus`, `--init`, `--user`, `--name`,
    the `loom-owner` label). BOTH `buildDockerRunArgs` (grade) and `buildActorRunArgs` (actor) compose
    it; each adds its OWN `--network` + mount-mode + command. So a future hardening fix reaches BOTH
    backends (no copy-paste drift). `buildDockerRunArgs`'s existing output is byte-identical → its unit
    tests stay green.
  - Refactor `runInContainer` to accept optional `runArgs` (pre-built; else built via
    `buildDockerRunArgs` as today) + `input` (stdin) + `spawnEnv` (merged into the `docker run` child's
    env). Defaults preserve the grade path VERBATIM (no stdin, no extra env). This lets the actor REUSE
    the proven reaper (`installReaper`/`_inflight`/`reapOrphans`) + cleanup + wall-clock-kill + close
    handling rather than re-introduce the SIGKILL-stranded-container leak. **Re-run the FULL grade
    backend + ContainerAdapter suite to prove zero regression** (the refactor touches a proven primitive).
- **`docker-actor-backend.js`** (NEW, OUTSIDE `tests/unit/**` so Linux CI never globs it; PURE halves
  exported + unit-tested with no daemon):
  - `buildActorRunArgs({ image, workDir, command, argv, network, limits, name })` — PURE: the
    `docker run` argv that runs an arbitrary `command`/`argv` over the bind-mounted clone at `/work`,
    composing `dockerHardeningFlags`. REUSES `assertSafeMountPath` + `assertSafeName`. INVERTED vs the
    grade backend: `/work` is bind-mounted READ-WRITE (the actor Edits files) and `--network` defaults
    to a NON-`none` mode (egress for the API) gated by a NEW per-actor `ACTOR_ALLOWED_NETWORKS`
    (e.g. `{bridge}`) — NEVER widen the grade backend's frozen `{none}` set (hacker #3/#6). The
    arbitrary-`command` seam is load-bearing: the attest payload (a node probe) AND the `claude -p`
    actor both flow through it. The API key is passed as `-e ANTHROPIC_API_KEY` (NAME-only in argv; the
    VALUE rides the child's `spawnEnv`, never argv).
  - `runActorInContainer({ dockerBin, image, workDir, prompt, apiKey, model, allowedTools, limits })` —
    IMPURE: build the actor argv (`claude -p --output-format stream-json --verbose --model M
    --allowedTools T [--max-budget-usd N]`, the toolset PINNED to `['Read','Grep','Glob','Edit','Write']`
    — NO Bash, NOT the `runActorTrajectory` default which includes Bash; honesty #4a/hacker #1), then
    `runInContainer({ runArgs, input: prompt, spawnEnv: { ANTHROPIC_API_KEY: apiKey } })` — prompt on
    STDIN (the firsthand-proven contract; a wrapper `exec "$@"` preserves stdin), capture stdout
    (stream-json) under a `maxBuffer` cap (mirror `runActorTrajectory`'s 16 MiB; hacker #6), parse the
    events + `total_cost_usd`. **SCRUB the captured `stdout`/`events` through the existing
    `scrubEmitDiff` BEFORE returning/logging/persisting (hacker #1 HIGH — the `/proc/self/environ` →
    tool_result → captured-stdout key-leak sink; build-now, NOT deferred).** Return contract is a STRICT
    SUPERSET of `runActorTrajectory` (architect F2): success
    `{ ok:true, events, stdout, cwd: <HOST bind-mount path, NOT /work>, costUsd }`; failure
    `{ ok:false, reason, events, [status], costUsd: number|null }` — `reason`/`status` preserved
    (③.2.2c reads `cap.reason`), `cwd` is the HOST path (③.2.2c's host-side `captureActorDiff` needs it),
    `costUsd` present (nullable) on BOTH branches (a budget-capped / non-zero-exit run may still have spent).
  - `attestActorContainment({ dockerBin, image })` (async, cached `containmentAttested`) — proves the
    WRITE boundary with a NON-LLM **node** payload (free; the image has node, NOT python, so the grade
    backend's python `ATTEST_PAYLOAD` is NOT reusable — hacker #2). The negative leg MIRRORS
    `attestDocker`'s NON-VACUOUS `/proc/self/mountinfo` host-tree scan (`CLEAN` = no host-tree bind
    mount but `/work`; honesty #1 — a write to a host path the Linux image LACKS is vacuous and proves
    nothing). The positive leg is the effect oracle (`/work` write lands on the host mount, host-side
    `existsSync`). The `claude` binary is present (`claude --version` exit 0). This attest DROPS the
    `netBlocked` gate (network is deliberately ON — hacker #6/architect #5) but KEEPS the host-mount
    scan + scopedWrite; the attest result NAMES its scope as "write-boundary + binary-presence only"
    (network reachability is proven by the dogfood EC.b5, NOT the attest — honesty/architect #5).
    Invariant restated: **"no write reaches the HOST"** (the actor MAY write tmpfs `/tmp` — outside
    `/work` but not the host), NOT "confined to `/work`."
- **`cost-ledger.js`** (NEW; PURE-ish — the ledger path is the only I/O, injectable for tests). **The
  ledger file is HOST-ONLY, OUTSIDE any bind-mount, chmod 600 (hacker #5 HIGH — never under `/work`,
  which is actor-writable RW):**
  - `parseCostFromStreamJson(stdoutOrEvents)` — extract `total_cost_usd` from the final `result` event;
    fail-safe to `null` on an absent/malformed field (NEVER a fabricated 0 that hides a real spend).
  - `recordCost({ ledgerPath, runId, issueId, costUsd, now })` — append one JSONL line
    `{ ts, runId, issueId, costUsd, cumulativeUsd }`. `now` injected (testable); the cumulative is
    re-derived from the prior total + this cost (never trusts a caller-supplied cumulative).
  - `readLedgerTotal({ ledgerPath })` — sum `costUsd` over all lines; returns `{ total, malformed }`
    (the malformed-line COUNT is surfaced, NOT silently swallowed — architect #3 / hacker #5).
  - `resolveBudgetCap()` — `LOOM_COST_CAP_USD` env → fail-safe to the **$20** default (frozen-default
    discipline, mirrors `resolveMaxCloneBytes`; an absent/non-finite/<=0 env can NEVER fail-open).
  - `DEFAULT_ESTIMATED_USD` — a FROZEN conservative per-run estimate (the plan's ~$1.30/issue upper
    bound), NOT a caller-supplied value (architect #3 — a too-low caller estimate lets one run overshoot).
  - `assertWithinBudget({ ledgerPath, capUsd, estimatedUsd })` — fail-CLOSED on TWO conditions:
    (a) if `readLedgerTotal.malformed > 0` → REFUSE (a corrupt ledger means the total CANNOT be proven
    under cap — never skip-to-undercount, hacker #5 HIGH); (b) if `total + estimatedUsd > capUsd` →
    REFUSE. WARN at >=80%. Called BEFORE a run starts (the pre-spend gate); the in-container
    `--max-budget-usd` per-call cap is a SECOND independent ceiling (build-time probe #2).
  - `resolveActorApiKey()` — read `~/.config/loom/anthropic-api-key` (or `LOOM_ANTHROPIC_KEY_FILE`
    override), trim, return `null` if absent/empty. The key VALUE is never returned in any log/report.
- **`Dockerfile.actor`** (NEW, `loom-actor:latest`) — a DIGEST-pinned node base + a VERSION-pinned
  `@anthropic-ai/claude-code` global install; `HOME=/tmp` (writable tmpfs at run time); NO git, NO repo
  COPY (the clone is host-side, bind-mounted; the diff capture is host-side). Built from stdin context
  (sends ONLY the Dockerfile), mirroring the grade image's build discipline.
- **Tests** (unit, CI-globbed) + a **containment spike** + a **real dogfood** (both `_spike`,
  green-or-skip, never CI-globbed).

**OUT of scope (deferred):**
- The puller → solve → grade → DRAFT LOOP → ③.2.2c (reuses `runActorInContainer` from this wave).
- The no-regression behavioral leg → ③.2.2d (the contained-online-install network path).
- Egress RESTRICTION to only the Anthropic API host (an egress-proxy/firewall sub-project) → a
  ③.2.3-ARMING precondition, NOT ③.2.2b (this wave emits nothing, gates nothing, trust ZERO — the
  open-egress residual is documented + carried, see the security model).

## Build-time probes (ADR-0012 — verify the PATH that exercises the mechanism, before relying on it)

1. **`claude -p --output-format stream-json` final `result` event carries `total_cost_usd`** — a tiny
   real call; parse the stream, assert the cost field is a positive number. (If stream-json omits it,
   fall back to a `--output-format json` summary call shape — decide at build.) **This probe is a HARD
   PREREQUISITE of EC.b5** (honesty #2 — an EC must not assert an un-probed runtime field as fact);
   EC.b5 gates on "the cost field resolved by WHICHEVER output-format this probe confirms," not on
   stream-json specifically.
2. **`claude -p --max-budget-usd <n>` exists + caps spend AND a capped run STILL emits `total_cost_usd`**
   (architect #3 — joint probe). Confirmed present on the host `claude` (2.1.177, hacker firsthand);
   re-confirm in the VERSION-pinned image. If a budget-terminated run emits NO `result` event,
   `parseCostFromStreamJson` → `null` → the ledger under-counts that run's spend (a cumulative
   fail-open). If absent/renamed, the pre-spend `assertWithinBudget` + the frozen `DEFAULT_ESTIMATED_USD`
   are the only ceiling.
3. **An out-of-`/work` Write in-container does NOT reach the host** — the containment spike (the INVERSE
   of the host-side probe that showed the escape was OPEN). Deterministic, non-LLM NODE payload; the
   negative leg is the `/proc/self/mountinfo` host-tree scan (NOT a vacuous write to a host path the
   image lacks).
4. **The built `loom-actor` image authenticates `claude -p` with `-e ANTHROPIC_API_KEY` + network-on** —
   the dogfood; re-confirms the arc's earlier in-container auth verification against the ACTUAL image.
5. **The pinned actor toolset EXCLUDES `Bash`** (honesty #4a / hacker #1) — assert `runActorInContainer`
   invokes with `['Read','Grep','Glob','Edit','Write']`, NOT the `runActorTrajectory` default
   (which carries Bash); a Bash-bearing actor on network-on reopens the direct socket-exfil channel the
   threat model dismisses.

## Security & correctness model (the network-on tension is the headline)

This sandbox is deliberately WEAKER than the grade sandbox in exactly one axis — **network is ON** (the
actor must reach `api.anthropic.com`). Everything else (host-write boundary, cap-drop, non-root,
read-only root, pid/mem bounds) is at PARITY with the proven grade backend. The threat model for the
open egress (sharpened by the VERIFY board):

- **No DIRECT in-actor exfil channel — PINNED, not inherited.** The actor toolset is PINNED to
  `['Read','Grep','Glob','Edit','Write']` (NO Bash) inside `runActorInContainer` — NOT the
  `runActorTrajectory` default, which includes `Bash` ([trajectory-friction-run.js:73]). With no Bash
  the actor cannot open an arbitrary socket; the only egress is `claude -p`'s own API calls. A
  build-time probe asserts the exclusion (a Bash-bearing actor on network-on would reopen the channel).
- **The TRANSCRIPT sink is scrubbed THIS WAVE (hacker #1 HIGH — build-now, not deferred).** The Read
  tool CAN read `/proc/self/environ` → `ANTHROPIC_API_KEY` (`--read-only` does not unmount `/proc`;
  `cap-drop`/`no-new-privileges` don't block reading one's own env), and `--output-format stream-json
  --verbose` puts every tool_result into the captured `stdout`/`events` that `runActorInContainer`
  RETURNS — the IN-WAVE artifact EC.b5/b6 claim is key-free. Mitigation (buildable NOW; the scrubber
  exists): scrub `stdout`/`events` through `scrubEmitDiff` (it redacts `sk-ant-*`) BEFORE returning/
  logging/persisting. The DIFF sink is separately inert (③.2.2b captures no diff; ③.2.2c adds the diff scrub).
- **The DEFERRED residual (carried, honestly scoped).** The HARD close — egress RESTRICTION to only the
  Anthropic host (an egress proxy/firewall) so the key can't leave even via a hypothetical channel — is
  a ③.2.3-ARMING precondition, loudly carried, NOT built here (KISS/YAGNI: emits nothing, gates nothing
  this wave; the transcript scrub + no-Bash cover the in-wave artifact). The residual is that the key is
  transiently present in the raw in-container stdout BEFORE the scrub — in-process, never persisted.
- **The API key is never argv / never in the ledger.** `-e ANTHROPIC_API_KEY` passes the NAME; the
  VALUE rides the child's `spawnEnv`. A test asserts NO `sk-` substring in the argv-builder output and
  no key in the ledger-writer output (the PROVABLE scope — honesty #5; NOT the unprovable "never in any
  log line"). Stored chmod 600, gitignored, OUTSIDE the repo. (No hardcoded secret; no literal `*_KEY`
  assignment — the key is a runtime-resolved value.)
- **Host-write-confinement is ATTESTED with a NON-VACUOUS oracle (honesty #1 — the most damaging
  catch).** The negative leg is the `/proc/self/mountinfo` host-tree scan ([docker-backend.js:281-294],
  `CLEAN` = no host-tree bind mount but `/work`), NOT a write to a host path the Linux image LACKS
  (which fails regardless of containment — proof-shaped, proves nothing). The positive leg is the effect
  oracle (`/work` write lands on the host mount; host-side `existsSync`). The invariant is **"no write
  reaches the HOST"** (the actor MAY write tmpfs `/tmp` — outside `/work`, not the host). A clean unit
  suite is NOT this proof (Rule 2a); the non-LLM node payload makes the boundary deterministic.
- **Mount-spec injection** on the bind source is closed by `assertSafeMountPath` (reused; rejects
  `:`/`,`/whitespace/leading-`-`/non-absolute) — the clone path flows from `mkScoped` → `os.tmpdir()`,
  honoring `TMPDIR`. The actor backend defines its OWN `ACTOR_ALLOWED_NETWORKS` (never widens the grade
  backend's frozen `{none}`).
- **The cost ledger is HOST-ONLY (hacker #5 HIGH).** It lives OUTSIDE any bind-mount (never under
  `/work`, which the actor can write), chmod 600. `assertWithinBudget` fails CLOSED on a corrupt ledger
  (any unparseable line → REFUSE, never skip-to-undercount) AND over-cap.
- **Fail-closed everywhere:** un-attested backend → skipped; over-budget / corrupt-ledger → refuse to
  start; absent key → no run; absent image → a build-hint error, never a silent build-on-demand.

**Explicitly NOT claimed:** this wave proves the HOST-WRITE boundary + COST capture. It does NOT solve a
real issue, grade anything, or emit anything. **"Trust-ZERO" scopes the SIGNAL (nothing emitted/graded),
NOT the stakes** (honesty #3): a leaked key or a fail-open budget guard is a real-secret / real-money
incident even with zero emission — handled as such. Only a maintainer merge hardens trust (OQ-NS-6).
