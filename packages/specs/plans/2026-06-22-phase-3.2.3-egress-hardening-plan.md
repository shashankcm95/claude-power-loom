### VF1 — the CORRECTED H1 rule (owner-vs-repo-typed)
After `const [owner, name] = repo.split('/')` (keep the existing `REPO_RE` shape gate + `..`/`:`/host-allowlist):
- **OWNER** (GitHub login: alnum + single internal hyphens, no dots/underscores, no leading/trailing hyphen):
  reject unless `/^[A-Za-z0-9](?:-?[A-Za-z0-9])*$/.test(owner)`. This alone rejects `.`-owner, `.github`-as-owner,
  leading-dot/`..`/leading-`-` owners, and a dotted owner (`o.w`).
- **REPO** (alnum + `.` `_` `-`; may lead with `.`/`_` but NOT `-`; cannot be exactly `.`/`..`; no trailing `.`):
  reject unless `/^[A-Za-z0-9._-]+$/.test(name)` AND `!/^\.{1,2}$/.test(name)` AND `!name.startsWith('-')` AND
  `!name.endsWith('.')`. (Leading-`_`/leading-`.` ALLOWED — `.github`, `_repo` are legal repo names.)
- **EC1 (corrected) — REJECT**: `.github/x` (dotted owner), `./r`, `o/.` (dot-only repo), `o.w/r` (dotted owner),
  `owner/..`, `owner/x.` (trailing-dot), `owner/-x` (leading-`-` repo), `-o/r` (leading-`-` owner), `o..o/r` (`..`).
  **ACCEPT**: `owner/repo`, `owner/.github`, `owner/react.dev`, `o-w-ner/re.po`, `owner/_repo`.
  (NB the plan's earlier `o.w-ner/re.po` accept-case was WRONG — a dotted OWNER is illegal; corrected to `o-w-ner`.)

### VF3 — the CORRECTED H5 gate (fail-CLOSED)
`verifyToollessRuntime({ bin })` returns `{ ok, tools, reason }`. It returns `{ok:true, tools:[]}` ONLY when a
stream-json INIT event is successfully parsed AND `Array.isArray(tools) && tools.length === 0`. It returns
`{ok:false, reason}` on EVERY other path: bin absent, `res.error`/`ETIMEDOUT`, `res.status !== 0`, no init event
parsed (`initTools === null`), `tools` not an array, OR `tools.length > 0`. The caller proceeds ONLY on
`r && r.ok === true` (fail-closed by default). `runLiveDraftLoop` runs it ONCE as the FIRST preflight (cheapest
fail-closed check, before `preflightEnv`'s container attest), via an injectable `deps.verifyToollessFn` (default
the real impl); when judges are dependency-INJECTED (`deps.semanticFn`/`deps.frictionFn` both provided — the test
path), the preflight is SKIPPED (no real claude is used). **EC5 (corrected)** tests all dispositions with a fake
runner: `tools:[]`→proceed; `["LSP"]`→fatal abort; bin-absent→abort; non-zero-exit→abort; no-init→abort;
non-array-tools→abort. This INVERTS the Leg-P/G4 canary's SKIP-on-inconclusive (fail-open is right for a
non-blocking canary; fail-CLOSED is mandatory for a gate admitting stranger input).
