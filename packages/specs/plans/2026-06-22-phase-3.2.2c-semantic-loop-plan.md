## REWORK NOTE (what the recon changed vs the seed)

The seed named the **host-side** `runActorTrajectory` for actor-solve and a host clone "UNSANDBOXED on the host." Three
parallel read-only recon agents (codebase-analyzer ×3, against this worktree @ `b3de0b7`) firsthand-verified the actual
contracts and changed three load-bearing premises:

1. **`real-solve.js`/`runActorSolve` runs the actor UNSANDBOXED on the host** (`real-solve.js:163` calls
   `runActorTrajectory({cwd: actorDir})`, a bare `spawnSync` — NOT `runActorInContainer`). The #391 sandbox is built +
   exported + spike-proven but **wired into NO live path** (only `_spike/` calls it). ③.2.2b exists precisely because a
   host actor is not write-confined — so ③.2.2c **MUST** route the live actor-solve through `runActorInContainer`, which
   means **building the wiring**, not reusing `makeRealSolve`. `makeRealSolve` is doubly wrong here: host-side AND it
   GATES on `backend.containmentAttested` for a behavioral grade we are NOT running (semantic-only).
   → **Decision: a lean new contained live-solve** (`solveLiveIssueContained`), composing the #391 + ③.2.0 primitives
   directly. It reuses the *hardened clone* (`prepareClone`/`captureActorDiff`) AND gets the *container* — strictly better
   than the seed's option-(a) stub-backend-into-`makeRealSolve` (which would have run host-side).
2. **`captureActorDiff` is already git-plumbing-only** (`_clone-lifecycle.js:300-303`: `git add -A` + `git diff --cached`,
   no working-tree `readFileSync`). A symlink the contained actor plants in `/work` (the host bind-mount) is captured **as a
   symlink** (git mode-120000 blob whose content is the link *target path string*, never the target's bytes) — so M1's
   `~/.ssh/id_rsa` exfil is **structurally prevented by the existing capture**. `.git/config` is separately `O_NOFOLLOW`-
   guarded (`restoreConfigNoFollow`). → M1 carry is largely DISCHARGED; the only residual is whether a `mode 120000`
   symlink **entry** should be neutralized in the candidate diff (hardening, not an exfil hole) — a VALIDATE-hacker question.
   **Invariant the build MUST hold:** the ONLY read of the actor's tree is via `captureActorDiff` (git plumbing); the loop
   never `readFileSync`/globs the `/work` tree to build the draft (it works from the diff string only).
3. **`emitPR(data, opts)` contract** (the seed's 3-field draft was wrong): TWO positional params; `data={repo,issueRef,diff}`
   UNTRUSTED + must carry NO policy key (`assertDataIsPolicyFree` throws on `disposition`/`mode`/`token`/…); `data.repo` must
   be **bare `owner/repo`** (`REPO_RE` rejects the `https://…` URL on the `:`), but `buildPublicRecord` emits
   `repo: https://github.com/owner/repo` → **a repo-slug transform is required**; `issueRef` is parsed from the record `id`
   (`${owner}__${repo}-issue-${N}` — there is NO separate number field); the dry-run return is
   `{ok:true, emitted:false, disposition:{mode:'dry-run',draft:true}, draft:{repo, issueRef:<Number>, title, touched_paths, diff:<scrubbed>}}`
   — 5 draft fields + top-level `disposition`. emitPR **scrubs the diff on every path**, rejects RAW `.github`/`.git*`/CI
   paths before drafting, and **spawns no `gh` on any path** (`child_process` not imported this wave; `armedEmit` throws only
   in the live branch).

## Decision trail (why semantic-only, why now) — unchanged, still valid

The original ③.2.2b plan proposed a Docker-executed **no-regression** behavioral leg. The pre-build VERIFY board falsified
its load-bearing premise: the no-regression leg cannot REUSE `runTests` (the pytest wrapper emits results ONLY for
explicitly-named `test_ids` → empty baseline `G` → `G ⊆ pass(G′)` VACUOUSLY true), and the grade sandbox is `--network none`
with a `python3+pytest`-only image, so running an arbitrary repo's tests needs its deps → network + running untrusted
`setup.py`. A semantic-only grade executes nothing → no Docker-grade, no deps roadblock, works on EVERY repo.

**USER decision:** ship the non-limiting semantic-only loop now; take up the no-regression signal LATER (③.2.2d) on a
contained-online-install network path. The behavioral signal is SHADOW (narrows-not-hardens, OQ-NS-6) — additive.

## Scope (③.2.2c — the full semantic-only loop)

**IN scope:**
- **`gradeLiveIssueSemantic({ record, candidate, semanticFn, frictionFn })`** (NET-NEW; `packages/lab/causal-edge/live-grade.js`)
  → a SHADOW verdict composing ONLY the oracle-free legs (verified public-safe):
  - **Leg B (REUSE)** — blind `claude -p` semantic judge: `buildLegBInput(record)` → `{input, dropped}` (a public-only record
    has no `criteria_only_rubric` → the rubric branch is skipped, `dropped:false`, no throw); `semanticFn(input, candidate)`
    → `{status:'advisory_llm_checked', supported: bool|null, outcome_source:'model'|'harness_fallback', fallback_reason?}`;
    fail-closed to `supported:null` on refuse/throw.
  - **Leg D (REUSE)** — friction diagnostic: `buildFrictionLabelerInput({problem_statement_digest, candidate_patch, processGraph})`
    (public-safe metrics only — path list stripped); `frictionFn(input)` → friction block | **null**;
    `validateResolutionFriction` re-checks the closed enums. Report-only.
  - **Verdict shape:** `{ behavioral: 'UNAVAILABLE', semantic_supported: bool|null, friction: {...}|null, oracle: 'none',
    shadow: true }`. Carries **NO** `score`/`grade`/`pass`/`overall`/`reference`/`recall_eligible` key (the `reference`
    oracle leg reads `accepted_diff` → OMITTED; `recall_eligible` is structurally false for a live issue and is OMITTED to
    avoid implying a recall path exists). Structural markers, never a proof-of-fix gate.
- **`solveLiveIssueContained({ record, … })`** (NET-NEW lean contained solve; in `live-draft-run.js` or a focused sibling —
  VERIFY decides the file home). The pipeline, all primitives REUSED:
  1. `resolveActorApiKey()` → key; **null → return `{ok:false, reason:'actor-key-absent'}`** (no run).
  2. `assertWithinBudget({ ledgerPath, capUsd, estimatedUsd })` — throws on corrupt ledger or projected-over-cap (fail-closed).
  3. `attestActorContainment()` — **must be `attested:true`** else `{ok:false, reason:'containment-unattested:<reason>'}` (no run).
  4. `prepareClone({ repo: record.repo, base_sha: record.base_sha })` → `{workDir, configSnapshot}`.
  5. `runActorInContainer({ workDir, prompt, apiKey, model, maxBudgetUsd, timeout })` → `mapActorResult` shape

  6. `recordCost({ ledgerPath, runId, issueId: record.id, costUsd: result.costUsd })` (re-derives cumulative; rejects negative).
  7. `captureActorDiff({ workDir, configSnapshot })` → candidate diff (git plumbing; symlink-safe). Size-cap the diff.
  8. `safeDiscard(workDir)` in `finally`.
  Returns `{ ok, candidate, costUsd, redacted, reason? }`. **The ONLY tree read is step 7's git capture** (M1 invariant).
- **The DRAFT artifact via the egress chokepoint (REUSE):** transform `record` → `{ repo: <bare owner/repo from record.repo
  URL>, issueRef: <int parsed from record.id>, diff: candidate }`; call `emitPR(data, {})` (default disposition = dry-run)
  → `{ok:true, emitted:false, disposition, draft}` WITHOUT emitting. Write the `draft` (+ verdict + run metadata) to a local
  artifacts dir; emit NOTHING. (③.2.3 becomes a disposition flip + token wiring, nothing more.)
- **The orchestration driver `runLiveDraftLoop(...)`:** loop over live-puller records → `solveLiveIssueContained` →
  `gradeLiveIssueSemantic` → `emitPR` dry-run → write artifact + a JSON run report. Per-record fail-soft (one bad record does
  not abort the loop). Emits nothing.
- **Unit tests** (mock `semanticFn`/`frictionFn` + injected mock contained-solve + a real dry-run `emitPR`) + a **live dogfood**
  (`_spike/live-draft-dogfood.js`, not CI-globbed): the REAL `claude -p` contained actor on ONE real good-first-issue repo
  end-to-end (Rule 2a-corollary — a green mock suite is a hypothesis; the real contained path is where bugs hide).

**OUT of scope (deferred):**
- The no-regression behavioral leg → ③.2.2d (contained-online-install α network path; its own security plan + VERIFY).
- Any emission / token / live PR → ③.2.3 (flips the `emitPR` disposition to live + wires the token).
- A "does it FIX the bug" oracle for open issues — impossible without a human (OQ-NS-6).
- **Cost-metering the leg-B/leg-D JUDGE calls** — the cost-ledger meters the ACTOR (`runActorInContainer.total_cost_usd`),
  not the two host-side judge `claude -p` calls. Acceptable this wave (a small confirmation round; judges are cheap); if the
  loop is ever batched at scale, the judges need their own budget guard. Logged, not silently capped.

## Architecture decision (the VERIFY board confirms or overrides)

- **Contained live-solve, not `makeRealSolve` reuse** (recon finding #1). New `solveLiveIssueContained` composes #391 + ③.2.0
  primitives. Rationale: `makeRealSolve` runs host-side AND gates on a behavioral backend we do not have. The hardened clone
  IS reused (`prepareClone`/`captureActorDiff`); only the host-side `runActorTrajectory` orchestration is bypassed.
- **File homes (VERIFY to confirm SRP factoring):** `live-grade.js` (pure-ish grade composer, causal-edge); `live-draft-run.js`
  (driver + `solveLiveIssueContained` + the record→emitPR transform, persona-experiment). A small pure helper
  `parseRecordRef(record)` → `{ slug:'owner/repo', issueRef:<int> }` (transforms the URL + parses the id) — co-located with
  the grade or the driver; VERIFY picks the owner of the seam.
- **Judges run host-side and MUST be tool-PINNED** (`claudeOnce` + `--tools "" --strict-mcp-config --disallowedTools LSP`). VERIFY proved the
  default `makeBlindSemanticJudge`/`makeFrictionLabeler` spawn a BARE `claude -p` (no tool flags → DEFAULT tools incl. Bash),
  so reusing them verbatim would run a prompt-injectable judge on the HOST with write/exec reach (the API key, ledger, gh
  creds). FOLD: parametrize `claudeOnce(bin, prompt, timeout, extraArgs=[])` (additive, default = byte-identical) + a
  `toolless` option on both makers that threads the [[capability-free-claude-headless]] recipe; the live driver builds judges
  with `toolless:true`. The real dogfood PROBES that tool-less actually disables tools (Rule 2a-corollary; the recipe is
  firsthand-verified in memory but load-bearing here).

## Security & correctness model (the ACTOR is contained; the host-side judges are tool-PINNED LLM calls)

The risk surface vs the seed: the actor now runs **contained** (#391: mount-namespace, `--cap-drop ALL`, `--read-only` root +
`--tmpfs`, non-root, `--pids-limit`/`--memory`, write-confined to the `/work` bind-mount), network-ON for the API only. The two
SHADOW judges run host-side but **tool-pinned** (`--tools "" --strict-mcp-config --disallowedTools LSP`) — they cannot take a host action, so the only
host-side execution of stranger content is inert LLM classification whose worst case is a corrupted SHADOW verdict.
- **Prompt-injection of the contained actor** (issue text + repo files are uncurated stranger content): blast radius is a bad
  DIFF in a throwaway clone → the actor cannot write outside `/work` (container) and cannot execute the repo (no Bash). The
  bad diff is scrubbed by `emitPR`, graded SHADOW, emits nothing.
- **M1 `/work` symlink** (the actor plants `/work/evil -> ~/.ssh/id_rsa`): `captureActorDiff` git-captures it as a symlink
  (target *path string*, not bytes) → no content exfil. Invariant: the loop reads the tree ONLY via `captureActorDiff`. The
  VALIDATE hacker live-probes this with a crafted hostile repo. (Optional hardening: reject/flag `mode 120000` entries in the
  candidate diff — decided at VALIDATE.)
- **The container bind-mount is the host clone dir** — `prepareClone` makes it under `mkScoped`, `safeDiscard` removes it. The
  actor's writes land on that host dir (the point of write-confinement-to-host); no other host path is writable.
- **Prompt-injection of the judges** (issue + diff are attacker-influenceable): the judges are **tool-pinned**
  (`--tools "" --strict-mcp-config --disallowedTools LSP`), so an injection can corrupt only the SHADOW verdict (which gates nothing) — it cannot
  drive a host tool action. The judge INPUT is bounded by the same diff size-cap as the actor capture (a 5 MiB hostile diff
  cannot balloon the judge prompt). Judge per-call COST is un-metered this wave (logged in OUT-of-scope, not silently capped);
  tool-pinning bounds each judge to a single classification turn, keeping it cheap.
- **A `mode 120000` symlink ENTRY is rejected from the candidate** (M1 fully closed): `captureActorDiff` already prevents
  content-exfil (git stores the symlink target *path string*, never the bytes), but a symlink *entry* could still ride into the
  DRAFT (an attacker-chosen path-string leak, and a foot-gun if ③.2.3 ever emits). The transform rejects any candidate whose
  diff contains a `mode 120000` line → a fail-soft draft-rejected outcome. Closes M1 now rather than deferring.
- **The DRAFT diff is scrubbed** by `emitPR` (`scrubEmitDiff`) + `.github`/`.git*` rejected before drafting. **No emission,
  structurally** (`armedEmit` throws; dry-run; no token in any env). API key never in argv/log (NAME-only `-e`; transcript
  scrubbed by `mapActorResult` before return). No secrets in source (ADR-0006; ASCII-only); immutable record handling.

**Explicitly NOT claimed:** `semantic_supported:true` does NOT mean the candidate fixes the issue — only that a blind reviewer
found it plausible. `behavioral:'UNAVAILABLE'` is honest (no oracle). Trust stays ZERO; only a maintainer merge hardens (OQ-NS-6).
