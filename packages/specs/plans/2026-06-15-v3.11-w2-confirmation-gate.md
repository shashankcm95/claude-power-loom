# v3.11 W2 — the confirmation gate + the first `confirmed-by` edge

**Goal (RFC Sec 5/7):** a lesson is **provisional** when minted (W1) and only enters the **predictor lane** when a
**same-requirement confirming delta** arrives — a DIFFERENT run that resolves the SAME issue's SAME test-set. Build
(1) the pure confirmation gate, (2) the first graph EDGE `(failure-context, lesson) --confirmed-by--> (delta-ref)`,
(3) the lane split (confirmed = predictor; unconfirmed = hazard). Persona-agnostic; composed at the call site, NOT
merged into `isEligibleForPopulation` (the W1 architect SRP fold). Everything SHADOW/lab; no live spawn touched.

**Exit (RFC, scoped — VERIFY board fold):** **GIVEN store integrity (the standing #273 model: a lab-dir-write
attacker is out of scope, as for every lab store), a lesson provably cannot enter the predictor lane without a
same-requirement confirming delta sourced from a VERIFIED run.** Proven on fixtures: the gate rejects wrong-issue,
**empty/absent `fail_to_pass`**, subset/superset `fail_to_pass`, self-delta, **accepted-diff-as-delta**, a
**non-passing** run, **a tampered node `fail_to_pass` (caught by the exact-set check against the corpus-trusted
confirming side)**, and a non-lesson node; `canEnterPredictorLane` is false for a node with no edge. Residuals
documented (Drift): byte-distinct ≠ logically-independent (W3 trust-weight may require N distinct provenances); the
edge store proves INTEGRITY not PROVENANCE (a forged self-consistent edge is the #273 standing residual; a live
consumer re-verifies).

## Runtime Probes (firsthand, 2026-06-15)

- **P1 (node carries the join keys — the RFC's "fail_to_pass not on the node" is STALE post-W1):** a built lesson
  node carries `worked_example_ref.issue_id`, `fail_to_pass` (exact array), `candidate_patch_sha` (full 64-hex),
  `node_id` (64-hex), `lesson_signature`. **Probe:** built a node -> `{issue_id, fail_to_pass:["t_a","t_b"],
  candidate_patch_sha:<64hex>, node_id_len:64}`. **=> the cross-run join is node.fail_to_pass (exact-set) ==
  new-run.fail_to_pass AND node.issue_id == new-run.issue_id; no extra node field needed.**
- **P2 (the edge-store precedent EXISTS):** `packages/lab/persona-consumer/authorship-store.js` is a
  content-addressed `(node_id, built_by)` EDGE ledger (v3.10): `deriveAuthorshipId = sha256(canonical([basis]))`,
  verify-on-WRITE-and-READ (filename==field + body re-derives id + strict shape guards), dedup first-wins,
  `recorded_at` TOP-LEVEL outside the basis, `opts.dir`, deep-freeze on read, `retireAuthorship`. **=> build
  `recall-edge-store.js` by MIRRORING it** (the deliberate-duplication DRY decision: each lab store's verify
  predicate is security-load-bearing and differs; a shared factory obscures each firewall — authorship-store
  header). NOT a shared factory.
- **P3 (no edge store yet — the graph is edgeless):** `ls packages/lab/attribution/*.js` = bootcamp-gates,
  candidate-sidecar, recall-graph, recall-graph-store. No edge store. The RFC ("build the first edge first")
  confirms this is W2's net-new artifact.
- **P4 (the delta-ref):** RFC Sec 5 — the kernel's delta-ref is `post_state_hash` (the record-store key). For a
  BACKTEST confirming run there is no kernel spawn-record; the delta-ref is the confirming attempt's
  **`candidate_patch_sha`** (the full-sha content-address of the confirming delta, recoverable via the W1
  sidecar). **=> the edge's `to_delta_ref` is a HEX64 content-address (candidate_patch_sha for backtest); a future
  live consumer can point the same edge type at a `post_state_hash` — same shape.**
- **P5 (persona-agnostic — the v3.10 C2 laundering guard):** the gate keys ONLY on `issue_id` + `fail_to_pass` +
  `delta_ref`; it MUST NOT read `built_by`/`graded_by` (else re-import the key-fragmentation laundering lever). The
  consolidation TRUST-weight is W3; W2 only writes the edge + exposes the lane.

## Design

### The confirmation gate (pure — `causal-edge/lesson-confirm.js`) — EVIDENCE-BACKED (VERIFY board fold)

The confirming side is NOT a free-form `{delta_ref, passed}` arg (that made `passed`, `delta_ref`, and the
requirement all attacker-assertable — hacker C1/C2/H1). It is a **VERIFIED passing attempt from a NEW calibration
run**: `confirmingAttempt = {issue_id, fail_to_pass, candidate_patch, behavioral_verdict}` where `fail_to_pass` is
**corpus-declared** (it rode from `record.fail_to_pass` through `scoreAttempt`, NOT caller-chosen) and
`behavioral_verdict` is the **real** leg-A verdict (NOT a caller boolean). The confirm pass MUST source these from
a verified run; the gate enforces what it can.

`confirmsLesson(node, confirmingAttempt)` -> bool. TRUE iff ALL hold (fail-closed; any miss -> false):

- the node HAS a valid lesson layer (`classifyLessonLayer(node) === 'valid'` — only lessons confirm; reuse W1) AND
  `node.node_id` is strict HEX64 (re-verify the node handed in — MEDIUM-4; don't trust an unverified node).
- `confirmingAttempt.behavioral_verdict === 'BEHAVIORAL_PASS'` (the REAL verdict; FAIL/PARTIAL/FALLBACK never
  confirms — never a caller `passed:true`).
- `node.worked_example_ref.issue_id === confirmingAttempt.issue_id` (same issue).
- **EXACT-SET equality** on `fail_to_pass` against the **corpus-trusted requirement** (`confirmsLesson(node, att,
  trustedFailToPass)` — the requirement is resolved from the W0 corpus by `issue_id`, the SINGLE source of truth;
  BOTH the node's AND the attempt's `fail_to_pass` must exact-match it). `sameRequirement` = `missing[]`+`unexpected[]`
  both empty, NEVER subset/superset, BOTH non-empty (C1), **string-members-only (M-B: never `String()`-coerce —
  `[true]` must not match `["true"]`)**. This makes requirement-trust **GATE-ENFORCED, not a caller contract**
  (VALIDATE-hacker H-A / honesty Claim-2): a **tampered node** `fail_to_pass` (unhashed, #273 class) OR an
  attacker-influenced confirming attempt both mismatch the corpus truth -> REJECTED. Absent trusted requirement ->
  fail-closed.
- `delta_ref = sidecarSha(confirmingAttempt.candidate_patch)`; `delta_ref !== node.candidate_patch_sha` (no
  self-confirm) AND `delta_ref !== node.accepted_diff_ref` (H2: never let the sealed ground-truth confirm its own
  lesson) AND the candidate is **non-empty** (`delta_ref !== sidecarSha('')` — no trivial/empty patch).
- NEVER reads persona fields (P5).

`canEnterPredictorLane(node, confirmedNodeIds)` -> `confirmedNodeIds.has(node.node_id)`. `isConfirmed` builds the
set from the edge store **using STRICT-HEX64 `from_node_id`** (M1 — the `authorship-store.js:54` strict form, NOT
`recall-graph-store.js:57`'s coercing `String()` form). A node with no `confirmed-by` edge is HAZARD-lane. The
bool is the W2 contract; W3 may WIDEN it additively to a degree/count (MEDIUM-2) — do not hard-code the binary.

### The edge store (`attribution/recall-edge-store.js` — mirror authorship-store)

- `deriveEdgeId(rec) = sha256(canonical([from_node_id, to_delta_ref, edge_type, sorted(fail_to_pass)]))` — the
  matched requirement is IN the basis so a flipped `fail_to_pass` perturbs the id (tamper-evident; harmless to
  dedup since a node's requirement is fixed).
- body: `{edge_id, from_node_id, to_delta_ref, edge_type:'confirmed-by', fail_to_pass, recorded_at}`.
- `EDGE_TYPE = Object.freeze(['confirmed-by'])` (closed set; W3 may append).
- verify-on-WRITE-and-READ: filename==edge_id + body re-derives edge_id + **STRICT shapes** (`typeof===string` &&
  HEX64 for from/to — the `authorship-store.js:54` strict form, NOT the coercing `String()` form; type in the
  frozen set; fail_to_pass a non-empty array of strings; recorded_at parseable). dedup first-wins. `recorded_at`
  TOP-LEVEL outside the basis. `opts.dir` (`recall-edge/`), deep-freeze on read. `retireEdges({dir, before})`.
- **The store stays LESSON-AGNOSTIC** (MEDIUM-3): it validates only the edge's OWN shape (structural), NEVER imports
  `classifyLessonLayer`/persona — exactly like `authorship-store.js` knows nothing about lessons. The lesson
  classification lives in the GATE (`lesson-confirm.js`), not the store.
- **One-way-door freeze (LOW-1, mirror W1's `LESSON_HASH_FIELDS`):** `EDGE_TYPE` is an APPEND-ONLY frozen set
  (rename/remove orphans every edge keyed on it — it's in the basis); the edge BASIS field-set is
  additive-only/versioned (a new basis field requires a basis version, never an in-place add).

### The confirm pass (orchestration — in `lesson-confirm.js`, dir-injectable)

`runConfirmationPass(provisionalNodes, confirmingAttempts, opts)`: for each (node, attempt) with
`confirmsLesson(node, attempt)`, **write the confirming `candidate_patch` to the sidecar** (so `to_delta_ref` is a
real, recoverable entry — raises the H3 forgery bar: a forged edge to a phantom delta is detectable) THEN write the
`confirmed-by` edge. `provisionalNodes` MUST be store-loaded (verify-on-read passed) before reaching the pass.
Returns `{n_pairs_evaluated, n_confirmed, n_written, n_sidecar_failed, edges, confirmed_node_ids}`. Reuses the W1 admission gate as a
SEPARATE step (not merged into `isEligibleForPopulation`).

## Build (TDD; RED first)

1. **`recall-edge-store.js`** — mirror authorship-store (the verify predicate is edge-specific). RED:
   write->read round-trip; filename==edge_id; a tampered `to_delta_ref`/`fail_to_pass`/`from_node_id` -> null on
   read; a forged edge_id -> null; a non-`confirmed-by` type -> reject; dedup first-wins; `retireEdges`.
2. **`lesson-confirm.js` gate** — `sameRequirement` (exact-set; subset AND superset both false; BOTH non-empty) +
   `confirmsLesson` (the evidence-backed condition set) + `canEnterPredictorLane`/`isConfirmed` (strict-HEX64 set).
   RED: a verified confirming attempt passes; each rejection returns false — **empty/absent ftp**, wrong issue,
   subset ftp, superset ftp, **tampered node ftp (≠ the corpus-trusted confirming ftp)**, self-delta,
   **accepted-diff-as-delta**, **empty/trivial candidate**, **a non-PASS `behavioral_verdict`**, a non-lesson node,
   a junk `node_id`, persona-only difference; the lane is false without an edge, true with one.
3. **`runConfirmationPass`** — join + sidecar the confirming candidate + write edges. RED: 1 node x 1 verified
   passing attempt -> 1 recoverable edge + confirmed; a non-confirming attempt -> 0 edges; a self-delta -> 0;
   persona-agnostic (built_by differing does not change the outcome).
4. **The EXIT-PROOF test** — assert a provisional node CANNOT reach the predictor lane without a same-requirement
   confirming delta sourced from a verified run: enumerate the FULL rejection set above, assert
   `canEnterPredictorLane` stays false after each non-confirming attempt, and flips true only after a genuine
   non-empty same-`fail_to_pass`, different-non-trivial-delta, BEHAVIORAL_PASS confirm.

## Drift Notes

- The RFC said `fail_to_pass` is "not stored on the node" (Sec 5) — W1 stored it. But the VERIFY board showed the
  node's `fail_to_pass` is UNHASHED (outside `node_id` + `lesson_content_hash`), so it is NOT a trustworthy auth
  key on its own (C2). The fix is NOT to hash it (a one-way-door taxonomy change) but to make the CONFIRMING side
  trusted (corpus-declared `fail_to_pass` from a verified run) so the exact-set check catches a tampered node ftp.
- The edge `to_delta_ref` is a content-address abstraction (candidate_patch_sha now / post_state_hash later) —
  live-consumer-ready without building the live bridge (RFC Sec 9 defers it).

## Documented residuals (VERIFY board — accepted for W2, handed to W3 / the live consumer)

- **Byte-distinct ≠ logically-independent (hacker H2):** `delta_ref !== candidate_patch_sha` blocks a byte-identical
  self-confirm; it CANNOT detect a whitespace/refactor-perturbed copy of the same fix (same logic, new sha). True
  independence needs a provenance signal (distinct run-id/actor/session). W2 closes the cheap cases
  (accepted-diff, empty/trivial, byte-identical); **W3's trust-weight may require N distinct provenances.**
- **Edge integrity ≠ provenance (hacker H3, the #273 standing residual):** the edge store verifies a file is
  self-consistent (could-have-been-written-by-us), NOT that the gate actually accepted a real confirming delta. A
  lab-dir-write attacker can hand-write a self-consistent `confirmed-by` edge — the SAME residual every lab store
  shares (authorship-store, recall-graph-store, sidecar). W2 raises the bar (the confirm pass is the only
  legitimate writer; `to_delta_ref` is sidecar-recoverable so an edge to a phantom delta is detectable) and SCOPES
  the exit claim to store integrity. The future LIVE consumer must re-verify (the edge is an index, not the
  authority); a verified run-record store is a v-next dependency, NOT built this phase.
