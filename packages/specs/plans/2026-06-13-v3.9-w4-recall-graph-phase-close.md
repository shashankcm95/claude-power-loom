## 1. Scope (AS-BUILT intent — what W4 ships)

W4 is the **consumer** wave: it turns the per-attempt scorer output (W2) + the
friction blocks (W3) into the bootcamp's TWO artifacts (RFC §1) — a **recall graph
of worked examples** and a **friction map** — then ships the bootcamp's closing
gates and runs `/phase-close`. **Nothing here moves a weight, calls a model in the
unit path, or touches Path-2** (reputation/breaker — deferred to v3.10 per §6/§8/EC7).

Six deliverables:

1. **Recall-graph populator** (`packages/lab/attribution/recall-graph.js`, PURE) — builds a
   causal-recall-graph **node** per worked example, **leg-B-gated AND contamination-gated**,
   with the OQ-7 provenance firewall (`provenance` in the content-address basis) and NO weight field.
2. **Recall-graph node store** (`packages/lab/attribution/recall-graph-store.js`) — a dir-injectable
   **per-node-file** store (mirrors `manage-proposal/store.js`, NOT the single-ledger verdict
   store — A-LOW-1): `$LOOM_LAB_STATE_DIR/recall-graph-backtest/<node_id>.json` via kernel/_lib
   atomic-write, dedup-by-file-exists (first-eligible-wins), **content-verify + node_id-verify +
   deep-freeze on read** (#273 + the read-path immutability rule), **provenance-reject** any
   non-`backtest` node (the OQ-7 physical firewall). CI-testable with a temp dir.
3. **Friction-map aggregate** (in `recall-graph.js`) — the W3-deferred cross-issue roll-up:
   harvest `resolution_friction` blocks across attempts → `clusterFriction` (W3) →
   a friction-map object reported per-cluster.
4. **Judge precision/recall agreement** (in `recall-graph.js`) — the W3-deferred "the judge is
   itself a calibration target" (§3.1): agreement of leg-A behavioral PASS vs leg-B
   semantic supported over MODEL-decided attempts, with the labeler error bar reported
   **`UNKNOWN-until-measured`** on this corpus (never the borrowed 87%/13%).
5. **Two grep gates** (`packages/lab/attribution/bootcamp-gates.js`, PURE + a thin CLI) — the
   **wording-audit** gate (forbidden "learns/trains/improves over time" near a metric +
   proposed-schema-field-asserted-as-pre-existing) and the **EC7** gate (ZERO
   `recordVerdict`/reputation/circuit-breaker references from any bootcamp module).
6. **Path-1 wiring (incl. the friction leg) + dogfood + `/phase-close`** — WIRE the existing
   W3 `makeFrictionLabeler` + `runActorTrajectory` (`trajectory-friction-run.js`) into the
   runner's leg set (without it the friction map is empty-by-construction — §6a), thread the
   aggregate + judge agreement + recall-graph population into `calibration-issue-run.js`'s
   Path-1 record (impure, immutable persist), drive every NEW path LIVE in a dogfood
   (`_spike/`), then `/phase-close` over the integrated bootcamp vs the §7 AS-BUILT EC table.

  // --- W4 additive (RFC §3.4 — R12: worked_example_ref is NET-NEW) ---

    behavioral_verdict, reference_divergence,   // null-OK (leg C divergence never gates; HON-LOW-2)

**The leg-B gate (R3, RP-2):** `populateRecallGraph(attempts)` keeps ONLY attempts where
`recall_eligible===true && reference != null`. A `BEHAVIORAL_PASS` whose leg B did not
affirmatively support (`recall_eligible===false`) is DROPPED — a gamed/behavioral-only
example must not poison retrieval. A negative control is structurally excluded
(`is_negative_control` flips `recall_eligible` false at the source).

**The contamination gate (VERIFY hacker H-MED-3 — the strongest OQ-7 poison):** a
memorized/training-represented issue PASSES behavioral *because* it is memorized, so the
leg-B gate alone does not protect retrieval. Add a second gate: a worked example whose
`contamination_tier ∉ CLEAN_FOR_RETRIEVAL = {'clean-pending-probe','clean','unknown'}` is
DROPPED from population (known-contaminated `grey`/`stale` never enter the graph). Every
admitted node still carries `contamination_tier` + the derived `contaminated` boolean for
defense-in-depth. (`contamination_tier` is W2/W3-populated — ABSENT at W0 per `corpus.js:114`
— and reaches the populator via the scorer's threaded `tierOf`.)

**The OQ-7 physical firewall (VERIFY hacker H-CRITICAL — `provenance` was cosmetic):** a
stamped-but-unread `provenance` tag excludes NOTHING — there is zero live-retrieval consumer
in the repo today (verified: `grep -rn provenance packages/**/*.js` → no readers). So the
firewall is made STRUCTURAL, not a label: bootcamp nodes write to a PHYSICALLY SEPARATE store
dir `$LOOM_LAB_STATE_DIR/recall-graph-backtest/`, and the store REJECTS on read/write any node
whose `provenance !== 'backtest'`. A future v3.10 live retriever pointed at a live store can
NEVER reach a path under `recall-graph-backtest/`. The plan does NOT claim runtime live-exclusion
is enforced today (no live retriever exists) — provenance + physical separation is a FORWARD
contract that makes every bootcamp node queryable-as-excluded and physically unreachable from live.

**Content-verify-on-read + dedup (VERIFY hacker H-MED-2 / #273):** `loadNode` re-derives
`node_id` from its basis and `content_hash` from the body, and rejects (fail-soft → null) any
file whose name/id/body disagree — the store is not a sandbox. Dedup is FIRST-eligible-wins
(mirrors the W3 FIFO pairing): a re-run with a divergent `reference_divergence` does not
overwrite an existing node (the id is patch-stable; the body varies because `claude -p` has no
seed). The read-back is DEEP-frozen incl. the nested `worked_example_ref` (the read-path
immutability leak that bit the Lab store twice — VERIFY hacker H-HIGH-2; do NOT inherit the
verdict store's write-only freeze).

## 4. The aggregates (the friction map + the judge's own calibration)

- **`aggregateFrictionMap(attempts)`** → harvest every `attempt.resolution_friction`
  (non-null) PAIRED with its stable `(issue_id, attempt_index)` → `clusterFriction(blocks)`
  → re-map each cluster's `members` (which are positional INDICES into the input array, NOT
  blocks — VERIFY arch A-MED-2) back to those stable refs. Returned shape (a contract the
  consumer + test bind to): `{clusters: [{friction_class, friction_phase, detection_leg,
  count, member_refs: [{id, attempt_index}]}], n, n_blocks}`. Report-only; the cluster key
  is the W3 closed-enum tuple, NEVER the free-text `_diagnostic`. Stable refs survive the
  `result.attempts` drop.
- **`computeJudgeAgreement(attempts)`** → over MODEL-decided attempts only
  (`behavioral.outcome_source==='model' && semantic.outcome_source==='model'`), the
  confusion of `tests_consistent` (leg A) vs `semantic.supported` (leg B):
  `{tp, fp, tn, fn, precision, recall, n_model, error_bar:'UNKNOWN-until-measured'}`.
  Below `N_CLEAN_LARGE_MIN`, precision/recall report `INSUFFICIENT-N` (RP-9). This is
  §3.1's "the labeler's error bar is UNKNOWN until same-task agreement is measured" —
  the number is reported, the error bar is honestly unknown, never the borrowed 13%.
