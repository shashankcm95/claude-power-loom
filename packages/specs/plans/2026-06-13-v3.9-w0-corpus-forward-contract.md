## Design

W0 ships ONE pure module + a seed manifest + its test. No network, no `claude -p`, no ContainerAdapter.

### D1 — the corpus record shape + the EXHAUSTIVE THREE-WAY partition (the security boundary)

**The partition is THREE-way, not two** (VERIFY MF-1 — a two-way split throws `unknown-field` on its own required `resolved_at`). Every raw-input key is assigned to exactly ONE class; a key outside the UNION of all three is the hard `unknown-field` throw:

- **`PUBLIC_FIELDS = { id, repo, base_sha, problem_statement }`** — the ONLY fields copied into the object the actor-builder + blind judge receive.
- **`SEALED_FIELDS = { accepted_diff, fail_to_pass, pass_to_pass, test_patch, is_negative_control, repo_familiarity, per_repo_test_strength, repo_review_strictness, rubric_refs, review_thread_ref, criteria_only_rubric, contamination_tier }`** — grader-only oracle. `contamination_tier` is **reserved-sealed, ABSENT at W0** (MF-2 — populated only by W2/W3 model-driven demotion).
- **`METADATA_FIELDS = { resolved_at, perturbation_of, difficulty_bucket, provenance }`** — required grader-side, non-oracle (not actor-public, but not a leak): `resolved_at` (read by the temporal tier), `perturbation_of` (a recall-probe back-link), `difficulty_bucket` (a **COMMITTED** raw field, hand-set from the upstream SWE-bench-Verified label — FP-3: deterministic, NO diff-parse/model), `provenance`.
- **DERIVED (output, NOT a raw-input key)** — `temporal_tier` is computed by W0 and written to the manifest instance; it is **forbidden on raw input** (throws if present).

`splitRecord(raw) -> { public, sealed, metadata }` is a **WHITELIST-COPY, never a spread/delete** (MF-3 — the load-bearing security direction): `for (const k of PUBLIC_FIELDS) if (Object.prototype.hasOwnProperty.call(raw, k)) pub[k] = raw[k]`. A denylist/`{...raw}`-and-delete leaks any unlisted/aliased/inherited/Symbol sealed key. The `public` output's key-set must EXACTLY equal `PUBLIC_FIELDS` (exact-set, never a subset check — security canon).

`provenance: 'backtest'` is **reserved at W0** (FP-2 — matches the RFC §6 / OQ-6 verdict-record `backtest|live` enum so the reservation actually pre-empts the v3.10 retrofit; reserving `'bootcamp'` would NOT match the consumer and buy nothing). W0 sets it; nothing reads it yet (OQ-7).

### D2 — `validateIssueCorpus(records)` — the frozen forward contract (sibling of `validateCorpus`)

Throw-on-first-violation (mirrors `calibration.js:36`). Invariants:

1. **Shape** — required `PUBLIC_FIELDS` + `METADATA_FIELDS` present + typed; `base_sha` is 40-hex; `resolved_at` is ISO-8601; enums (`repo_familiarity`, `per_repo_test_strength`, `repo_review_strictness`, `difficulty_bucket`) in range. Sealed fields are validated when present; `contamination_tier` MUST be absent at W0.
2. **The unknown-field throw** — any raw key outside `PUBLIC_FIELDS ∪ SEALED_FIELDS ∪ METADATA_FIELDS` **throws `unknown-field: <key>`** (the no-silent-default rule). `temporal_tier` on raw input throws (derived-output, not input). Keys compared by exact `===` via `hasOwnProperty` — NEVER `==`/`.includes`/`String()` (the #273 array-coercion decoy class).
3. **The oracle-leak throw (headline security invariant — direction PINNED, MF-3)** — after `splitRecord`, `Object.keys(public).sort()` MUST deep-equal `PUBLIC_FIELDS.sort()` AND `Object.getOwnPropertySymbols(public).length === 0`, else **throw `oracle-leak`**. (A raw record legitimately HOLDS sealed fields — the throw is about a sealed key reaching the PUBLIC OUTPUT, not about the raw record carrying sealed fields.)
4. **Conditional `fail_to_pass` (EXACT-set sentinel, BN-2)** — non-empty UNLESS `is_negative_control`; valid iff `is_negative_control===true && (fail_to_pass.length===0 || arrayEquals(fail_to_pass, [NEG_CONTROL_SENTINEL]))`. EXACT array equality — `[SENTINEL, realTest]` must NOT pass (never `.includes`). A `NEG_CONTROL_SENTINEL` with `is_negative_control===false` throws (control-only value).
5. **Temporal-tier precondition** — `temporal_tier='clean-pending-probe'` (the W0-derived value, MF-2) REQUIRES `resolved_at > MODEL_CUTOFF` AND `repo_familiarity='novel'`. (`contamination_tier='clean'` is NOT a W0 value — no W0 record holds it.)

### D3 — `assignTemporalTier(resolved_at, cutoff) -> 'clean-pending-probe' | 'grey' | 'stale'`

Axis-1 ONLY (deterministic), the ONLY tier W0 writes (MF-2). `clean-pending-probe` (post-cutoff; the `-pending-probe` suffix is load-bearing — axes 2/4 may DEMOTE it in W2/W3, never promote), `grey` (training-vs-reliable band), `stale` (pre-cutoff). `contamination_tier` (the 4-valued `clean|partial|grey|stale`, where `partial` is **never a W0 value** — it is axis-2/4 model-derived) is the SEPARATE sealed field a later wave populates by demotion; W0 never sets it. `MODEL_CUTOFF` is a single pinned constant tied to OQ-1 (the cutoff is itself error-barred; W0 uses the conservative self-reported Jan-2026 boundary).

### D4 — `computeManifestHash(instances)` + the manifest schema

The committed, content-hashed manifest: `{ manifest_version, model_cutoff, instances: [{ id, repo, base_sha, temporal_tier, difficulty_bucket, is_negative_control }], manifest_hash }`. Order-independence is achieved by **SORTING the instance array by a stable composite key (`id` then `base_sha`) and canonicalizing the SORTED ARRAY** — NEVER by object-keying on `id` (BN-3: a map-by-id launders a duplicate `[A,A] -> [A]` and last-write-wins an id-collision). A **duplicate `id` is a HARD THROW** (`duplicate-instance-id`) BEFORE hashing. `computeManifestHash` imports `canonicalJsonSerialize` from `kernel/_lib/canonical-json` (FP-1), **wraps it in try/catch and fails-closed** (`manifest-uncomputable`, mirroring `appendRecord`'s `record-uncomputable` past the depth/node bound — BN-4). The hash EXCLUDES the `manifest_hash` field itself (self-exclusion, INV-22 pattern).

### D5 — `reportStratification(instances) -> { clean_large_n, familiar_large_n, insufficient_n, ... }`

The two-part floor, **REPORTED not ENFORCED** (final-review fold): counts `>1hr` (canonical axis) clean-tier vs familiar-tier large changes, read ONLY from the **committed `difficulty_bucket`** field (FP-3 — no diff-parse, no model; the no-LLM invariant stays checkable); flags `insufficient_n: true` when `clean_large_n < N_CLEAN_LARGE_MIN`. **Never throws, never hard-fails the run** — enforcement waits on the OQ-4 source decision. `N_CLEAN_LARGE_MIN` is a named constant with a `TODO(OQ-4)` comment. Mirrors `calibration.js:152` surface-never-suppress.
