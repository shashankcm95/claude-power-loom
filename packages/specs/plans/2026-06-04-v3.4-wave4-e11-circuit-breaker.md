## Design

### `packages/lab/circuit-breaker/project.js`
- **Constants — env-overridable, clamped with an explicit CEILING (verify-plan CR-3):** the clamp must cap thresholds *below the ledger bound* so a large env can't disable the breaker. `WINDOW_MS` (default 10min; `LOOM_BREAKER_WINDOW_MS`; floor 1000ms, ceiling 24h). `MAX_DENIALS` per-persona (default 5; `LOOM_BREAKER_MAX_DENIALS`; floor 1, **ceiling `MAX_DENIALS_HARD_CAP=50`**). `GLOBAL_MAX_DENIALS` (default 10; `LOOM_BREAKER_GLOBAL_MAX_DENIALS`; floor 1, **ceiling `GLOBAL_MAX_DENIALS_HARD_CAP=200`** — both well below E1's 10k `MAX_LEDGER_RECORDS`, so `env=10000` clamps to the cap, NOT "never fires"). Clamp helper: `clampInt(env, def, lo, hi) = Number.isFinite(env)&&env>0 ? Math.max(lo, Math.min(Math.floor(env), hi)) : def`.
- `isBypassed()` → `process.env.LOOM_DISABLE_CIRCUIT_BREAKER === '1'` (strict — only the exact string `'1'`).
- **`nowMsFrom(opts)` — NaN-guarded (verify-plan CR-2, the E4 precedent at `reputation/project.js:36-45`):** an injected garbage `now` → throw a clear Error BEFORE any read/`toISOString` (else `new Date(NaN).toISOString()` RangeErrors mid-projection). Both entry points use it.
- `projectBreaker({now})`:
  - `bypassed` → return the FULL shape with `{ bypassed:true, generated_at, global:{denials_in_window:0,tripped:false}, personas:[], … }` (all-clear; same keys as non-bypass so a consumer never NPEs).
  - else `listAttestations({now})` → for each record `ts = Date.parse(recorded_at)`: **`!Number.isFinite(ts)` → `excluded_undated++`, skip; `ts > nowMs` → `excluded_future++`, skip (a future-dated/clock-skewed/injected line must NOT inflate the window — verify-plan CR-1, the HIGH); `ts <= nowMs - WINDOW_MS` → outside (too old, silently aged out); else IN-WINDOW.** (The window is the half-open interval `(nowMs - WINDOW_MS, nowMs]` — a denial exactly at `nowMs - WINDOW_MS` is OUT.) Group in-window by `identity.subagent_type` (non-string → `'unknown'`). Per persona `denials_in_window`, `tripped = count >= MAX_DENIALS`. Global `denials_in_window` = total in-window, `tripped = total >= GLOBAL_MAX_DENIALS`.
  - Return `{ generated_at, source:'negative-attestation', label:'denial-rate circuit-breaker (shadow — halts nothing yet)', bypassed:false, window_ms, max_denials, global_max_denials, excluded_undated, excluded_future, global:{denials_in_window,tripped}, personas:[{persona,denials_in_window,tripped}…] }` (personas sorted; only those with ≥1 in-window denial). `excluded_undated` = live (non-expired) records whose `recorded_at` is unparseable — they survive E1's fail-safe expiry (`store.js:124-128` keeps an undatable witness) but can't be windowed, so they are NOT in any denominator (verify-plan CR-7).
- `evaluate({persona, now})` → the consumer decision. **Checks `bypassed` FIRST and returns before any persona lookup (verify-plan CR-4 — else `personas.find(...).tripped` NPEs under bypass):** `{ tripped, scope, global_tripped, persona_tripped, denials_in_window, threshold, window_ms }`. `scope` order (global supersedes — verify-plan CR-6): `bypassed` → `global` (if global.tripped) → `persona` (if the named persona tripped) → `clear`. `tripped = (NOT bypassed) AND (global_tripped OR persona_tripped)`. `global_tripped`/`persona_tripped` are reported SEPARATELY (architect A-1) so a future consumer can tell "you specifically tripped" from "the system-wide breaker caught you" without changing the decision. An absent `persona` → the global-only decision (persona arm vacuously false), no throw. **This is the contract a future un-darkening trigger calls — it returns a HALT signal (narrow), never a grant.**
- **Prototype-safety** (E4 VALIDATE lesson): group with a `Map`; output objects built from Map values (no `__proto__`/`toString` persona name can poison the accumulator or the output).
- **Determinism**: pure fn of (E1 ledger, injected now). No writes; one ledger read (already bounded by E1's `readJsonlBounded`). **Two-horizon note (architect A-3):** E11's `window_ms` filters over E1's ALREADY-expiry-filtered live set (30-day expiry) — two INDEPENDENT knobs (the same coincidentally-related-time-constant DRY-trap the recency-decay leaf warns about; do NOT couple them).

### `packages/lab/circuit-breaker/cli.js`
- `show` → `projectBreaker()` as JSON. `check --persona <p>` → `evaluate({persona})` as JSON (exit 0 always — a tripped breaker is a valid state, not a CLI error; the consumer reads `tripped`).

### project.test.js contract
1. empty store → no personas, global not tripped. 2. denials **inside** the window count; denials **older than `window_ms`** do NOT (the sliding reset); **boundary: a denial exactly at `now - WINDOW_MS` is OUT** (half-open `(now-WINDOW_MS, now]`). 3. per-persona trip at `>= MAX_DENIALS`; below → not tripped. 4. global trip at `>= GLOBAL_MAX_DENIALS` across personas. 5. **bypass**: `LOOM_DISABLE_CIRCUIT_BREAKER=1` → all-clear + `bypassed:true` regardless of denials, **same shape keys** as non-bypass. 6. **determinism**: pinned `now` → deep-equal across calls; no ledger write. 7. **★ prototype-safe**: a `__proto__`/`toString` persona name is counted in its own bucket without polluting Object.prototype. 8. **malformed-record fail-soft**: a missing/garbage `recorded_at` → excluded + counted in `excluded_undated` **and NOT in any persona/global denominator**, no throw; a non-string `subagent_type` → `'unknown'` bucket. 9. **env-clamp (bad)**: an absurd `LOOM_BREAKER_MAX_DENIALS` (Infinity/0/neg/NaN/'abc') falls back to the default. 10. `evaluate({persona})`: clear / persona-tripped / global-tripped / bypassed each return the right `scope` + `tripped` + the separate `global_tripped`/`persona_tripped`. **11. ★ future-timestamp (verify-plan CR-1 HIGH):** a `recorded_at` 1ms in the future is excluded + counted in `excluded_future`, does NOT inflate the window (5 future-dated denials for one persona must NOT trip it). **12. ★ env-clamp ceiling (CR-3):** `LOOM_BREAKER_MAX_DENIALS=10000` clamps to `MAX_DENIALS_HARD_CAP` (50), so the breaker still fires (can't be disabled by an absurd-large threshold). **13. ★ bypass NPE (CR-4):** `evaluate({persona:'code-reviewer'})` under bypass returns `{tripped:false, scope:'bypassed'}` WITHOUT throwing. **14. NaN-now (CR-2):** `projectBreaker({now:'garbage'})` / `evaluate({now:NaN})` throw a clear Error before any read.
