## 3. Design (RATIFIED — architect VERIFY folded; both ⚑ forks resolved)

### 3a. Layer + home
- `packages/lab/negative-attestation/` — files carry `// @loom-layer: lab` **as documentation only** (K12
  classifies by PATH, not headers — W1-P3/F1). The `packages/lab/` home is what makes it `lab`.
- **Dependency direction (K12-enforced, by path):** E1 MAY import `packages/runtime/verify/failure-signature.js`
  (lab→runtime, legal) to reuse enum constants in *tests*. NO runtime/kernel file may import E1 (a CI finding).

### 3b. The store — ⚑FORK-1 **RESOLVED: Lab-owned append store** (raw/distilled tiering correct)
- A **Lab-owned append store** at `~/.claude/lab-state/negative-attestations/` (mirrors `self-improve-store`'s
  `~/.claude/` pattern, W1-P4): an append-only JSONL ledger. Rationale (architect): raw witnesses are high-volume
  and read *programmatically* by E2/E4 — NOT the human/agent-recallable surface (that's E3's distilled policy-axioms
  → library). Library-now would pollute the recall graph with one row per failure. Single-responsibility cut: E1 =
  witness ledger; E3 = recallable policy.
- **Store root is a named constant resolved ONCE** (mirror `registry.js` `STORE_PATH`), never per-call string concat
  — path-traversal hygiene (F7). `pruneExpired` rewrites ONLY this ledger; it never touches `agent-identities.json`
  or any kernel store (a lab→runtime *write* would cross into E4 state — F7).
- **Concurrency (F9):** reuse `_lib/atomic-write.js` (`writeAtomic`) + `_lib/lock.js` (`withLock`). `human_message`
  is free-form/unbounded (ADR-0015 minLength 1, no max) → a raw `appendFileSync` line can exceed PIPE_BUF (512 on
  Darwin) and interleave under concurrency → **lock the append OR atomic-rename the whole ledger** (do not raw-append).

### 3c. The record shape — **E4's input contract; FROZEN per architect VERIFY (F2/F4/F5)**
```jsonc
{
  "attestation_id": "sha256(run_id + '\\0' + leaf_ref)",  // STABLE EVENT id (F2): replay-idempotency key ONLY.
                                                          //   leaf_ref = decompose-run rejected[].id (the consumer-
                                                          //   visible leaf id — DESIGN-SPIKE: the signature-internal
                                                          //   leaf_ref is OFTEN null, so key on rejected[].id).
                                                          //   id unavailable/null → omit dedup, append-always.
  "schema_version": "v3.3",
  "failure_signature": { /* the ADR-0015 8 fields, VERBATIM — no re-validation (producer fail-closed, W1-P1) */ },
  "identity": {
    "subagent_type": "<bare-agentType>", // DESIGN-SPIKE: bare agentType (e.g. 'code-reviewer'), NOT numbered-roster
                                          //   ('04-architect') — a depth-1 spawn has NO identity env-injection
                                          //   (v3-entry-probes.md:74); it knows only its bare type. Numbered grain
                                          //   needs root-side prompt injection → deferred to E4-era/build-team spawns.
    "task_signature": "<tag|null>",      // F4: E4's per-task-type BUCKET key (drives bucketTaskComplexity); null → 'standard'
    "tags": ["…"]                        // E2 scope-derivation (Jaccard)
  },
  "run_id": "<decompose-run runId>",     // event-id component + provenance
  "recorded_at": "<ISO-8601>",           // F3/FORK-2: wall-clock expiry basis AND E4's EWMA decay timestamp
  "expires_after_days": <int>            // OQ-9; default + per-failure-mode override map
  // NB1: failed_criterion_id (inside failure_signature) is the future-E4 per-quality-axis key (F4) — E4's
  //      existing axes are reviewer metrics, NOT R9 criteria. Carried once in the signature; consumed as the axis.
  // NB2 (R1 — coherence-pass must-fix): verifier_kind (inside failure_signature) is the MEASURED-vs-DECLARED
  //      stratifier. R9 is a DECLARATION-conformance gate, not a measured-property gate (leaf-criteria.js:13-23):
  //      only `test-run` (tdd → R12) is a measured failure; `structural`/`predicate` are DECLARATION defects
  //      ("forgot output_schema"), NOT "did bad work." E1 stores verbatim (so verifier_kind IS preserved) — the
  //      contract REQUIREMENT is that the future E4 stratifies on it and never conflates the two (Axiom 1: agent
  //      self-reports aren't authoritative). Documented here so v3.4's E4 can't silently flatten it.
}
```
- **ACCUMULATE, do not dedup (THE DEDUP DECISION, F2):** two DISTINCT failures with identical `(signature, identity)`
  but different `(run_id, leaf_ref)` → BOTH persist (E4 needs the frequency signal — 10 fails > 1). The ONLY dedup is
  replay-idempotency: a re-record of the SAME `(run_id, leaf_ref)` → `{deduped:true}`, no new row. Null `leaf_ref`
  (ADR-0015 allows it) → cannot form a stable id → **append-always** (a possible dup is far safer than dropping a real
  failure). The old `…+seq` key is DELETED (it never dedups + invites a frequency-collapsing "fix").
- Stored **verbatim**; E1 re-validates only its OWN wrapper (identity present, `recorded_at` an ISO string) — Lab-edge
  boundary discipline. Forward-tolerant (INV-FS-AppendOnlyEnums): a reserved/dormant member stores + lists fine.

### 3d. Expiry — ⚑FORK-2 **RESOLVED: wall-clock**
- **`expires_after_days`** (default e.g. 30 to match E4's `RECENCY_HALF_LIFE_DAYS`; per-failure-mode override map).
  Prune (on-write + on-demand CLI) drops records where `now − recorded_at > expires_after_days`. The fake
  `recorded_spawn_seq` counter is DELETED (architect F3: it counted *failures* not *spawns*, and was store-global →
  one persona's failure rate expired another's witnesses).
- **`INV-E1-LedgerNotReputationSourceOfTruth` (F6 — named, Wave-3-checkable):** E1's ledger is NOT the source of
  truth for reputation. E4 ingests **incrementally** at record-time and owns its own decayed aggregate. Any E4
  recompute-from-`listAttestations()` is INCORRECT-by-construction once expiry is live (pruned records would vanish
  from the recompute → silent signal loss). Expiry bounds the *ledger*; reputation decays *separately* (both on the
  SAME wall-clock axis now — commensurable).

### 3e. API surface (Lab-internal, consumed by E2/E4)
- `recordAttestation(signature, identity, opts)` → the frozen record, or `{deduped:true}` on a same-event replay.
- `listAttestations(filter)` → live (non-expired) records — E2/E4 read here.
- `pruneExpired(now)` → drop expired (wall-clock); returns count. Ledger-local only.
- A thin CLI (`record`/`list`/`prune`/`stats`) mirroring `self-improve-store`, as the dogfood vehicle.

   (`spawn-verify.js:30-34`), so the producer never emits a `failure_signature` carrying it → no attestation is ever
   created for it. Asserts the "zero-record member = absence of signal, not signal" invariant (OQ-FS2 / ADR-0015).
8. **Layer-boundary test:** `layer-boundary-lint` over `packages/lab/negative-attestation/` → 0 new findings
   (path-classified `lab`; only-inward imports hold).
9. **Prune-containment test (F7):** `pruneExpired` rewrites ONLY the ledger under `~/.claude/lab-state/…` (resolved-
