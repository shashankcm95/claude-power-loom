## Design (VERIFY-folded) — `packages/kernel/egress/scrub.js` + emitPR gates

1. **`scrubEmitDiff(text)` — the coarse redactor (PURE; defense-in-depth).** Compose
   `getCanonicalSecretClasses()` + `getScrubberOnlyClasses()` (mint-fresh; NO re-literal) → `replace(re,'[REDACTED]')`.
   Two **NET-NEW** passes (no reuse precedent — pinned params, FP-tolerant by design):
   - **base64-decode-rescan:** for each base64 run of length **≥ 40** (`[A-Za-z0-9+/=_-]`, incl. url-safe),
     decode; if the decoded bytes match a canonical class, redact the **ENCODED run** (never the decoded
     text — that would corrupt the diff). **SINGLE pass** (KISS); double/nested-base64 + line-wrap-split +
     misaligned-window are the NAMED residual.
   - **high-entropy:** redact a token of length **≥ 32** with Shannon entropy **≥ 4.0 bits/char** over the
     charset `[A-Za-z0-9+/=_-]`. **FP-tolerance is by design** (over-redaction in a defense net is safe — the
     scrubber-only precedent says so) BUT to avoid corrupting legitimate diffs the entropy pass scans **only
     added (`+`) content lines** and **skips structural lines** (`diff --git` / `index` / `@@` / `+++` / `---`).
   Pure: returns a NEW string, no I/O, no clock; fresh-mint every regex; `.replace` global hygiene (no
   stateful `lastIndex` reuse).
2. **Scrub the WHOLE artifact body (hacker HIGH).** `emitPR` computes `scrubbedDiff = scrubEmitDiff(data.diff)`
   FIRST, then derives `touched_paths` from the **scrubbed** diff (single source of truth — a secret in a
   FILENAME is redacted in the path too) and builds a **freshly-frozen** draft with the scrubbed diff. The
   egress-path-scope (PR-A) already validated the RAW paths at entry; redaction does not alter `+++ b/`
   headers, so the path set is stable. INVARIANT: the PR title/body/diff derive SOLELY from the scrubbed
   bounded diff + the custody-validated repo/issueRef — never an actor stdout/log/trace field.
3. **Per-window cap + 429 backpressure — custody state via opts (architect+hacker HIGH).** A
   **`custodyCapStatePath`** (threaded through opts EXACTLY like `custodyTokenPath`; never from `data`) holds
   the rolling-window counter. The cap is **GLOBAL** (counts ALL emits in the window — varying the
   (repo,issue) key cannot multiply the budget). Over-cap → fail-closed refuse. The 429-backpressure
   threshold (default **3 consecutive** in-window) halts further attempts (consumed at ③.2.3; this wave the
   counter + threshold logic are unit-tested with an injected clock + injected responses). The
   counter read-modify-write runs **INSIDE the existing `withLockSoft` critical section** (shared state).
4. **Etiquette gate — one PR per issue, CANONICAL key (hacker HIGH).** A **`custodyEtiquetteLedgerPath`**
   (via opts; custody) holds the set of already-emitted keys. The key is **canonicalized BEFORE lookup/insert**:
   `lower(owner)/lower(repo without trailing .git)` + the integer issueRef the draft already computes
   (`Number(String(issueRef).replace(/^#/,''))`) — so `Owner/Repo.git` + `#7` and `owner/repo` + `7` collapse
   to ONE key (no bypass-by-casing). A 2nd emit to the same canonical key → fail-closed refuse; inside the lock.
5. **Poisoned-data rejection extends to the new policy vocabulary (hacker LOW).** Add the cap/window/ledger
   keys (`cap`,`window`,`windowCount`,`windowStart`,`emitCount`,`ledger`,`backpressure`,`rateLimit`,
   `perWindowCap`,`emittedKeys`,`seen`) to `DISPOSITION_KEYS` so a poisoned `data` is fail-closed REJECTED,
   not silently ignored.
6. **Honest residual (carried — HUMAN review is the backstop, NOT the scrub).** A SPLIT/novel/double-base64/
   url-safe/line-wrap-misaligned secret survives the scrub — bounded by Path-1 **human review** (and this
   wave, `armedEmit` throws so nothing emits regardless). The fixtures prove only the ANTICIPATED classes;
   custody is the control for the TOKEN, the scrub is a coarse net for the BODY.

## Exit criteria (INTERNAL — no network this wave; armedEmit still throws)

- **EC1bB.1** — `scrubEmitDiff` redacts: a `github_pat_…`, an `AIza…` (canonical reuse); a base64-encoded
  canonical secret (≥40-run decode-rescan, ENCODED run redacted); a ≥32-char ≥4.0-entropy NON-hash token on
  a `+` line. **FALSE-POSITIVE corpus is byte-unchanged:** an ADDED `package-lock.json` `sha512-` integrity +
  an added git sha (the `isBenignHash` skip — git sha / npm-yarn-SRI / go.sum / hex digest), plus the
  structural `index <sha>` line + context/removed lines. **CORRUPTION RESIDUAL:** a NOVEL non-hash
  high-entropy `+` token (a non-standard integrity, a random added blob) may be over-redacted to
  `[REDACTED-ENTROPY]` — FP-tolerant, bounded by human review of the DRAFT (and `armedEmit` throws). The
  split/double-base64/url-safe SECRET cases are the NAMED security residual (not asserted closed).
- **EC1bB.2 (wire-point)** — `draft.diff` is the OUTPUT of `scrubEmitDiff` (not the raw input); `touched_paths`
  derive from the scrubbed diff (a secret-in-filename is redacted in the path). Proves the WIRE-POINT, not
  exhaustive secret-freedom (see EC1bB.1's residual).
- **EC1bB.3** — the GLOBAL per-window cap refuses an over-cap emit (injected clock); the 429 threshold halts
  (injected responses); a poisoned `data` cannot pre-seed/clear/bypass the cap (the state path is custody-only).
- **EC1bB.4** — the etiquette gate refuses a 2nd CANONICAL `(repo,issue)` emit (`Owner/Repo.git#7` == `owner/repo`+7);
  the ledger path is custody-only; a poisoned `data` policy key is fail-closed REJECTED.

## VALIDATE result — PR-B (2026-06-21 board, folded)

  `isBenignHash` (git sha / npm-yarn-SRI / go.sum / hex digest are skipped) — the added integrity is now
  byte-unchanged (regression-tested). This resolved the reviewer+honesty MEDIUM doc-overclaim at the same time.
- **LOW hygiene:** `writeAtomic` collision-proof temp name + orphan-unlink on rename failure; `isRateLimited`
  made private; `DISPOSITION_KEYS` extended with the custody opts-path names; the `secret-patterns.js`
