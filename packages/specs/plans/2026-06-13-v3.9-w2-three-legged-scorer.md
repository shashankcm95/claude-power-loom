## Design

### D1 — the pure scorer `calibration-issue.js` (the calibration.js analog; injected legs; CI-safe)

Mirrors calibration.js's three functions, swapping the three RFC §2 strategy points:

1. **Corpus validator** → reuse `corpus.js` `validateIssueCorpus` (W0-frozen; do not re-roll).
2. **Input builder** (replaces `edgeOf`) → `buildActorInput(record) = splitRecord(record).public` — the actor/leg-B see ONLY public fields; `accepted_diff`/`fail_to_pass`/`test_patch` are STRUCTURALLY absent from that object (anti-anchor invariant).
3. **Verdict classifier** → `{supported}` widens to a **3-axis never-blended grade** `{behavioral, semantic, reference}` (+ a reserved `trajectory:null` slot W3 fills). **NO function ever averages/sums the axes into one number** (the firewall — mirrors calibration.js's `harness_fallback`-distinct-from-true-negative).

Injected leg seams (so the pure scorer is TDD'd CI-safe with MOCKS, exactly like calibration.js injects `judgeFn`):
```
scoreAttempt(record, candidate_patch, { behavioralFn, semanticFn, referenceFn }) -> {
  id, attempt_index,
  behavioral: { verdict: BEHAVIORAL_PASS|PARTIAL|FAIL, tests_consistent: bool, outcome_source: model|harness_fallback, tamper_flags[] },
  semantic:   { status, supported: bool|null, outcome_source: model|harness_fallback, self_graded_optimistic: true },
  reference:  null,                 // leg C runs AFTER A+B, ONLY for recall_eligible attempts (computed by the aggregate)
  trajectory: null,                 // reserved — W3
  recall_eligible: bool,            // FAIL-CLOSED 4-conjunct (R3/AF-HIGH/H1/M1) — see below
}
```
**`recall_eligible` is a FAIL-CLOSED, no-truthiness, exact-equality gate (VERIFY-hacker H1+M1; the project's exact-set/`=== true` security canon):**
```
recall_eligible =
     behavioral.verdict === 'BEHAVIORAL_PASS'
  && behavioral.outcome_source === 'model'      // a harness_fallback "pass" is NOT a pass
  && semantic.outcome_source === 'model'        // a judge-unavailable affirmative does NOT gate
  && semantic.supported === true                // strict ===, never truthiness over bool|null
  && record.is_negative_control !== true        // a no-bug control yields a VACUOUS resolved:true — never recall
```
A behavioral-only pass NEVER reaches recall; a `harness_fallback` on EITHER leg → not eligible; a negative control → never eligible.

- **LEG A behavioral (`behavioralFn`, IMPURE — wraps W1's backend lifecycle + `evaluateOutcome`).** Pinned contract (VERIFY-arch F2): `behavioralFn(record, candidate_patch) -> { issue_tests: PASS|FAIL|FALLBACK, full_suite: PASS|FAIL|FALLBACK|SKIPPED, test_tree_mutated: bool, outcome_source: model|harness_fallback }`. The three-valued verdict is derived in **`scoreAttempt` (the pure aggregate), NOT in `behavioralFn`**: `BEHAVIORAL_PASS` = issue_tests PASS + pass_to_pass held + full_suite PASS + `test_tree_mutated` false; `BEHAVIORAL_PARTIAL` = issue tests pass but full_suite fails OR **leg B flags a discrepancy** (a pure CROSS-LEG combine in the aggregate — leg A NEVER sees leg B's verdict, preserving the blind-firewall direction; this is NOT a blend into one number); `BEHAVIORAL_FAIL` else (incl. the D2 tamper forces). A `refused`/non-`CONTAINED_RESULT`/`KILLED_FOR_DOS` → `outcome_source: harness_fallback` (A2), NOT a FAIL. **Field is `tests_consistent`, NEVER `correct`** (leaky proxy — two-sided error). A negative-control (`is_negative_control`) yielding `BEHAVIORAL_PASS` is a FALSE-POSITIVE (`evaluateOutcome([])` is vacuously `resolved:true`), surfaced as the false-positive rate.
- **LEG B semantic (`semanticFn` — blind, bias-hardened).** Reads ONLY `buildActorInput(record)` (= `splitRecord(record).public`, the 4 PUBLIC_FIELDS) + the candidate patch; `accepted_diff` is structurally absent. Advisory-capped via `RUNG2_MAX_STATUS`. Per-criterion rubric (not "is this good?"). **`criteria_only_rubric` enrichment is guarded by a CONSUME-TIME leak-tripwire (VERIFY-hacker H2):** before passing the rubric to leg B, scan its serialized values for substring overlap against the grader-side `accepted_diff`; on a hit, DROP the rubric for that instance (enrichment-disabled, not a silent pass) and increment a `rubric_leak_dropped` count. (The §3.7 mining-side tripwire is deferred; W2 must guard at the consumption boundary or the leak the RFC calls "closed" is open here.) A timeout/parse-fail/unavailable → `outcome_source: harness_fallback` (A2). Always tagged `self_graded_optimistic` (same Claude family → wider error bars).
- **LEG C reference-as-teaching (`referenceFn` — separate pass, NEVER a gate; WRITE-ONLY).** Runs ONLY after A+B AND ONLY for `recall_eligible` attempts. MAY read `accepted_diff` to compute an informational `reference_divergence` + a worked-example object (the named `WORKED_EXAMPLE_FIELDS` contract — the W2→W4 handoff, VERIFY-arch F6): `{issue_id, repo, problem_statement_digest, candidate_patch_ref, behavioral_verdict, reference_divergence, contamination_tier}` (retrieval-flavored — `worked_example_ref`, never `learned_weight`). **Leg C MUST NOT mutate `record`** (VERIFY-hacker M2 — the `k`-attempt loop shares the record; `buildActorInput` is re-derived per attempt from the immutable record, so a leg-C write-back can never reach a later attempt's leg B). A distinct field that can NEVER be summed into a verdict. (The recall-graph WRITE is W4; W2 computes the eligible object only.)

### D2 — tamper-resistance (R3 reward-hack — the LOAD-BEARING control is the test-tree REHASH, not a path denylist; VERIFY-hacker C1, CRITICAL)

The behavioral oracle is reward-hackable: the candidate is applied BEFORE tests run (W1 `applyPatch candidate` then `applyPatch test`), so it is arbitrary code with collection-time + import-time reach, AND the result sink (`parseTestStatus`) trusts the child's stdout. A path-string denylist is **bypassable** (the hacker proved live: a pure `git mv` into `tests/` emits ZERO `+++ b/` lines; `pyproject.toml`/`setup.cfg`/`tox.ini` `addopts`/`--ignore`, a `.pth`/`sitecustomize.py`, a `conftest.py` at any depth, an `__init__.py` under a test dir all evade a `+++ b/` + test-dir-regex parse). So the control hierarchy is:

- **PRIMARY (load-bearing) — the test-tree rehash.** The IMPURE `behavioralFn` (composing the backend's `prepareClone`/`applyPatch`/`runTests` lifecycle directly so it owns the mid-run hook — NOT `ContainerAdapter.run`, which applies both patches internally) hashes the **collection-relevant tree** (the test dirs + every collection-config file below) AFTER applying the candidate and BEFORE applying the UNMODIFIED `test_patch`; any delta → `test_tree_mutated:true`. The PURE scorer treats **`test_tree_mutated ?? true` (FAIL-CLOSED — an absent signal IS a tamper)** as a forced `BEHAVIORAL_FAIL` + `tamper_flags:['test-tree-mutated']`. This is the only control that is not a denylist; it closes the open-ended surface.
- **SECONDARY (advisory flag, hardened) — `parsePatchTouchedPaths(candidate_patch)`** — parse `+++ b/<path>` **AND** `rename to <path>` / `copy to <path>` / `diff --git a/… b/<path>` headers (a pure rename emits no `+++`); an **unparseable hunk header → fail-CLOSED `BEHAVIORAL_FAIL`** (never a silent empty touched-set). Flag `tamper_flags:['touches-test-infra']` if any touched path matches the test-dir heuristic OR the **collection-config denylist**: `pytest.ini`, `setup.cfg`, `tox.ini`, `pyproject.toml`, `*.pth`, `sitecustomize.py`, `usercustomize.py`, `conftest.py` (any depth), `**/__init__.py` under a test dir. (Advisory — the rehash is the gate; this sharpens the diagnostic + catches the cheap cases pre-execution.)
- **`flagsTestVectorConstant` — DEFERRED (AS-BUILT, VALIDATE-reviewer):** NOT implemented in W2. The test-tree rehash + the now-force-failing `touches-test-infra` gate subsume the practical vector-constant attack (a candidate embedding a held-out constant still cannot mutate the graded tests); a held-out-vector check belongs with the corpus-ingestion surface (which supplies the vectors) — folded into the same deferred residual as test-input special-casing.
- `tests_consistent` stays **necessary-not-sufficient** — an affirmative leg B is required for recall (D1 `recall_eligible`).

**Documented residual (NOT closed by W2; VERIFY-hacker C1.4):** a candidate that special-cases the exact test INPUTS without touching any infra (or injects a collection hook that forges a `__LOOM_TEST_RESULT__` line) defeats a behavioral pass undetectably — the irreducible leaky-oracle gap. It is WHY `tests_consistent` is necessary-not-sufficient and WHY leg B AFFIRMATIVELY gates recall; full closure is held-out-vector / test-input-mutation, deferred (RFC "run a held-out vector where available"). Named as residual, never implied closed by the path-parse.

### D3 — the aggregate (`scoreIssueCalibration` — the scoreCalibration analog)

`scoreIssueCalibration(records, attemptsPerIssue, legs, { tierOf } = {}) → result`:
- Validate via `validateIssueCorpus` (raw records — `contamination_tier` is FORBIDDEN on raw input per the W0 frozen contract). For each record run `k = attemptsPerIssue` attempts through `scoreAttempt`.
- **pass@k** per issue via the numerically-STABLE unbiased estimator `1 − Π_{i=0..k−1}(n−c−i)/(n−i)` (the HumanEval form — NOT literal `C()` binomials, which overflow on large n), over the **model-decided attempts ONLY**: `n` and `c` EXCLUDE `harness_fallback` attempts (a sandbox refusal is not a model miss — the A2 carry-over from calibration.js; the excluded count is reported as `behavioral_fallbacks`, VERIFY-arch F4). `c` = count of `BEHAVIORAL_PASS` among the `n` model-decided. Report the distribution + a corpus-level pass@k, **never a point pass@1**.
- **Per-axis, never blended:** behavioral pass@k; semantic affirmative-rate (advisory); the A2 `harness_fallbacks` tally split per leg (behavioral-adapter-refused vs semantic-judge-unavailable, never summed into quality); negative-control false-positive rate; `recall_eligible` count; `rubric_leak_dropped` count. **NO top-level scalar `score`/`grade`/`overall` key exists** — a RED test asserts its absence (the never-blend firewall's regression guard, VERIFY-arch F3).
- **Stratification + floor (REPORTED not enforced, W0 contract):** the `contamination_tier` is taken from an injected `tierOf(record)` (manifest-/demotion-derived — NOT a raw-record field the W0 validator forbids; VERIFY-arch F1); absent → a single `unknown` bucket. Group pass@k by tier (`clean` headline vs `familiar` GAP arm, never summed); below the committed `N_clean_large` → `INSUFFICIENT-N` (the calibration.js:152 low-sample-never-suppress pattern), never a headline pass@k.
- The result carries the **manifest hash** (reproducibility) and an explicit `not_a_trust_score` provenance note.

### D4 — the impure runner `calibration-issue-run.js` (real legs; OUT of the unit glob; manual spike)

Mirrors `calibration-run.js`. Holds the real leg implementations + `runIssueCalibration` (drives `scoreIssueCalibration` over the real legs + writes the un-gated record):
- **Real leg A** = `selectBackend()` + `new ContainerAdapter({backend})` (W1); `behavioralFn` wraps `run()` + `evaluateOutcome` over the sealed `fail_to_pass`/`pass_to_pass`; fail-closed to `harness_fallback` on refuse.
- **Real leg B** = a blind `claude -p` judge (the `claudePJudge` pattern: untrusted bytes as ONE argv element, shell:false, fail-closed parse → `fallback_reason`); prompt carries ONLY the public input + candidate; **accepted_diff is never in the prompt**.
- **Real leg C** = a teaching `claude -p` pass that MAY include `accepted_diff`, runs in a SEPARATE process/context, WRITE-ONLY to the (W4) recall store, never returns to leg B.
- Writes `rung2-calibration-record/v1`-style record (no evidence-link precondition — Path-1). macOS-only (leg A needs sandbox-exec); manual spike, not CI.
