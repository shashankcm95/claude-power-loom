## P7 RESOLVED (was the open design question) -> MIRROR

A hardening signal is an OUTCOME about a worked-example node, NOT a peer-review verdict. The
verdict-attestation record schema flatly rejects it (mandatory `agentId`/`verifier`/`verifier.kind`;
`projectReputation` drops any record with `transaction_id==null`) AND has no `source` field — so reusing it
would (a) semantically force a square peg and (b) leave the OQ-NS-6 firewall homeless. **Decision: the
consumer MIRRORS the reputation-projection SHAPE over its OWN `hardening-signal-store`.** The projection MATH
is the live lane's discipline (stateless-recompute, per-persona accumulator, `computeRecencyDecayAt` as a
DISPLAY scalar) but operates on the isolated signal store, where the firewall CAN be enforced.

### The `built_by` adapter (C2 — explicit, was an undeclared bridge)

`cmdAssign` outputs `{persona:'99-test-probe', name:'t1', identity, synthid, ...}` — NONE of `role`/
`roster_name`/`actor_kind`. `built_by` REQUIRES `{role,roster_name,actor_kind}` (token-legal
`/^[a-z][a-z0-9-]{0,30}$/`) or THROWS. The adapter (shown, probed at P-C2): registry persona key
`99-test-probe` (NN- convention, legal as a roster KEY) -> `built_by = { role: 'test-probe' (the NN- prefix
stripped, token-legal), roster_name: name (`t1`, legal), actor_kind: 'agent_spawn' }`. **`built_by` is
DERIVED from the real assign output via this explicit adapter — NOT "the raw assigned identity"** (the shapes
differ). Honest framing: the IDENTITY is real (a real `assign`); the TAG is a declared projection of it.

### The recalibration RULE (FROZEN-candidate; the mechanics we lock)

Unweighted Beta-style support ratio (architect HIGH — "recency-weighted" was a conflation;
`computeRecencyDecayAt` is a mean SCALAR, not a per-signal weight): `posterior = (1 + n_support) / (2 +
n_total)` (prior 0.5 via alpha0=beta0=1). **STATELESS-recompute** (deterministic given `(signals, now)` —
the property E1 tests; matches `projectReputation`). Edges defined as assertions: `n_total==0 -> exactly
0.5`; all-refute -> `1/(2+n)` (a damped floor, NEVER 0). `computeRecencyDecayAt` is carried as a DISPLAY
scalar ONLY (NOT a support weight), via the `recorded_at -> {ts}` adapter (E8). Per-signal recency-weighting
is YAGNI for a solo W1 — if ever wanted, it is its OWN function, explicitly NOT "mirroring projectReputation".
