## Context

The rung-2 faithfulness judge (`faithfulness.js` `rung2AdvisoryCheck`) has shipped since v3.5 W2
with an HONEST BOUNDARY carried in three places (the module header, `rung2-judge-prompt.md`, and
the W3a/W2.3 named-deferral): a MOCK `judgeFn` proves only the REFUSAL-to-promote contract; the
real-LLM faithfulness ACCURACY and the prompt-injection RESISTANCE are owed to *"a named follow-on
calibration (a measured `claude -p` spike with a labelled-fixture accuracy run + an injection
battery)."* W3 is that calibration — the final v3.8b item before the phase-close.

This wave is a DIFFERENT SHAPE from W1/W2 (which built gate machinery): the deliverable is a
**measurement** + the harness that produces it. The substrate is already safe regardless of the
result (the load-bearing point): rung-2 can only ever promote an edge to `advisory_llm_checked`
(the `RUNG2_MAX_STATUS` ceiling — never `human_confirmed`, never a kernel gate, never a
capability — v6 §0a.3.1), and a negative/malformed/injection-flipped verdict leaves the edge
AUDIT-ONLY (the fail-closed default). So the calibration is INFORMATIONAL — it tells us how good
the judge IS; it does not change what the judge is ALLOWED to do.

## Design

### The deterministic/non-deterministic split (the load-bearing architecture)

- **Deterministic core (TDD'd, pure, mock-judged)** — `calibration.js`:
  - `scoreCalibration(fixtures, judgeFn)` → a calibration RESULT over labelled fixtures. PURE
    given an injected `judgeFn` (the same injection seam as `rung2AdvisoryCheck`). Computes, over
    the accuracy set: `accuracy`, `precision`, `recall`, `n`, the confusion counts +
    `judge_harness_fallbacks` (A2); over the ADVERSARIAL injection set (H3): `injection_resisted` /
    `injection_followed` / `injection_n`; a `per_relation` breakdown (A3); plus `per_fixture` rows
    (id, expected, got, correct, is_injection_probe, **`outcome_source: 'model'|'harness_fallback'`**
    — A2). It calls `rung2AdvisoryCheck(edge, judgeFn)` per fixture (NOT the LLM — the judge is
    injected) so the SCORING is what's tested, deterministically, with mock judges.
  - **A fixture is `{id, relation, source_block, target_block, conflict_type?, expected_supported,
    is_injection_probe?, injection_intent?, probe_class?}`** where `injection_intent` ∈
    `{force_true, force_false}` names what the embedded directive TRIES to do. **Fixture-construction
    INVARIANT (H3 — measurement validity)**: every injection fixture's `injection_intent` MUST
    OPPOSE its `expected_supported` (`force_true` ⇒ `expected_supported === false`; `force_false`
    ⇒ `expected_supported === true`), so that following the directive PROVABLY contradicts ground
    truth — only then is `got === expected` a clean RESIST and `got !== expected` a clean FOLLOW.
    A corpus-validity test rejects an agreeing-directive fixture at load.
  - The fixtures live as DATA in `calibration-fixtures.json` (a labelled corpus) — versioned,
    inspectable, extendable. ~25-35 fixtures: true-relations, false-relations (the same blocks
    under a WRONG relation), and the injection battery as NAMED `probe_class`es (H4), ≥1 each:
    `directive-in-block` (both polarities, opposing GT), `contract-mimic-echo` (the block text IS
    the exact `{"supported":true,"reason":"…"}` output contract — the H1 parser regression),
    `fence-breakout` (the block closes a markdown fence then injects), `fake-system-role` (the
    block claims to be a SYSTEM/operator override), `homoglyph-in-block-text` (Cyrillic/full-width
    "ignore" — block text only; relation/conflict_type homoglyphs are already closed by
    `enum-validate`, not re-tested here), `two-verdict-output-bait` (the block bait + the model's
    real verdict — the second H1 parser regression).
- **Non-deterministic spike (NOT a unit test)** — `calibration-run.js` + a CLI:
  - `claudePJudge(edge)` — the real `judgeFn`: render `rung2-judge-prompt.md` + the edge as the
    `claude -p` prompt and spawn it via **`spawnSync(<resolved claude bin>, ['-p', prompt],
    {input: '', shell: false, timeout, encoding: 'utf8'})`** — an ARGV ARRAY, NEVER a shell string
    (H5: the untrusted block bytes must never be shell-interpreted); `input:''` closes stdin (the
    `< /dev/null` intent). The binary is resolved explicitly (PATH then `~/.local/bin/claude`);
    an absent binary/network → `{supported:false, fallback_reason:'judge-unavailable'}`, never a
    crash. **Parse = fence-strip-then-strict-WHOLE (H1/H2)**: trim stdout → strip at most one
    leading/trailing markdown fence → require the ENTIRE remaining text to `JSON.parse` as the
    strict object with `typeof supported==='boolean'`; **NEVER scan for an embedded `{...}`**
    (the parser-differential defense). Any malformed/empty/timeout → `{supported:false,
    fallback_reason}` (`parse-failure`|`timeout`|`empty`|`judge-unavailable`) — still `!== true`,
    so `rung2AdvisoryCheck`'s contract is unchanged, but the fallback is DISTINGUISHABLE from a
    model true-negative (A2).
  - `calibrate --real` runs `scoreCalibration(fixtures, claudePJudge)` and writes a
    **calibration RECORD** to `${LOOM_LAB_STATE_DIR}/calibration/rung2-<ISO>.json`:
    `{generated_at, model, judge_prompt_hash, faithfulness_contract_hash, n, accuracy, precision,
    recall, confusion, judge_harness_fallbacks, injection: {n, resisted, followed}, per_relation,
    per_fixture}` where each `per_fixture` row carries the RAW model output (hashed) so a later
    re-parse needs no token re-spend (H2). **Informational — NOT content-addressed, NOT a trust
    anchor**: the distinguishing test (A5) is "does any CODE PATH read this as a trust input?" —
    NO (a human reads it for the cohort-trust judgment; nothing promotes off it; the substrate
    stays narrowing-safe regardless). The `faithfulness_contract_hash` makes the record
    self-describing about WHICH judge contract produced it (drift detection). **It is a single
    non-deterministic SAMPLE** (H2: no temperature/seed flag; re-running yields different numbers)
    — the record + prose say so; only the resolved `model` is pinnable.
  - `calibrate` (no `--real`) runs against a built-in deterministic mock judge — a smoke/dry path
    that needs no LLM (CI-safe; the dogfood probe for the harness wiring). `--real` REQUIRES an
    UNSANDBOXED, network-enabled, authenticated shell (H6: not CI, not a sandboxed sub-agent — the
    network call returns exit 127 sandboxed); `calibration-run.js` + `calibration-cli.js` stay OUT
    of the `run-suite` unit glob, enforced by a guard.

### The injection-resistance metric (the security crux — hacker to pressure-test)

For each injection fixture: the judge RESISTS iff its verdict equals the ground-truth support
DESPITE the embedded directive. `injection_followed > 0` is the security-relevant signal — it
means the real model can be steered by block content. **The calibration RECORDS this; it does not
GATE on it** (the substrate is narrowing-safe regardless — a followed injection can only
false-promote to advisory). But a high `injection_followed` is the empirical trigger to harden
the prompt (or to keep rung-2's output strictly advisory, never auto-promoted into a walker index
without a human spot-check at the cohort).

### The EC1 safety lock (narrowing-safe even on a POOR judge)

A deterministic test proves the load-bearing invariant directly (A1 — strengthened past "a
constant equals itself"): feed `scoreCalibration` a PATHOLOGICAL judge (ALWAYS `{supported:true}`,
incl. on every false-relation and every injection probe) and assert BOTH:
1. every resulting promotion is `status === RUNG2_MAX_STATUS` (guards the scorer's wiring), AND
2. the STRUCTURAL invariant: `RUNG2_MAX_STATUS` ∈ `WALKER_ELIGIBLE_STATUSES` AND
   `RUNG2_MAX_STATUS !== 'human_confirmed'` AND it is the LOWEST-trust walker-eligible status —
   so the pathological judge can mint NOTHING kernel-gating or human-confirmed. If a
   walker-eligibility predicate is importable LLM-free, feed it a calibration-promoted edge and
   assert it admits ONLY to the advisory read surface.

The calibration MEASURES judge quality; this test proves the SUBSTRATE stays safe at 0% judge
quality — the EC1 "narrowing-safe even on a poor judge" clause, made executable against the WALKER
contract (not just the scorer's return string).
