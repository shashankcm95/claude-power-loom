## Wave Scope Decision (resolves a documented doc-inconsistency)

The notes carried two orderings: spike "(1) record (2) E4 (3) A6" vs MEMORY "Wave 2 = A6 → Wave 3 = E4". **Resolution: record → E4 → A6** (the spike ordering), because:
- **E4-before-A6 is consumer-coherent.** E4 (Wave 2) reads records that exist from Wave 1 and produces an inspectable distribution (display-only). A6-before-E4 would build a snapshot mechanism with nothing to snapshot (an empty-mechanism producer-without-consumer — the very anti-pattern we're avoiding).
- **Producer–Consumer Phasing (USER rule):** a producer must be *planned* to be consumed at the next phase. Wave 1's record → consumed by E4 in Wave 2 (next phase). E4 → consumed by A6 in Wave 3 (next phase). Each link is one-phase-forward. ✓
- **v3.3 lesson honored:** Wave 1 builds **no amplifier** over the new (empty) store. E4 (a simple group-by-persona distribution, not Jaccard/EWMA) waits for Wave 2, by which point real verdict volume can accrue.

**This wave (Wave 1) = the record producer ONLY** (store + enricher + CLI + tests). E4 + A6 + routing-wire are explicitly OUT (see Out of Scope). The in-wave consumer that closes a real shadow loop end-to-end: the **enricher** (verifies each record's link resolves to a kernel record) + the **CLI list** (human/inspect).

## Design

### `store.js` — verdict-emission attestation (the producer)
- `recordVerdict({ verdict, subject, verifier, agentId, runId?, expiresAfterDays?, now? })`
  - **Validation (fail-fast at the boundary):** `verdict ∈ {pass,partial,fail}`; **`agentId` non-empty (REQUIRED — the §0a.3.1 evidence-link; a verdict with no spawn-link is rejected, never stored)**; `verifier.identity` non-empty (who emitted — e.g. `03-code-reviewer.nova` or `orchestrator`); `verifier.kind` non-empty (R1 stratification — `test-run`|`structural`|`advisory`, carried verbatim, never flattened); `subject.persona` non-empty (the judged persona = reputation subject, orchestrator-provided).
  - **id (the MEDIUM-3 fix):** content-addressed via `canonical-json`. Dedupable id = `sha256(canonical([agentId, verifier.identity, verifier.kind, verdict]))` → a true replay (same spawn, same verifier, same kind, same verdict) dedups; **distinct verifiers about one spawn ACCUMULATE** (two reviewers agreeing is stronger evidence — must not collapse); a verifier changing its verdict also accumulates (audit-honest, mirrors E1).
  - **record (frozen) — `subject`/`verifier` split:**
    ```
    { attestation_id, schema_version, verdict,
      subject:  { persona },                         // WHO is judged (E4 groups by this)
      verifier: { identity, kind },                  // WHO emitted + the verification kind (R1)
      evidence_refs: { agent_id (req),               // the orchestrator-formed link
                       run_id: null,                 // enricher-filled
                       transaction_id: null,         // enricher-filled (content-addressed kernel record)
                       record_status: null },        // enricher: appended|deduped|not-appended
      recorded_at, expires_after_days }
    ```
    Stores the **emission attestation** ("verdict V emitted about spawn A's work by verifier W at time T"), never verdict-as-truth.
  - `enrichRecord(attestationId, { runId, transactionId, recordStatus })` (the HIGH-1/F5 persistence path): read-modify-`writeLedger` under `withLabLock`; find by `attestation_id`; replace with a NEW frozen record carrying the filled `evidence_refs`. **NOT** a `recordVerdict` re-call (that would dedup + drop the enrichment). Idempotent (re-enrich → same values).
  - **Disciplines (copied from E1, proven):** `withLabLock` soft-lock (advisory, never `process.exit`); atomic whole-ledger rewrite (no raw append — PIPE_BUF); wall-clock expiry; `MAX_LEDGER_RECORDS` cap; prune-on-write; fail-soft `readLedger`.
  - Ledger: `$LOOM_LAB_STATE_DIR/verdict-attestations/ledger.jsonl`.
- `listVerdicts(opts)`, `pruneExpired(opts)` — same shapes as E1.
- **Imports ONLY** `kernel/_lib` (atomic-write, lock, canonical-json) — containment-clean (the test enforces). `enrichRecord` does NOT read spawn-state — the enricher passes resolved values in (keeps store.js containment-clean).

### `enrich-from-spawn-state.js` — the link resolver (separate module; the in-wave consumer)
- **Frozen data-contract doc block** (F4) at the top of the file — the 4 kernel facts the Lab depends on, each citing the resolver line: (1) base = `LOOM_SPAWN_STATE_DIR || ~/.claude/spawn-state` (`spawn-close-resolver.js:85`); (2) run subdir = `sha256(session_id)[:16]` (`:153-179`); (3) journal filename `resolver-journal-<agentId>.jsonl` (`:203-205`); (4) the journal line shape `{kind:'shadow-provenance-record', spawn_id, transaction_id, record_appended, deduped}` (`:522-541`).
- `resolveKernelRecord(agentId)` — **SYNC** (MEDIUM-4; no `fs/promises.glob` — async would propagate through the whole stack):
  - **path-guard** `agentId` via `isSafePathSegment` PRE-join (defense-in-depth; the #215/C1 trap-class).
  - base = the contract path. Iterate `fs.readdirSync(base)` run subdirs; `fs.existsSync(<base>/<runId>/resolver-journal-<agentId>.jsonl)` per entry — collect hits.
  - parse the hit journal (fail-soft per line); **filter to `kind === 'shadow-provenance-record'`, take the LAST** (F2/HIGH-2). Extract `transaction_id` + `record_appended`/`deduped` → `record_status`. `runId` = the parent dir name.
  - **0 hits OR no `shadow-provenance-record` line** (skipped/error/verdict-only journal) → `null` (fail-soft — not every spawn produces a committed work-record). **2+ file hits** → first, with `collision:true` (detectable; agentId is ~2⁶⁸ so near-impossible, but flagged not silent).
  - returns `{ agentId, runId, transactionId, recordStatus } | null`. (No record-store read, no `writer_persona_id` cross-check — F1: the journal is the sole surface; subject.persona is orchestrator-provided.)
- `enrichLedger({ now? })` — `listVerdicts` the unenriched (`transaction_id===null`); for each, `resolveKernelRecord(agent_id)`; on a hit call `store.enrichRecord(...)`. Returns `{ enriched, unresolved, skipped }`.
- Reads spawn-state by **path only** — never `require()`s a kernel STATE module (containment-safe by construction; its own containment test enforces — MEDIUM-5b).
