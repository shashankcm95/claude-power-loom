## Design

### The cross-layer data contract â€” the snapshot file `<lab-state>/reputation-snapshot.json`
The file IS the K12 boundary (a documented JSON shape, not a module import). **Path resolution is ONE shared function** `resolveSnapshotPath()` (verify-plan CR-HIGH-2 â€” the single highest-risk integration bug is a writer/reader path split-brain â†’ silent permanent reputation-blind). It lives in the kernel `_lib` reader leaf and is imported by the lab materializer, so there is exactly one formula: `LOOM_EVOLUTION_SNAPSHOT_PATH` if set, else `${LOOM_LAB_STATE_DIR || ~/.claude/lab-state}/reputation-snapshot.json`. Shape:
```
{ schema_version:'v1', kind:'evolution-snapshot/reputation', generated_at, source:'verdict-attestation',
  label:'advisory-verdict distribution over kernel-attested spawns â€” NOT a quality score',
  content_hash: sha256(canonicalJsonSerialize(snapshotHashBody(snap))),   // shared basis; EXCLUDES content_hash itself
  watermark: { record_count, max_recorded_at, excluded_unenriched },
  personas: [ <the W2 projectReputation persona entries> ] }
```
`snapshotHashBody(snap)` (shared, in the reader leaf) returns the canonical body to hash â€” `generated_at` IS included (deterministic given injected `now`) but `content_hash` is excluded (no self-reference). Both writer + reader call it â†’ one basis, no drift (verify-plan A-LOW-5, the M1 forward-coupling class).

### `packages/lab/reputation/materialize.js` (off-hot-path producer)
- `materializeSnapshot({ now?, outPath? })`: `projectReputation({now})` â†’ assemble the body â†’ `content_hash = computeSnapshotHash(body)` (the SHARED basis from the reader leaf) â†’ write atomically to `outPath || resolveSnapshotPath()` (`writeAtomicString` = write-tmp **in the same dir** + `rename()`; same-filesystem â†’ POSIX-atomic â†’ the torn-read defense) â†’ return `{ path, content_hash, persona_count, generated_at }`.
- **project.js stays pure** (the v6:509 deterministic-theorem property): the materializer is the ONLY I/O site (SRP â€” a separate file, not a method on the pure projection). Imports `./project` (labâ†’lab) + `kernel/_lib/{evolution-snapshot-read (for resolve+hash), atomic-write}` (labâ†’kernel/_lib).
- **Trigger = on-demand this wave** (`reputation materialize`). Wiring it to session-start or to the W1 batched enricher's tail is the deliberate ACTIVATION step (deferred with breakers â€” design-input (a)). Until then the snapshot is simply absent/stale â†’ the reader fails open â†’ reputation-blind (safe default).

### `packages/kernel/_lib/evolution-snapshot-read.js` (the hot-path reader â€” the 4th extract-to-leaf)
- Exports `resolveSnapshotPath()`, `snapshotHashBody(snap)`, `computeSnapshotHash(snap)`, `readEvolutionSnapshot(pathOrOpts?)`.
- `readEvolutionSnapshot`: resolve path â†’ `statSync` (absent/not-a-file â†’ `{present:false, reason:'absent'}`) â†’ size-guard (`> MAX_SNAPSHOT_BYTES` default 1MB â†’ `{present:false, reason:'oversized'}`) â†’ `readFileSync` + `JSON.parse` (throw â†’ `{present:false, reason:'unparseable'}`) â†’ shape-normalize (missing/garbage fields â†’ `{present:false, reason:'malformed'}`) â†’ **self-verify**: recompute `computeSnapshotHash(parsed)` and compare to the stored `content_hash` (mismatch â†’ `{present:false, reason:'hash-mismatch'}` â€” the INV-22 "never trust a self-asserted hash" discipline; cheap, ~hundreds of nodes) â†’ `{ present:true, content_hash, generated_at, source, watermark, value:<bounded personas>, truncated:false }`.
- **NEVER throws** (the spawn hot path is fail-soft â€” every branch returns an object). **Never reads the ledger** (only the small snapshot file). O(1) w.r.t. attestation volume.
- Reads a file by path â€” imports nothing from lab (K12); imports `kernel/_lib/canonical-json` (kernelâ†’kernel) for the hash. It is the SAME reader the advisory CLI uses (one read path, no drift).
- **TOCTOU note (verify-plan CR-LOW-6, accepted):** the `statSync`â†’`readFileSync` window is within the conceded same-UID local trust boundary (p-writescope); the size-guard is hardening, not a security gate; fail-open + the hash self-verify bound the blast radius. A code comment marks the accepted window. A `__proto__`-named persona key is faithfully hashed (it's a content-address) and normalized without prototype pollution (the project.js null-proto discipline carries â€” verify-plan A-LOW-4).

### `packages/kernel/spawn-state/spawn-record.js` (the kernel record â€” records-not-injects)
- **INSIDE `main()`'s existing `try{` at line 364** (verify-plan CR-MED-3 â€” keep it under the fail-soft catch; NOT buildEnvelope, for purity): `const evolutionSnapshot = readEvolutionSnapshot();` (fail-open, never throws). Pass into `buildEnvelope({ â€¦, evolutionSnapshot })`.
- In `buildEnvelope`: `axioms.evolution_snapshot = { reputation: evolutionSnapshot }` (verify-plan A-MED-3 namespacing â€” leaves `evolution_snapshot.prev_state_hash` free for the deferred INV-23 pin; `reputation:{present:false}` when absent). Byte-cap the inline `value` measured in UTF-8 bytes: `Buffer.byteLength(JSON.stringify(value),'utf8') > MAX_INLINE_SNAPSHOT_BYTES` (64KB) â†’ drop `value`, keep hash+watermark, set `truncated:true` (verify-plan CR-HIGH-1 â€” bytes not UTF-16 code-units, matching the on-disk encoding).
- Hot-path cost: one small-file read; well within <50ms p99. Backfilled duration metric will show it.

### Performance & concurrency boundary (answers the standing question directly)
- **Init / cold-start**: absent snapshot â†’ `{present:false}` recorded â†’ routing proceeds reputation-blind. Never blocks, never computes-on-miss.
- **"Invalidation" is epoch supersession, off the hot path**: a verdict burst does NOT invalidate the immutable snapshot (v6:179); the off-hot-path materializer recomputes + atomically `rename()`s a successor; it applies to spawns *after* the rename (v6:408).
- **The read/write race is closed by atomic rename, not locking**: POSIX rename â†’ a concurrent hot-path reader sees old-complete OR new-complete, never torn. No lock on the budget-critical path.
- **Cross-spawn concurrency is degenerate today** (K13-serial, v6:1632; Pattern-A blocked-orchestrator) â†’ init-snapshot == close-snapshot â†’ recording at PostToolUse-close is sound. Pattern B (v3.5+) needs the orchestrator-carries-envelope carrier â€” out of scope; the content-hash makes any future skew auditable.
- **Staleness = observability, not enforcement**: the `watermark` (record_count, max_recorded_at) is recorded verbatim; nothing acts on it. A freshness SLA, if ever wanted, belongs to the materializer.
