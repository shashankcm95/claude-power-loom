## The closes

### #3 — Unforgeable sentinel (per-run nonce + first-wins) — the headline

- **Generate** a per-run nonce (`crypto.randomBytes` hex) at the GRADER (the trust anchor: `makeBehavioralFn` +
  the `container-adapter.js:240` backend-run path). Thread it: `runTests({workDir, test_ids, nonce})` →
  `resolveTestCommand({workDir, test_ids, nonce})` → `buildPytestCommand({test_ids, nonce})` → the wrapper emits
  `__LOOM_TEST_RESULT__<nonce>{...}`. The grader passes the SAME nonce to `parseTestStatus(stdout, ids, nonce)`.
- **Parse** FIRST-wins + nonce-match: `parseTestStatus` accepts ONLY a line whose prefix is `__LOOM_TEST_RESULT__<nonce>`
  and takes the FIRST such line (later lines ignored). Nonce optional → no nonce = legacy last-wins (spike back-compat;
  the GRADE path always supplies one). Single-source the prefix+nonce contract (no re-typed literal).
- **Why first-wins + nonce defeats the realistic forges:** the legit result line is written right after `pytest.main`;
  an `atexit`/late forge ALWAYS runs after that write → it is the SECOND nonce line → IGNORED. A naive forge (no nonce)
  never matches. **HONEST RESIDUAL (documented, NOT closed):** a candidate that, DURING a test, reads the nonce from
  `/proc/self/cmdline` and prints `__LOOM_TEST_RESULT__<nonce>{forged}` BEFORE the wrapper's write beats first-wins.
  That is a deliberate, diff-visible, in-process attack — bounded by Path-1 human review + the non-adversarial backtest
  subject; FULL closure needs process/privilege isolation (a ContainerAdapter-tier concern, like the R12 residuals), NOT
  achievable while candidate code and the reporter share one process. The nonce is a BAR-RAISE, framed as such — no
  over-claim of "sentinel-forgery closed."

### #6 — Pristine-config grader apply

- The grader's `applyPatch` (`_clone-lifecycle.js:309`, candidate AND test) runs under `-c core.attributesFile=/dev/null`
  (mirror `captureActorDiff:302`), so a smuggled `.gitattributes` filter/eol/ident driver cannot execute on the grader's
  apply. (Close #2's diff-scope already REJECTS a `.gitattributes` touch in the real-solve path; this is the grader-apply
  backstop for the paths without front-line diff-scope.)

### #5 — Git clean-tree restore — PROPOSED DROP (redundant; VERIFY to confirm)

The plan's original close #5 (restore the test tree before the sealed test patch) is **redundant with the shipped
close #1** for the security property: a test-tree-mutating candidate is already FAILed by `test_tree_mutated`, so
restoring-then-running cannot produce a *false PASS* the gate doesn't already prevent. The only delta is a cleaner
FALLBACK→FAIL in the candidate-edits-a-test → test-patch-conflict edge (both NOT-PASS today). The non-hashed
source-import case #5's prose named is the documented proof-of-fix residual a test-infra-scoped restore cannot reach.
**Proposal: DROP #5** (do not build redundant mechanism); if VERIFY surfaces a real no-false-PASS case #1 misses,
re-scope it back in.
