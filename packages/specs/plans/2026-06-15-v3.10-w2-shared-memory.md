# v3.10-W2 — SHARED memory: multi-author attribution on a collided recall node

**Anchor (the USER design, ratified at W1).** "A new test persona + its recall graph; later a second persona
for SHARED memory. Existing personas + the plugin stay uncorrupted; dispose the test personas per round."
W1 built the SOLO consumer (one persona, one node, recalibrate-on-mock-signal). **W2 is the SHARED case:**
a second test persona `t2` builds the SAME worked example as `t1` -> a `node_id` COLLISION (node_id excludes
persona) -> a hardening signal about that SHARED node must attribute to BOTH authors (multi-author merge),
instead of W1's solo guard (credit NOBODY for a collision node).

**The mandate still binds (USER 2026-06-15, [[beta-internal-verification-mandate]]):** the multi-author
machinery is proven INTERNALLY (mock signal + the real kernel+runtime+lab stack on an isolated base). Nothing
HARDENS trust (OQ-NS-6); the mock signal swaps for a real one later with ZERO new machinery validation.

## What W1 left as the seam (the architect's forward-contract, NOT a frozen law)

- **`node_id` excludes persona** (`recall-graph.js:102` `deriveNodeId(workedExampleRef, provenance)` -- TWO
  args; internally it hashes `[workedExampleRef.{issue_id, candidate_patch_ref, repo}, provenance]`, with
  `built_by` added at `recall-graph.js:153` OUTSIDE both the node_id basis (`:141`) and the content_hash
  (`:146`)). Two personas building the same worked example -> the SAME `node_id` (honesty MED: signature fix).
- **`writeNode` collision today (W0', `recall-graph-store.js:89-99`):** first-eligible-WINS persists the FIRST
  author's `built_by`; on a differing 2nd author it RETURNS `{ persona_collision:true, kept_built_by,
  incoming_built_by }` -- but the **2nd author is NEVER persisted** (the collision is return-only, ephemeral).
- **W1 consumer (`recalibrate.js:92`):** a `node_id` in `collisionNodeIds` is DROPPED (credit nobody -- the
  confused-deputy guard). The architect (W1 VERIFY) blessed this as **the seam W2 upgrades, NOT a frozen law**:
  "W1's accumulator must NOT bake in a 1-node-1-persona assumption that makes W2's merge a freeze-reopening."

## The proposed design (for the VERIFY board to pressure-test)

W2 introduces a PERSISTED authorship record so a shared node's authors survive across the two (different-time)
builds, then upgrades the consumer's JOIN from single-author to multi-author.

### 1. An authorship LEDGER (new lab store, mirrors `hardening-signal-store.js`)

`packages/lab/persona-consumer/authorship-store.js` -- a content-addressed `(node_id, built_by)` edge store
under `$LOOM_LAB_STATE_DIR/recall-authorship/` (own dir; own firewall; mirrors the signal store EXACTLY:
verify-on-write AND on-read, dedup first-wins, `opts.dir`, deep-freeze read-back).

- Record: `{ authorship_id, node_id, built_by:{role,roster_name,actor_kind}, recorded_at }`.
- `authorship_id = sha256(canonicalJsonSerialize([node_id, role, roster_name, actor_kind]))` -- the SAME
  (node, author) DEDUPS to one file; DIFFERENT authors on the same node are DISTINCT files (so a shared node
  accumulates one edge per author). **`recorded_at` is TOP-LEVEL, OUTSIDE the `authorship_id` basis** (mirror
  the NODE store's outside-the-hash stamp at `recall-graph-store.js:79-83`, NOT the signal store's IN-hash form
  -- folding it in would break the (node,author) dedup, since a different-time re-record would stop colliding;
  hacker MED). It supports the disposal lifecycle (`retireAuthorship({before})`, the analogue of
  `retireBacktestNodes`) + an audit trail to distinguish a legitimately-recorded edge from a back-dated plant.
- `writeAuthorship` REJECTS: a `node_id` that fails the **STRICT** guard `typeof node_id === 'string' &&
  node_id.length > 0 && HEX64.test(node_id)` -- the SIGNAL store's form (`hardening-signal-store.js:63`), NOT
  `recall-graph-store.js:57`'s `HEX64.test(String(node_id))` (hacker HIGH: `String([NID])` coerces a
  one-element array PAST the recall-graph guard, and that store is saved ONLY by a SEPARATE re-derivation
  source -- `worked_example_ref` -- which the authorship lane LACKS, since `node_id` is itself a direct basis
  input; so a forged `{node_id:[NID]}` would be self-consistent under the loose form). Also REJECTS a `built_by`
  whose `role`/`roster_name` fail `ROSTER_TOKEN` or whose `actor_kind` is not in `ACTOR_KINDS` (reuse
  recall-graph's shapes; closes the garbage/dot-merge class), and an unparseable `recorded_at`.
  `verifyAuthorship` re-derives the id + re-applies the SAME strict guards on READ (read/write parity, the
  CodeRabbit #323 lesson); a forged id / tampered author / coerced node_id -> fail-soft null.
- **The ledger stores `built_by` FIELDS only -- NEVER a precomputed `persona_key` string** (hacker LOW): the
  consumer ALWAYS re-derives the key via `personaKeyOf` from each edge's `built_by` and never trusts a stored
  key (else the `.`-delimiter collision-proofing at `recalibrate.js:48-54` is bypassable). No signal/edge field
  ever PICKS a persona; credit is re-derived.

**DRY arbitration (architect LOW -- the 3rd content-addressed store):** authorship-store is the THIRD
near-duplicate of the content-address-per-record skeleton (after `hardening-signal-store` + `recall-graph-store`).
**Decision: accept the deliberate copy, do NOT extract a factory.** Each store's verify predicate is
DIFFERENT and security-load-bearing (signal: `source==='mock'`; node: `provenance==='backtest'`; authorship:
strict-node_id + roster-shape); a shared base would need injected `derive`/`verify` closures that OBSCURE each
lane's distinct firewall. Independent auditability > DRY here. (Stated so "mirrors 1:1" is not read as
copy-paste-by-default.)

**Why a SEPARATE store (not enrich-the-node, not in-memory):**

- *Enrich-the-node* (append `authors[]` to the persisted node) was REJECTED: the node is immutable
  first-eligible-WINS (`recall-graph-store.js:84-99` "a re-run never overwrites"); appending a 2nd author is a
  read-modify-write MUTATION of a content-addressed artifact (concurrency/atomicity hazard). A separate
  append-only ledger keeps the node store untouched -- the W1 MIRROR philosophy (a new concern -> a new lane).
- *In-memory thread only* (pass an `authorsByNode` map captured from `writeNode` returns, no persistence) was
  REJECTED: "shared memory" is inherently cross-build/cross-session (t1 and t2 build at DIFFERENT times); an
  in-memory map models only a single process run. The mandate wants the PERSISTENCE mechanics proven.

### 2. The producer records an authorship edge per populated (node, author)

The round harness, after each `writeNode(node)`, calls `writeAuthorship({ node_id: node.node_id, built_by:
node.built_by })`. SOLO: 1 node + 1 edge. SHARED: 1 node (first author persisted) + 2 edges (t1, t2). The edge
store dedups, so a re-run is idempotent. (This is harness/producer wiring -- NOT a change to `writeNode`, which
stays immutable; the collision RETURN still LOGS the erasure as before.)

### 3. The consumer JOIN: single-author -> multi-author (additive, back-compat)

`recalibratePersonaReputation(nodes, signals, { now, collisionNodeIds, authorships })` -- `authorships` is a
NEW optional param (the validated ledger records). The consumer builds `authorsByNode` (a **`Map` keyed by the
re-validated STRING `node_id`** -- never a plain object, whose key string-coerces; hacker HIGH) -> a deduped
SET of personaKeys, each re-validated via `personaKeyOf`. Per signal about an existing node, the credited
persona SET is resolved by **the COLLISION-FIRST order (the load-bearing VERIFY fold -- hacker CRITICAL +
architect HIGH):**

1. **if `collisionNodeIds` has the node_id** (a collision was OBSERVED at write -> the node tag holds only the
   FIRST of >=2 real authors, so it is NOT a trustworthy single-author source): credit the ledger set **ONLY
   IF it AFFIRMATIVELY accounts for the collision** -- `authorsByNode.get(node_id)` exists with **>= 2 DISTINCT
   personaKeys**; else **SKIP (credit nobody)**. A collision node with a PARTIAL ledger (0 or 1 edge) stays
   un-attributable -- you cannot prove a lone edge is not a cherry-pick/plant.
2. **else (a genuinely SOLO node):** credit the single `personaKeyOf(node.built_by)` -- the W1 source of truth.
   The ledger is **NOT consulted for solo nodes** (so a planted edge on a solo node is IGNORED).

**Why collision-FIRST (the regression this closes):** the naive presence-first order (`authorsByNode` before
`collisionNodeIds`) let a single PLANTED or PARTIAL ledger edge on a collision node promote attacker-chosen
credit -- INVERTING W1's "credit nobody" guard (W1 `recalibrate.js:92` skips the node_id BEFORE persona
resolution). Collision-first + the >=2-completeness gate makes the collision flag DOMINATE the ledger, so the
guard can only be SATISFIED (full author set present), never OVERRIDDEN. **A legitimate collision always has
>= 2 edges** (the producer records BOTH `kept_built_by` and `incoming_built_by`), so this credits real
co-authors while refusing partial/forged sets.

**SSOT (architect MED):** `node.built_by` is authoritative for SOLO credit; the LEDGER is authoritative for
MULTI-author (collision) credit and consulted ONLY there. The first author is duplicated across node+ledger
INTENTIONALLY + one-directionally (the ledger is never written back to the node; the node never overwrites an
edge) -- a deliberate fallback hierarchy, not two competing truths.

**Source-blindness boundary (honesty MED -- mirror the `signals` NOTE at `recalibrate.js:77-79`):**
`authorships` MUST originate from `authorship-store.listAuthorships` (verify-on-read re-derives `authorship_id`
incl. node_id+built_by). `personaKeyOf` re-validates the token SHAPE only -- NOT edge AUTHENTICITY; a caller
hand-feeding unverified edges bypasses the content-address gate. (Same posture as signals; W3 authenticates credit.)

Each credited persona gets the FULL `+1` support/refute (UNWEIGHTED -- the frozen RULE; splitting across
co-authors is a weighting decision, deliberately NOT taken, YAGNI). **The honest semantic (honesty HIGH): FULL
credit REPLICATES the signal -- one real signal increments EACH co-author's `(s,n)` independently, so the
system-wide support count is NOT conserved; a k-author node lets one signal move k posteriors.** This
replication (not split) is the deliberate unweighted choice; a conservation-preserving split is a separate W3+
function. The per-persona accumulator + the Beta math `(1+s)/(2+n)` are UNCHANGED -- W2 changes only how MANY
personas a signal credits, never the math.

**The frozen RULE is NOT re-opened:** OQ-W1-1 froze the unweighted Beta ratio + stateless-recompute. W2 touches
the JOIN cardinality (the explicitly-blessed seam), not the posterior formula. Each author independently
accumulates its own (s, n); the recompute stays deterministic given `(nodes, signals, authorships, now)`.
