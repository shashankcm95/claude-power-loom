# v3.6 Wave 1 — the manage-layer lifecycle read CONSUMER

## Context

v3.5 shipped the manage-proposal store (a PRODUCER of `approved` destructive PROPOSALS) but its consumer is
unbuilt — `approved` is recorded-not-executed and **nothing reads it on a live path**. The v3.6 recon
(below) found that minting a committed `SUPERSEDE`/`TOMBSTONE` now would be **inert** (no live reader of
`projectLifecycleState`) — the ADR-0012 inert-theater smell. Per the USER's consumer-first decision
(2026-06-08, mirroring the v3.4 reshape), this wave builds the **live read consumer** that surfaces the
manage-layer's verdict on a record (advisory, shadow) — closing the v3.5 dark-producer edge and laying the
ready consumer the next wave's destructive mint will feed. **No mint, no enforce, no deletion.**

## Pre-Build VERIFY (architect, single-lens) — corrections folded

VERDICT: **NEEDS-REVISION -> corrections folded -> SOUND-WITH-FIXES.** The consumer-first STRATEGY is sound,
and `manageLifecycleStatus` is **genuinely-new** (the cross-layer join of the kernel's committed lifecycle +
the manage-layer's approved intent is the bridge the W2 mint feeds — NOT thin re-exposure of
`quarantinedRecords`). But the kernel-half contract was mis-specified on three axes; all folded above:

- **CRITICAL-1 (signature):** `projectLifecycleState` takes a RECORD object, not a txid (`provenance-projections.js:166-167`).
  Folded: resolve `txid->record` via `indexByTransactionId` (`provenance-walk.js:82`, lab-legal) first; `unknown` when absent.
- **CRITICAL-2 (states):** there is no `live` state (base = `active`); pre-mint, `archived` (age-only) +
  `aborted`/`informational`/`candidate` (commit_outcome) fire with ZERO destructive ops. Only `{tombstoned,
  superseded, stale}` light up on the mint, and only when co-located in `records`. Folded: test the DELTA, not a fixed string.
- **HIGH (records-provenance precondition):** the kernel-half is meaningful ONLY with the txid's run records
  supplied. DECISION (architect option b): the CLI defaults `kernel_state: 'unknown'`; **the manage-half is
  what closes the v3.5 dark edge** this wave; the kernel-half composition is built forward-correct + lib-tested
  (via injected synthetic records), and lights up in W2 when the mint + the records-wiring (the run-seam) land.
  Stated honestly, not buried as "defaults to unknown."
- **MEDIUM (narrowing-safety):** `effective` is a pure descriptive union, never a single resolved verdict; a
  unit asserts no `operation_class` value + no suppress/delete/gate key.
- **MEDIUM (sibling not superset):** `approvedOpsByRecord` is a SIBLING of `quarantinedRecords` (approved-only,
  tier-free), NOT a generalization; both kept.
- **LOW:** CLI `--txid` (not positional — `parseArgs` drops positionals); SHADOW rests on CLI-human-only +
  lab->kernel/_lib imports, not the grep alone.

Layer (K12) confirmed legal (lab->kernel/_lib, no cycle constructible). Producer-Consumer Phasing confirmed
RIGHT (the producer is the v3.5 store; this builds its missing reader — defuses the ADR-0012 inert-theater smell).
