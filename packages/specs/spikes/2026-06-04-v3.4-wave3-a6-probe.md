# Spike — v3.4 Wave 3 A6 seam probe + the "does the kernel leave shadow?" decision

> 2026-06-04. Firsthand, on main `7deec23`. ADR-0012 discipline: probe the harness seam BEFORE building on it.
> Lifecycle: persistent (decision record).

## Why this spike

MEMORY's ▶NEXT framed Wave 3 as *"capture E4 into `axioms.evolution_snapshot` at spawn-init"* and flagged: **"PROBE the A6/spawn-init hook seam FIRST — no A6 code seam exists yet; don't build on an assumed harness capability."** The spec language ("at spawn-init") is a design intent that the harness reality reshapes. This spike records what the seam actually is.

## Findings (firsthand)

1. **`axioms.evolution_snapshot` is spec-only.** `grep -rn evolution_snapshot` → every hit is in `packages/specs/**` (v6 §0a.3.1:179, Axiom 6:399/408, §3.6:323, INV-23:1632, the W1 spike); **zero in `packages/kernel/**/*.js`**. The seam is built from scratch.

2. **The capture hook fires at spawn-CLOSE, not init.** `spawn-record.js:3` + `hooks.json:197-202` → it is a `PostToolUse:Agent|Task` hook: it runs when the Agent/Task tool *returns*. There is no spawn-init capture hook. The `axioms` block (`spawn-record.js:293-302`) is the kernel's OWN record of the spawn — `evolution_snapshot` is a **net-new** sibling. **(verify-plan A-MED-2 correction)** it is NOT precedented by `policy_version`: `grep policy_version packages/kernel/spawn-state` → ZERO; `policy_version` lives only in `transaction-record.schema.json`, which governs a DIFFERENT envelope (the WAL transaction-record, `additionalProperties:false`, no `axioms` block). So (a) there is no schema-file to edit for the spawn-record envelope — its contract is `spawn-record.js` code + its test — and (b) A6 is the FIRST Lab-snapshot field in the spawn-record `axioms` block.

   **Field-name overload (verify-plan A-MED-3):** v6 reuses `axioms.evolution_snapshot` for TWO distinct contracts — Axiom 6 (the E4 reputation snapshot) and INV-23 (v6:984/1632 — the chain `prev_state_hash` MVCC pin). This wave namespaces under `evolution_snapshot.reputation`, leaving `evolution_snapshot.prev_state_hash` free for the deferred INV-23 pin. A6-reputation's `content_hash` is NOT the INV-23 chain pin; no INV-23 coverage is claimed here.

3. **The kernel CANNOT inject the snapshot into the spawn (ADR-0012).** `hooks.json:61` is a tombstone for `pre-spawn-tool-mask`, removed because a PreToolUse hook's `updatedInput` is **inert on Agent/Task spawns**. So A6 cannot push the snapshot into the spawn's LLM context at init. **A6 RECORDS which snapshot was current (an attestation in the kernel's own envelope); it does not INJECT it.** This is the central reframing — see Decision 1.

4. **The kernel reading a Lab-produced file is sanctioned, K12-clean.** v6 §3.6:323 — *"Lab writes that flow into Kernel-read corpora MUST go through the A6 snapshot mechanism."* The kernel reads the snapshot **as a data file by path**, importing no lab module — the exact mirror of W1's lab-reads-the-kernel-journal-as-data pattern. K12's containment test scans `require()`; a file-read by path is not an import.

5. **Concurrency is degenerate today.** k13-serial-enforcer = one spawn at a time (v6:1632 calls the MVCC pin "degenerate" under K13). In Pattern-A depth-1 the single-threaded orchestrator is *blocked on the Agent tool* between a spawn's init and close → the snapshot file cannot change mid-spawn → init-snapshot == close-snapshot → recording at close is sound. Pattern B (v3.5+, v6:412) breaks this and needs the orchestrator-carries-envelope carrier.

## Decision 1 — Record-not-inject (forced by ADR-0012)

A6 = the kernel writes `axioms.evolution_snapshot` into its OWN spawn-record envelope at close, as an axiom-class **provenance attestation** (content-hash pin + bounded inline value + watermark). It does NOT mutate the spawn. The actual pre-spawn consumption of reputation is the orchestrator's **advisory read** of the same snapshot file (the `reputation snapshot` CLI). This is the only design ADR-0012 permits, and it is exactly the v6:179 carve-out (a derived view promoted to axiom-class by being snapshotted) read through the harness's real capabilities.

## Decision 2 — The kernel STAYS shadow (the forced decision, resolved)

Wave 3 was flagged as forcing "does the kernel leave shadow?". **Resolution: NO.** A6 *records* the snapshot and the wire *advises* the orchestrator (v6:173 "MAY recommend"); reputation does **not** enter a K9 gate (no reject-on-reputation). Rationale:

- **§0a.3.1 compliance is satisfied without gating.** The carve-out (179) permits the snapshot as an axiom-class INPUT; it does not require the kernel to GATE on it. Recording + advising is the minimal coherent A6.
- **Leaving shadow is a separately-scoped high-risk phase.** Reputation→K9-reject is the `LOOM_RESOLVER_ENFORCE`-class change (the enforce-path exists but is OFF). It needs its own design + breakers + a 3-lens pass, per the v3.4 design-input (a) "symmetric guard activation — never wire a live gate without circuit breakers."
- **Conservative + reversible.** Building the mechanism in shadow lets the loop (W0→W1→W2→A6) close end-to-end and be observed before any enforcement. Consistent with the v3.4 opening cumulative-coherence verdict ("build the CONSUMER before the amplifier; kernel stays shadow for v3.4").

## Decision 3 — The materialize TRIGGER is on-demand this wave

The mechanism (materializer + reader + record + advisory wire) ships; the production trigger (a session-start hook or the W1 batched-enricher tail that refreshes the snapshot) is a deliberate ACTIVATION deferred with breakers. Until wired, the snapshot is absent/stale by default → the reader fails open → reputation-blind (the safe default). This keeps the wave honest (the primitive is LIVE-but-its-trigger-INERT, the E1 precedent).

## Coherence note

The chain closes in shadow: real spawn → (W1) evidence-linked verdict → (W2) E4 distribution → (A6) snapshot recorded as a spawn axiom + advisory-read for routing. Hash via the W0 `canonical-json` leaf; reader is the 4th `kernel/_lib` extract-to-leaf (canonical-json, recency-decay, jsonl-read, evolution-snapshot-read). No layer reaches past the A6 boundary. The kernel records; the wire advises; nothing gates.
